## 1. 뉴스 요약

ETRI(한국전자통신연구원)가 미션크리티컬 AI 시장 공략을 본격화하며, 투비유니콘과 손잡고 '고신뢰 LLM' 개발에 나섰다. 미션크리티컬 AI는 의료 진단, 국방, 자율주행, 금융 거래 등 오류가 치명적 결과로 이어지는 영역에서 요구되는 고신뢰·고안전 AI 시스템을 의미한다. 별도로 핀다-업스테이지는 금융 도메인 특화 LLM 기반 서비스를 확대하며 도메인 전문 AI의 상용화를 추진 중이다. LLM의 환각(Hallucination) 문제 해결과 신뢰성 보장이 산업 적용의 핵심 과제로 부상하고 있다.

## 2. 핵심 개념

### 미션크리티컬 시스템의 AI 신뢰성 요구사항

| 요구사항 | 설명 | 기존 SW 기준 | AI 시스템 과제 |
|----------|------|-------------|---------------|
| 정확성(Accuracy) | 올바른 결과 출력 | 결정론적 검증 가능 | 확률적 출력, 검증 곤란 |
| 신뢰성(Reliability) | 일관된 성능 유지 | MTBF 기반 정량화 | 분포 외 데이터에 취약 |
| 가용성(Availability) | 장애 시 서비스 지속 | 이중화/페일오버 | 모델 서빙 인프라 복잡 |
| 설명가능성(Explainability) | 판단 근거 제시 | 로직 추적 가능 | 블랙박스 모델 한계 |
| 안전성(Safety) | 위험 상황 방지 | 안전 무결성 등급(SIL) | AI 안전 표준 미정립 |

### LLM 환각(Hallucination) 유형과 대응 기술

```
[LLM 환각 유형]
├── 사실적 환각(Factual Hallucination)
│   └── 존재하지 않는 사실을 생성
├── 충실도 환각(Faithfulness Hallucination)
│   └── 입력 컨텍스트와 불일치하는 응답
└── 논리적 환각(Logical Hallucination)
    └── 추론 과정의 논리적 오류

[대응 기술]
├── RAG(Retrieval-Augmented Generation)
│   └── 외부 지식 기반 검색 + 생성 결합
├── Self-Consistency / Chain-of-Verification
│   └── 다중 추론 경로 교차 검증
├── Guardrails / Output Validation
│   └── 출력 필터링 및 규칙 기반 검증
├── Constitutional AI / RLHF
│   └── 학습 단계에서 안전성 내재화
└── 근거 기반 생성(Grounded Generation)
    └── 출처 명시 및 인용 기반 응답
```

### 고신뢰 LLM 아키텍처 설계 패턴

```
[사용자 질의]
     ↓
┌────────────────────────────────────────┐
│  ① 입력 검증 계층(Input Guardrail)      │
│  - 프롬프트 인젝션 탐지                  │
│  - 도메인 범위 검증                      │
└────────────────┬───────────────────────┘
                 ↓
┌────────────────────────────────────────┐
│  ② RAG 파이프라인                       │
│  - 도메인 지식 DB 검색                   │
│  - 관련 문서 컨텍스트 주입               │
└────────────────┬───────────────────────┘
                 ↓
┌────────────────────────────────────────┐
│  ③ LLM 추론 + CoT(Chain-of-Thought)    │
│  - 단계별 추론 과정 명시                 │
│  - 불확실성 수치(Confidence Score) 산출  │
└────────────────┬───────────────────────┘
                 ↓
┌────────────────────────────────────────┐
│  ④ 출력 검증 계층(Output Guardrail)     │
│  - 사실 검증(Fact-checking)             │
│  - 근거 문서 매핑 및 인용 확인           │
│  - 안전성 필터링                         │
└────────────────┬───────────────────────┘
                 ↓
[신뢰도 등급 부여된 응답 + 근거 출처]
```

### 도메인 특화 LLM 구축 방법론

| 방법 | 설명 | 비용 | 도메인 적합성 |
|------|------|------|-------------|
| 사전학습(Pre-training) | 도메인 코퍼스로 처음부터 학습 | 매우 높음 | 매우 높음 |
| 지속 사전학습(CPT) | 범용 모델에 도메인 데이터 추가 학습 | 높음 | 높음 |
| 파인튜닝(SFT) | 도메인 태스크 데이터로 미세조정 | 중간 | 중간~높음 |
| RAG | 외부 도메인 지식 DB 연동 | 낮음 | 중간 |
| 프롬프트 엔지니어링 | 도메인 컨텍스트를 프롬프트로 주입 | 매우 낮음 | 낮음~중간 |

## 3. 기술사 출제 포인트

### 예상 문제
1. **미션크리티컬 시스템에 AI를 적용할 때의 신뢰성 요구사항을 정의하고, 기존 소프트웨어 안전성 표준(IEC 61508)과의 차이점을 분석하시오**
2. **LLM의 환각(Hallucination) 현상을 유형별로 분류하고, 이를 해결하기 위한 기술적 접근 방법을 설명하시오**
3. **도메인 특화 LLM 구축 방법론을 비교하고, 금융·의료 등 규제 산업에서의 AI 신뢰성 확보 방안을 제시하시오**

### 답안 구성 가이드
- 미션크리티컬 시스템의 정의와 안전 무결성 등급(SIL 1~4) 체계
- AI 시스템의 비결정론적(Non-deterministic) 특성과 기존 V&V 방법론의 한계
- 환각 유형별 원인 분석: 학습 데이터 편향, 디코딩 전략, 지식 경계 문제
- RAG + Guardrails + 사실 검증의 다층 방어 아키텍처
- 설명가능 AI(XAI)와 감사 가능성(Auditability)의 규제 요구사항
- EU AI Act, 국내 AI 기본법의 고위험 AI 분류와 규제 프레임워크

## 4. 관련 토픽 연계

- **AI 거버넌스**: 고위험 AI 시스템에 대한 규제 프레임워크와 인증 체계
- **설명가능 AI(XAI)**: 블랙박스 모델의 판단 근거 제시 기술
- **소프트웨어 안전성**: IEC 61508, DO-178C 등 안전 무결성 표준의 AI 확장
- **RAG 아키텍처**: 외부 지식 기반 검색 증강 생성의 설계와 최적화
- **AI 품질 보증**: AI 모델의 테스트, 검증, 모니터링 체계
