## 1. 뉴스 요약

한국경제·KB Think·네이트·지디넷코리아 등 다수 매체가 동시 보도한 바에 따르면, 과학기술정보통신부는 **2026년 상반기 중 「에이전틱 AI(Agentic AI) 생태계 발전 전략」**을 수립하고, 이를 위한 **민관 합동 워크숍**을 개최했다. 4월 초 출범한 「에이전틱 AI 얼라이언스」가 산업 생태계 협력체였다면, 이번 발전 전략은 **국가 단위 AI 에이전트 표준·프로토콜·거버넌스**를 정립하는 정책 프레임워크다. 핵심 의제는 ① **에이전트 간 상호운용성(Interoperability) 표준** 마련(MCP, A2A 등), ② **멀티에이전트 오케스트레이션** 레퍼런스 아키텍처 정립, ③ **에이전트 SDLC(Software Development Life Cycle)** 가이드라인 정비, ④ **에이전트 안전·평가·책임 체계** 구축 등이다. 어도비가 같은 시점에 "에이전틱 AI로 마케팅 워크플로 주도권"을 발표한 것처럼, 글로벌 빅테크는 이미 에이전트 기반 워크플로 자동화로 이동 중이며, **에이전틱 AI는 LLM 응용 SW 개발의 주류 패러다임**으로 자리잡고 있다. 소프트웨어공학 관점에서 이는 **"객체지향 → 마이크로서비스 → 에이전트지향"** 으로 이어지는 SW 아키텍처 패러다임 전환의 새 변곡점이다.

## 2. 핵심 개념

### 에이전틱 AI 시스템의 4계층 참조 아키텍처

```
┌────────────────────────────────────────────────┐
│ ① Orchestration Layer                          │
│   Plan-Reflect-Act 루프, Workflow Graph, Memory │
├────────────────────────────────────────────────┤
│ ② Agent Layer                                   │
│   Specialist Agents, Role Prompt, Persona      │
├────────────────────────────────────────────────┤
│ ③ Tool/Protocol Layer                           │
│   MCP, A2A, Function Calling, RAG, Code Exec   │
├────────────────────────────────────────────────┤
│ ④ Foundation Model Layer                        │
│   LLM/VLM, Reasoner, Embedding, Guardrail       │
└────────────────────────────────────────────────┘
```

### 멀티에이전트 시스템(MAS) 아키텍처 패턴 비교

| 패턴 | 구조 | 적합 도메인 | 제약 |
|------|------|-----------|------|
| Supervisor-Worker | 중앙 조정자가 작업 분배 | 콜센터·민원처리 | 단일 장애점 |
| Hierarchical | 계층형 매니저 에이전트 | 복잡 의사결정 | 지연 누적 |
| Network/P2P | 에이전트 간 직접 통신 | 협업·협상 | 일관성·디버깅 난이도 |
| Pipeline | 단계별 순차 처리 | ETL·문서처리 | 유연성 부족 |
| Swarm | 분산 자율 협업 | 탐색·시뮬레이션 | 수렴 보장 어려움 |

### 에이전트 상호운용성 핵심 표준

- **MCP(Model Context Protocol)**: 앤트로픽 주도, LLM ↔ 외부 도구·데이터 표준 인터페이스
- **A2A(Agent-to-Agent)**: 구글 주도, 에이전트 간 통신·태스크 위임 프로토콜
- **OpenAPI/JSON Schema**: 도구 명세 표준, Function Calling 호환
- **W3C DID/VC**: 에이전트 신원·권한 인증 (검토 단계)
- **AgentCards**: 에이전트 역량·SLA 메타데이터 (제안 단계)

### 에이전트 SDLC 6단계

기존 LLM 기반 SW SDLC와 달리, 에이전트 SDLC는 다음 단계가 강조된다.

1. **태스크 분석**: 의사결정 그래프 설계, 도구 인벤토리 식별
2. **에이전트 설계**: 역할·페르소나·정책·메모리 구조 정의
3. **도구·프로토콜 통합**: MCP/A2A 인터페이스 구현
4. **시뮬레이션·평가**: 에이전트 벤치(GAIA, AgentBench), 시나리오 회귀
5. **가드레일·안전성 검증**: 도구 호출 권한, 액션 격리, 휴먼-인-루프
6. **운영·관측**: LLM 옵저버빌리티, 토큰·지연·비용 추적, 트레이싱

## 3. 기술사 출제 포인트

- **에이전틱 AI의 정의·필요성과 기존 생성형 AI·RPA와의 차별점**: 자율성·도구사용·반복 실행
- **멀티에이전트 시스템(MAS) 아키텍처 패턴 비교와 도메인별 적합성**: Supervisor·Hierarchical·Swarm 등
- **에이전트 상호운용성 표준(MCP·A2A·OpenAPI) 비교와 도입 시 고려사항**: 표준 채택 전략
- **에이전트 SDLC 6단계와 기존 SW SDLC와의 차이점**: 시뮬레이션·가드레일·관측 강화
- **에이전틱 AI의 안전·평가·책임 체계 설계 방안**: 가드레일, 휴먼-인-루프, 권한 격리, 회귀 테스트
- **에이전트 오케스트레이션의 4계층 참조 아키텍처와 핵심 컴포넌트**: Plan-Act-Reflect 루프
- **국가 차원의 에이전틱 AI 발전 전략의 4대 축과 거버넌스 구조**: 표준·아키텍처·SDLC·안전
- **에이전트 기반 워크플로 도입 시 보안·프라이버시·비용 측면 고려사항**: Tool Use 권한, 데이터 노출, 토큰 비용 관리

## 4. 관련 토픽 연계

- **에이전틱 AI 얼라이언스와 산업 생태계 전략**: 민관 협력체와 정책 프레임워크 정렬
- **AI 보안 표준화와 에이전틱 AI 위협 대응 체계**: 에이전트 특유 위협(Tool Misuse, Prompt Injection 2.0)
- **앤트로픽 미토스와 보안 특화 AI 에이전트**: 에이전트 보안 활용 사례
- **LLM 옵저버빌리티와 생성형AI 운영관리 플랫폼**: 에이전트 운영 단계 관측
- **바이브 코딩 열풍과 AI 생성 코드 품질 거버넌스**: 에이전트 코드 생성 SDLC
- **과기정통부 AI 어벤저스와 공공부문 시민개발자 에이전틱 AI SDLC**: 공공 적용 사례
- **에이전틱 AI 데이터베이스와 자율운영 DB 기술**: 에이전트의 데이터 계층 자동화
