## 1. 뉴스 요약

정부가 AI의 사고·판단 능력 강화를 위해 추론 데이터 10종 구축에 나선다. 기존 단순 패턴 인식용 학습 데이터를 넘어, LLM의 논리적 추론과 피지컬 AI의 물리 세계 이해를 위한 고품질 추론 데이터 확보가 핵심 과제로 부상했다. 엔코아-업스테이지는 'AI 레디 데이터 플랫폼' 기반 국내외 공동 진출을 추진하며, 데이터 품질이 AI 경쟁력의 핵심 차별화 요소임을 입증하고 있다.

## 2. 핵심 개념

### AI 학습 데이터 유형 분류

```
┌─────────────────────────────────────────────────┐
│           AI 학습 데이터 계층 구조                │
├─────────────────────────────────────────────────┤
│                                                 │
│  Level 1: 원시 데이터 (Raw Data)                │
│  ├─ 텍스트, 이미지, 음성, 영상, 센서            │
│  ├─ 웹 크롤링, 공공 데이터, 기업 내부 데이터     │
│  └─ 품질: 낮음 / 양: 대규모                     │
│         ↓ 전처리·정제                            │
│  Level 2: 레이블링 데이터 (Labeled Data)         │
│  ├─ 분류·객체탐지·개체명인식 등 태깅             │
│  ├─ 인간 어노테이터 / 자동 레이블링              │
│  └─ 품질: 중간 / 양: 중규모                     │
│         ↓ 구조화·맥락화                          │
│  Level 3: 추론 데이터 (Reasoning Data)           │
│  ├─ 사고 과정(Chain-of-Thought) 포함             │
│  ├─ 논리적 단계, 판단 근거, 반례 포함            │
│  ├─ 수학·과학·법률·의료 도메인 전문 추론          │
│  └─ 품질: 높음 / 양: 소규모(고비용)              │
│         ↓ 물리 세계 연동                         │
│  Level 4: 체화 데이터 (Embodied Data)            │
│  ├─ 로봇·자율주행·디지털트윈 행동 데이터          │
│  ├─ 물리 법칙, 공간 추론, 인과 관계              │
│  └─ 품질: 최고 / 양: 극소규모(희소)              │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 추론 데이터 구축 방법론

| 구축 방법 | 설명 | 장점 | 한계 |
|----------|------|------|------|
| 전문가 직접 작성 | 도메인 전문가가 추론 과정을 단계별 기술 | 최고 품질, 도메인 정확성 | 고비용, 확장 어려움 |
| CoT 증류(Distillation) | 대형 모델의 추론 과정을 소형 모델 학습용으로 추출 | 비용 효율적, 대규모 가능 | 원본 모델 의존, 품질 편차 |
| 자기 학습(Self-Play) | AI가 문제를 풀고 검증하는 반복 과정으로 데이터 생성 | 자동화 가능, 난이도 조절 | 편향 증폭, 검증 필요 |
| RLHF/RLAIF | 인간/AI 피드백 기반 선호 데이터 구축 | 정렬(Alignment) 효과 | 보상 해킹 위험 |
| 시뮬레이션 합성 | 물리 시뮬레이터로 체화 추론 데이터 생성 | 안전, 다양한 시나리오 | 현실 격차(Sim-to-Real Gap) |

### AI 레디 데이터(AI-Ready Data) 성숙도 모델

| 성숙도 단계 | 특징 | 데이터 활용 수준 |
|-----------|------|----------------|
| Level 1: Ad-hoc | 비정형, 사일로, 수작업 관리 | 단순 보고·조회 |
| Level 2: Managed | 표준화, 메타데이터 관리 시작 | 기술 통계·BI |
| Level 3: Defined | 품질 기준 수립, 데이터 카탈로그 | ML 모델 학습 가능 |
| Level 4: Quantified | 자동 품질 모니터링, 리니지 추적 | LLM 파인튜닝·RAG |
| Level 5: Optimized | AI 기반 자동 정제, 지속 품질 개선 | 추론 데이터·체화 AI |

## 3. 기술사 출제 포인트

### 예상 문제
- **AI 학습 데이터의 유형을 분류하고, 추론 데이터(Reasoning Data)의 구축 방법론과 품질 관리 전략을 제시하시오**
- **데이터 품질 관리의 6대 차원을 설명하고, AI 레디 데이터(AI-Ready Data) 구축을 위한 데이터 거버넌스 방안을 논하시오**
- **LLM의 추론 능력 향상을 위한 Chain-of-Thought 데이터 구축과 RLHF 기반 정렬(Alignment) 전략을 설명하시오**

### 답안 작성 키워드
- 추론 데이터: CoT(Chain-of-Thought), 단계별 사고 과정 포함
- 데이터 품질 6대 차원: 정확성, 완전성, 일관성, 적시성, 대표성, 추적가능성
- RLHF(Reinforcement Learning from Human Feedback): 선호 정렬
- 데이터 리니지(Data Lineage): 출처부터 활용까지 이력 추적
- AI 레디 데이터: 전처리·라벨링·품질 검증 완료 상태

## 4. 관련 토픽 연계

| 연계 토픽 | 연관성 |
|----------|--------|
| 데이터 레이크하우스 | AI 학습 데이터 통합 저장·관리 플랫폼 |
| LLM 추론 비용 최적화 | 고품질 추론 데이터로 소형 모델 성능 극대화 |
| 데이터 거버넌스 | AI 데이터 품질·보안·컴플라이언스 통합 관리 |
| 마이데이터(MyData) | 개인정보 기반 AI 학습 데이터 활용 규제 |
| 디지털 트윈 | 피지컬 AI용 시뮬레이션 데이터 생성 |
| AI 모델 양자화·경량화 | 추론 데이터 증류(Distillation)와 연계 |
| 합성 데이터(Synthetic Data) | 추론 데이터 부족 문제 해결 기법 |
