## 1. 뉴스 요약

테크월드·지디넷코리아·디일렉·이데일리·네이트·한국경제 등 6개 이상 매체가 보도한 바에 따르면 메가존클라우드가 APM(Application Performance Monitoring) 전문기업 데이터독(Datadog)과 첫 번째 APJ(아시아태평양·일본) 지역 전략 파트너십을 체결했다. 이는 단순한 총판 제휴가 아니라 엔터프라이즈가 생성형 AI·LLM을 프로덕션에 도입한 이후 발생하는 **운영 가시성(Observability)·비용·품질 문제**를 해결하기 위한 통합 플랫폼 공략이다. 데이터독은 2025년부터 'LLM Observability' 제품군을 정식 출시하여 프롬프트·응답·토큰 사용량·지연시간·환각(Hallucination)·안전성을 엔드투엔드로 추적할 수 있게 했고, 메가존은 AWS·Azure·GCP 등 멀티클라우드 기반 AI 워크로드 운영에 이를 결합하여 한국 대기업·공공의 생성형 AI 운영관리(LLMOps) 시장을 공략한다. 이번 협력은 국내 시장이 'LLM 개발' 단계에서 'LLM 운영' 단계로 본격 전환되었음을 보여주는 사건이다.

## 2. 핵심 개념

### 전통 APM vs LLM Observability의 차이

```
┌────────────────────────────────────────────────────────────────┐
│ 전통 APM (Application Performance Monitoring)                  │
│  - 메트릭: 응답시간, 처리량(TPS), 에러율, CPU/메모리           │
│  - 트레이싱: HTTP 요청 → 서비스 → DB 호출                      │
│  - 로그: 구조화된 텍스트                                       │
├────────────────────────────────────────────────────────────────┤
│ LLM Observability (생성형 AI 관측성)                           │
│  - 메트릭: 토큰 사용량, TTFT(Time-To-First-Token), TPOT        │
│  - 품질: 환각률, 유해성(Toxicity), 정확도, 사용자 피드백       │
│  - 비용: 모델별·세션별·테넌트별 과금 추적                     │
│  - 트레이싱: 프롬프트 → RAG 검색 → LLM 호출 → 후처리 체인      │
│  - 안전성: 프롬프트 인젝션·데이터 유출·PII 감지                │
└────────────────────────────────────────────────────────────────┘
```

### LLMOps 참조 아키텍처

- **데이터 파이프라인 계층**: 벡터DB(Pinecone/Milvus/pgvector), 임베딩 서비스, 피드백 수집 스토어
- **모델 서빙 계층**: vLLM·TGI·Triton, 멀티모델 라우팅, A/B 테스트
- **오케스트레이션 계층**: LangGraph·LlamaIndex·MCP 기반 에이전트 파이프라인
- **관측·평가 계층**: LLM-as-a-Judge, RAGAS, 프롬프트 회귀 테스트, LangSmith·Datadog LLM Obs
- **거버넌스 계층**: 가드레일(Guardrails AI·NeMo), 감사 로그, 모델 카드·데이터 카드

### 핵심 지표(SLI/SLO)

다음은 LLM 서비스 운영에 필수적인 품질 지표와 권장 SLO 수준이다.

| 지표 유형 | 지표명 | 정의 | 권장 SLO |
|----------|-------|------|---------|
| 성능 | TTFT | 첫 토큰 응답시간 | < 500ms (챗봇) |
| 성능 | TPOT | 토큰당 생성시간 | < 50ms |
| 품질 | Faithfulness | 컨텍스트 충실도(RAG) | > 0.85 |
| 품질 | Hallucination Rate | 환각 응답 비율 | < 3% |
| 비용 | Cost per Session | 세션당 토큰 비용 | 예산 ±10% |
| 안전 | Prompt Injection Detect | 악성 프롬프트 차단율 | > 99% |

## 3. 기술사 출제 포인트

### 예상 문제 유형

- **25점 단답/약술**: LLMOps의 개념과 MLOps와의 차이점을 설명하시오.
- **25점 논술**: 엔터프라이즈 생성형 AI 서비스의 운영관리 체계(옵저버빌리티, 가드레일, 비용관리)를 설계하시오.
- **25점 아키텍처**: RAG 기반 챗봇의 품질·비용·안전성을 동시에 확보하기 위한 LLM Observability 플랫폼 아키텍처를 제시하시오.
- **도메인 연계**: 공공기관 생성형 AI 도입 시 SLA·SLO 수립 방안을 논하시오.

### 답안 핵심 키워드

- **LLMOps 5대 영역**: ①프롬프트·모델 버전관리, ②품질평가(LLM-as-a-Judge), ③관측성, ④가드레일, ⑤비용 최적화
- **옵저버빌리티 3요소(Three Pillars)**: Metrics, Logs, Traces → LLM 도메인에서는 **Evaluations(평가)·Prompts(프롬프트)·Costs(비용)** 3요소가 추가됨
- **품질 관리 기법**: RAGAS, DeepEval, TruLens, LangSmith, Datadog LLM Obs
- **비용 관리**: Prompt caching, 모델 라우팅(작은 모델 우선), 토큰 예산 할당, FinOps for AI
- **회귀 방지**: Prompt Regression Test, Golden Dataset, Shadow Deployment

### 도메인별 적용 시나리오

- **금융**: 웹케시 오페리아와 같은 금융 AI 에이전트 → 규제 대응 감사로그, 거래 관련 환각 탐지
- **공공**: 공공기관 LLM AI상담(나이스콜) → 민감정보 유출 방지, 응답 정확도 모니터링
- **의료**: AI 특화병원 AX 레디 → FDA/식약처 요구 추적성 확보

## 4. 관련 토픽 연계

- **DevOps/SRE**: 에러 버짓(Error Budget), 카나리 배포 → LLM은 프롬프트·모델·데이터 3축 카나리
- **AIOps**: 범용 AI 이상탐지 플랫폼 ↔ LLM 운영 이상탐지(드리프트 감지)
- **AI 거버넌스**: AI 기본법, NIST AI RMF, ISO/IEC 42001 AIMS → 운영 단계 증적 확보
- **AI 신뢰성 검증**: AI SW 품질관리 역량강화 사업 → 운영 단계 지속 검증 체계로 확장
- **에이전틱 AI SDLC**: 시민개발자가 만든 에이전트의 운영 단계 가드레일·비용 통제
- **FinOps**: AI 서비스 경제학 ↔ 토큰·GPU 예산 할당과 청구 모델

**결론**: 2026년 엔터프라이즈 AI는 '모델을 만드는 시대'에서 '모델을 운영하는 시대'로 이동했다. 기술사 수험자는 LLMOps·LLM Observability가 단순 모니터링 툴이 아니라 **품질·비용·안전·규제 대응을 통합하는 거버넌스 플랫폼**이라는 관점에서 아키텍처와 거버넌스 답안을 준비해야 한다.
