## 1. 뉴스 요약

시사일보·헤럴드경제·네이트·퍼블릭타임스 등 보도에 따르면, **조달청이 「AI 기술 기반 지능형 예정가격 작성지원시스템」을 개통**했다. 핵심은 ① **물품·용역·공사 발주 시 담당자가 시장가격·품목 코드·과거 낙찰 데이터·계약 단가를 일일이 검색·취합하던 작업을 AI가 추천·자동 산출**, ② **나라장터·국가종합전자조달 시스템과 연계해 입력 단계부터 가격 근거(데이터 출처)를 자동 첨부**, ③ **예정가격 산정 결과의 적정성·일관성·감사 추적성을 강화**한다는 점이다. 배경에는 **공공 발주의 가격 산정 오류·과도한 낙찰차·감사 지적·담당자 업무 과중** 등이 있다. 같은 기간 **기획예산처 AI 예산편성**, **행안부 온AI 모바일** 등 정부 부처 AI 전환이 동시 진행 중이다. 소프트웨어공학 관점에서 본 시스템은 **공공 미션크리티컬 SW**로서 ① **요구사항·데이터 사전조사 → 개발 → 감리 → 운영**의 공공 SDLC, ② **AI 추천·결정에 대한 검증·예외 처리·휴먼 인 더 루프**, ③ **AI 결과의 법적 책임·감사 대응**, ④ **데이터 품질·MLOps·재학습 거버넌스**를 종합적으로 적용해야 하는 사례다.

## 2. 핵심 개념

### 지능형 예정가격 작성지원시스템 처리 흐름

```
┌──────────────────────────────────────────────────────────────┐
│ ① 발주 요청 입력 : 품목·수량·납기·공사 내역서                │
├──────────────────────────────────────────────────────────────┤
│ ② 데이터 수집·정합 : 나라장터·통계·시장가·과거 낙찰          │
├──────────────────────────────────────────────────────────────┤
│ ③ AI 추천 : 가격 분포·이상치·유사 사례 검색(RAG)             │
├──────────────────────────────────────────────────────────────┤
│ ④ 근거 자동 첨부 : 출처·산출 공식·신뢰구간                   │
├──────────────────────────────────────────────────────────────┤
│ ⑤ 담당자 검토·확정 : 휴먼 인 더 루프, 사유 기재               │
├──────────────────────────────────────────────────────────────┤
│ ⑥ 감사·통계 : 추천 vs 확정 차이, 패턴 분석, 재학습            │
└──────────────────────────────────────────────────────────────┘
```

### 기존 예정가격 산정 vs AI 지원 시스템

다음 표는 두 모델의 차이를 정리한다.

| 구분 | 기존 수기·룰 기반 산정 | AI 지원 시스템 |
|------|------------------------|------------------|
| 데이터 수집 | 담당자 개별 검색 | 자동 통합 수집 |
| 산정 근거 | 문서·메모로 분산 | 시스템 자동 첨부 |
| 일관성 | 담당자별 편차 | 표준 모델·신뢰구간 |
| 이상치 탐지 | 사후 감사 | 입력 시 실시간 경고 |
| 감사 추적 | 사후 재현 어려움 | 모든 추천·근거 로그 |
| 재학습 | 룰 수동 갱신 | 데이터 기반 주기 학습 |

### 공공 AI SW 개발 핵심 구성요소

- **요구공학**: 발주·계약·감사·법규 요구사항을 정형화, 정책 변경 추적
- **데이터 파이프라인**: 나라장터·통계청·시장조사·과거 낙찰 ETL/CDC
- **AI 모델**: 가격 회귀·이상치 탐지·유사 사례 RAG, 설명가능성 우선
- **MLOps·거버넌스**: 모델 레지스트리·드리프트 감지·재학습·승인
- **휴먼 인 더 루프**: 추천 수용·반려 사유 기록, 의사결정 책임 명확화
- **공공 SDLC·감리**: 정보시스템 감리, 보안성 검토, CSAP·N2SF 준수
- **변경관리**: 정책·법규 변경 시 영향분석, 회귀 테스트, 운영 핸드오버

### 도입 시 고려사항

- **데이터 품질**: 입력 데이터 결측·이상값이 가격 추천 왜곡 → 데이터 품질 KPI
- **편향·담합 방지**: 특정 업체·지역·산업 편향 모니터링, 공정거래 검토
- **법적 책임**: AI 추천 오류로 예정가격 오산 시 책임 소재(부처·공급사·운영사)
- **감사·투명성**: 감사원·국정감사 대응을 위한 추천 근거·로그 보존
- **확장성**: 물품·용역에서 공사·연구 R&D·민간조달까지 단계적 확대
- **휴먼 역량**: 담당자의 AI 결과 비판적 검토 역량·교육 체계

## 3. 기술사 출제 포인트

- 공공 AI 의사결정 지원 시스템의 정의와 필요성
- 기존 수기 산정 방식과 AI 지원 시스템의 비교
- 공공 AI SW 개발 SDLC와 정보시스템 감리·CSAP 적용
- AI 추천 결과의 설명가능성(XAI)·근거 자동 첨부 설계
- 데이터 품질 관리(완전성·정확성·일관성)와 데이터 파이프라인 설계
- MLOps 기반 모델 드리프트 감지와 주기 재학습 거버넌스
- 휴먼 인 더 루프(HITL) 의사결정 구조와 책임 추적 설계
- 공공 SW 변경관리·회귀 테스트·운영 인수 전략
- AI 시스템 편향·담합·공정거래 위험 관리 체계
- 공공 AI 도입의 법적 책임·감사 대응 거버넌스
