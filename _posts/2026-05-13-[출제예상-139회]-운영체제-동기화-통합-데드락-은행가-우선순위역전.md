## 2. 개념·정의

### 임계구역(Critical Section)
> 공유 자원에 접근하는 코드 영역. **상호배제(Mutual Exclusion)·진행(Progress)·한정대기(Bounded Waiting)** 3조건 만족 필요.

### 교착상태(Deadlock)
> 둘 이상의 프로세스가 서로의 자원을 기다리며 영원히 진행 불가한 상태.

### 은행가 알고리즘(Banker's Algorithm)
> Dijkstra 제안. 자원 할당 전 **안전 상태(Safe State)**인지 검사해 교착 회피.

### 우선순위 역전(Priority Inversion)
> 고우선순위 프로세스가 저우선순위 프로세스의 자원 대기로 실행 지연되는 현상. **화성탐사선 패스파인더 사례**로 유명.

### 가상메모리(Virtual Memory)
> 물리 메모리보다 큰 논리 주소공간 제공. **페이징·세그멘테이션·디맨드 페이징·페이지 교체**.

## 4. 핵심 기술요소

### 4.1 교착 4대 발생 조건 (Coffman)

| 조건 | 설명 |
|------|------|
| 상호배제 (Mutual Exclusion) | 자원을 동시 사용 불가 |
| 점유와 대기 (Hold and Wait) | 자원 보유 + 추가 대기 |
| 비선점 (No Preemption) | 강제 회수 불가 |
| 순환대기 (Circular Wait) | 자원 대기 순환 |

### 4.2 교착 대응 4전략

| 전략 | 방법 |
|------|------|
| **예방(Prevention)** | 4조건 중 하나 부정 (자원 일괄 할당, 자원 번호 순서) |
| **회피(Avoidance)** | 은행가 알고리즘 (Safe State 확인) |
| **탐지(Detection)** | 자원할당그래프 사이클 탐지 |
| **복구(Recovery)** | 프로세스 종료·자원 선점·롤백 |

### 4.3 은행가 알고리즘 동작

```
입력:
  Available[m]        : 남은 자원 벡터
  Max[n][m]           : 최대 요청
  Allocation[n][m]    : 현재 할당
  Need[n][m] = Max - Allocation

요청 R 도착 시:
  1. R ≤ Need[i] 인가?  → No: 오류
  2. R ≤ Available 인가? → No: 대기
  3. 가상 할당 후 Safe State 검사
     - Work = Available - R
     - Finish[] = false
     - 반복: Need[j] ≤ Work 인 j 있으면 Work += Allocation[j], Finish[j] = true
     - 모두 Finish == true 면 Safe → 할당
     - 아니면 Unsafe → 거부·대기
```

### 4.4 우선순위 역전 해결

| 기법 | 동작 |
|------|------|
| **PIP** (Priority Inheritance Protocol) | 자원 보유 저우선순위 프로세스에 일시 고우선순위 상속 |
| **PCP** (Priority Ceiling Protocol) | 자원별 천장 우선순위 부여 |
| **Random Boosting** | 주기적 우선순위 상승 |
| **Disable Preemption** | 짧은 임계구역에서 선점 금지 |

### 4.5 가상메모리 핵심

| 구성 | 설명 |
|------|------|
| **MMU** | 가상→물리 주소 변환 |
| **TLB** | 최근 변환 캐시 |
| **페이지 테이블** | 가상 페이지 → 물리 프레임 매핑 |
| **디맨드 페이징** | 필요 시점에 페이지 적재 |
| **페이지 폴트** | 적재 안 된 페이지 접근 → 디스크 I/O |

**페이지 교체 알고리즘**:
- FIFO, LRU, LFU, Clock(Second-chance), Optimal, Working Set

**스래싱(Thrashing)** 대응:
- Working Set 모델
- PFF (Page Fault Frequency)
- 다중 프로그래밍 정도 조절

## 5. 운영 고려사항

### 5.1 실시간 시스템에서의 동기화
- **RTOS**: 우선순위 역전 치명적 → PIP/PCP 의무
- **인터럽트**: 짧은 임계구역에서 선점 비활성
- **잠금-free**: CAS·LL/SC 원자 연산

### 5.2 멀티코어 동기화
- **메모리 모델**: x86 TSO, ARM weak ordering
- **메모리 배리어**: load/store fence
- **캐시 일관성**: MESI 프로토콜

### 5.3 가상메모리 성능 튜닝
- **거대 페이지(Huge Page)**: TLB miss↓
- **NUMA aware**: 로컬 노드 할당
- **Swap**: SSD/NVMe로 swap 지연↓
- **Compressed Memory**: zswap
