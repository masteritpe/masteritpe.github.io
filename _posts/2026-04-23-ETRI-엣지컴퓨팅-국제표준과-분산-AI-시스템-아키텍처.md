## 1. 뉴스 요약

인공지능신문 보도에 따르면 한국전자통신연구원(ETRI)이 자율주행·원격의료의 핵심 기반인 '엣지 컴퓨팅' 국제표준 제정을 주도해 완료했으며, 인공지능·디지털 트윈 등 다양한 산업 분야에 파급 효과가 기대된다. 엣지 컴퓨팅은 단말 가까운 위치에서 연산·저장·AI 추론을 수행해 초저지연·대역폭 절감·데이터 주권 확보를 실현하는 분산 컴퓨팅 패러다임이며, 5G/6G·AI 도입 확산과 함께 ISO/IEC JTC1 SC38(클라우드·분산 컴퓨팅)과 ETSI MEC, ITU-T SG13 등에서 표준화 경쟁이 심화되고 있다. 이번 표준 완성은 한국이 차세대 네트워크·AI 인프라 표준 생태계에서 기술 리더십을 확보했음을 의미하며, AI 추론 엣지화·피지컬 AI 확산·스마트 팩토리 국제 상호운용성의 기반이 된다.

## 2. 핵심 개념

### 엣지 컴퓨팅 계층 아키텍처

```
┌─────────────────────────────────────────────────────────────┐
│ [Cloud / Core DC]  대규모 학습 · 전역 분석 · 정책 관리        │
│         ▲ (배치/주기적 동기화, 수십~수백 ms)                   │
├─────────────────────────────────────────────────────────────┤
│ [Regional Edge]    지역 캐시 · 공동 모델 · 집계 분석          │
│         ▲ (수~수십 ms)                                        │
├─────────────────────────────────────────────────────────────┤
│ [MEC / Access Edge] 기지국·네트워크 엣지, 실시간 AI 추론      │
│         ▲ (1~10 ms)                                           │
├─────────────────────────────────────────────────────────────┤
│ [Device / On-Device] 센서·차량·의료기기, 즉시 응답             │
│         (< 1 ms, 오프라인 동작)                                │
└─────────────────────────────────────────────────────────────┘
```

### 엣지 컴퓨팅의 4대 요구 품질(NFR)

- **초저지연(Ultra-Low Latency)**: URLLC 1ms 이내, 자율주행·촉각 인터넷·원격수술의 핵심 제약
- **대역폭 효율(Bandwidth Efficiency)**: 원천 데이터 전송 대신 엣지 전처리·모델 추론 결과만 업로드
- **자율성·회복성(Autonomy/Resilience)**: 중앙 네트워크 단절 시에도 로컬 판단 지속
- **데이터 주권·프라이버시**: 개인정보·의료정보가 경계 내에서 처리, GDPR·의료법 준수

### 주요 국제 표준 매핑

아래는 엣지 컴퓨팅 관련 핵심 국제 표준 체계이다.

| 기구 | 표준 | 범위 |
|------|------|------|
| ISO/IEC JTC1 SC38 | 22123 시리즈 | 클라우드·엣지 용어·개념·아키텍처 |
| ISO/IEC JTC1 SC41 | IoT·디지털 트윈 | 엣지 IoT 통합 참조모델 |
| ETSI MEC | ISG MEC 003/011 | 멀티 액세스 엣지 컴퓨팅 프레임워크·API |
| ITU-T SG13 | Y.3500 시리즈 | 네트워크 엣지 컴퓨팅 기능 요구사항 |
| 3GPP | TS 23.558 | 5G 엣지 애플리케이션 활성화(EDGEAPP) |
| O-RAN Alliance | xApp/rApp | RAN 인텔리전트 컨트롤러(RIC) 기반 엣지 AI |

### 분산 AI 추론 아키텍처 패턴

```
(1) On-Device Only       (2) Edge-Cloud Split      (3) Federated Edge
 ┌────────┐              ┌────────┐  ┌──────┐      ┌────────┐...[n]
 │ Device │              │ Device │─▶│ Edge │      │ Device │
 │ Model  │              │ (light)│  │ Model│      │ LocalMd│
 └────────┘              └────────┘  └──────┘      └────────┘
  예: 모바일 비전          예: AR/VR 렌더링          예: 의료 FL
  장점: 오프라인·저지연    장점: 자원 분담           장점: 프라이버시
  단점: 모델 경량화 필요   단점: 네트워크 의존       단점: 합의·집계 오버헤드
```

### 핵심 기술 스택

- **모델 경량화**: 양자화(INT8/INT4), 지식 증류, Pruning, NPU/TPU 추론 가속
- **오케스트레이션**: KubeEdge, OpenYurt, K3s, Akri 디바이스 플러그인
- **런타임**: ONNX Runtime, TensorRT, LiteRT(구 TFLite), WebAssembly(WASM) 엣지 실행
- **네트워킹**: 5G URLLC, TSN(Time-Sensitive Networking), DDS, MQTT-SN
- **보안**: 원격 검증(Remote Attestation), TEE(TrustZone·SGX), OTA 서명 업데이트

## 3. 기술사 출제 포인트

### 논술형 예상 주제

- 5G/6G 환경의 엣지 컴퓨팅 아키텍처(Cloud-Edge-Device 3계층)와 자율주행·원격의료 적용 설계 방안
- 엣지 AI 추론 시스템의 모델 경량화·오케스트레이션·보안 통합 운영 전략
- 분산 AI 환경에서 데이터 주권·프라이버시 보장을 위한 연합학습(Federated Learning)과 엣지 표준 활용 방안

### 단답형 핵심 키워드

- **MEC(Multi-access Edge Computing)**: ETSI 표준, 5G 기지국·네트워크 접점에 컴퓨팅 배치
- **URLLC(Ultra-Reliable Low-Latency Communications)**: 5G 3대 서비스 중 저지연·고신뢰, 1ms·99.9999%
- **TSN(Time-Sensitive Networking)**: IEEE 802.1, 결정적 지연 보장 이더넷
- **연합학습(Federated Learning)**: 원본 데이터 이동 없이 엣지에서 로컬 학습·중앙 집계
- **디지털 트윈(Digital Twin)**: 엣지·클라우드 계층에서 물리 객체 실시간 동기화
- **KubeEdge**: CNCF 오픈소스 엣지 쿠버네티스 플랫폼, 디바이스 트윈 지원
- **TEE(Trusted Execution Environment)**: ARM TrustZone·Intel SGX, 엣지 보안 연산
- **O-RAN RIC**: Open RAN 실시간 컨트롤러, xApp/rApp을 통한 RAN 지능화

### 비교·대조 포인트

아래는 클라우드 중심 AI와 엣지 중심 AI 아키텍처의 비교이다.

| 구분 | 중앙 클라우드 AI | 엣지 AI |
|------|------------------|----------|
| 지연 시간 | 수십~수백 ms | 1~10 ms |
| 대역폭 | 원천 데이터 업로드 | 전처리·추론 결과만 |
| 가용성 | 네트워크 의존 | 로컬 자율 동작 |
| 학습 방식 | 중앙 집중 대용량 학습 | 온디바이스 추론·연합학습 |
| 데이터 주권 | 클라우드 저장 | 경계 내 처리 |
| 모델 크기 | 수십~수천억 파라미터 | 경량·양자화 |
| 운영 | 중앙 MLOps | 분산 EdgeOps·OTA 배포 |
| 보안 위협 | 전송 구간·중앙 유출 | 물리적 탈취·모델 추출 |
