## 1. 뉴스 요약

연합뉴스와 뉴스제주에 따르면 제주특별자치도(제주관광공사)가 관광객 이동·소비 데이터를 통합 시각화하는 '제주관광 빅데이터 플랫폼'을 전면 개편했다. 개편 플랫폼은 통신사 위치 데이터, 카드사 결제 데이터, 관광 사업체 매출, 항공·선박 입도 데이터, SNS 텍스트 등 이종 데이터를 수집·통합해 관광객의 시간대·국적·목적별 이동 동선과 소비 패턴을 한눈에 보여주며, 공공·민간 사업자 모두에게 분석 도구·API를 개방한다. 단순 통계 조회를 넘어 ML 기반 수요 예측과 마케팅 의사결정까지 지원하도록 데이터 아키텍처를 재설계했다는 점이 특징이다. 이는 행정안전부·과기정통부가 추진해 온 빅데이터 플랫폼 및 센터(통신·금융·관광 등 16개 분야) 사업의 지역·도메인 특화 사례이며, 공공 데이터 거버넌스의 모범 사례로 의미가 있다.

### 데이터 통합 방식: ETL vs ELT vs CDC

아래는 빅데이터 통합 방식 비교다.

| 방식 | 처리 위치 | 적합 워크로드 | 장점 | 한계 |
|------|----------|--------------|------|------|
| ETL | 외부 ETL 엔진 | 정형 마트 적재 | 변환 표준화 | 확장 비용 |
| ELT | 데이터 웨어하우스 | 대용량 분석 | DW 성능 활용 | 변환 표준화 어려움 |
| CDC | 원천 트랜잭션 로그 | 준실시간 동기화 | 저지연·소부하 | 스키마 변경 민감 |
| 스트리밍 | Kafka·Flink | 실시간 분석 | 즉시성 | 운영 복잡 |

### 가명결합·비식별화 기법

- **가명처리(Pseudonymization)**: 개인정보 항목을 토큰·해시로 대체
- **결합 전문기관**: 개인정보보호위원회 지정 기관에서 가명정보 결합 수행(통신·카드 결합 필수 절차)
- **차분 프라이버시(DP)**: 통계 결과에 잡음을 추가해 개별 식별 위험 완화
- **k-익명성·l-다양성·t-근접성**: 그룹화 기반 재식별 위험 완화 모델
- **합성 데이터(Synthetic Data)**: 분포 보존 모형으로 가짜 데이터 생성, 분석 학습에 활용

### 데이터 메시(Data Mesh) 관점

지역·도메인 단위로 자율적인 데이터 제품(Data Product)을 구성하고, 중앙 거버넌스는 표준·검색 가능성만 책임지는 분산형 데이터 아키텍처. 관광·교통·환경 등 분야별 데이터 제품을 묶어 도시 단위 분석으로 확장 가능.

## 3. 기술사 출제 포인트

### 논술형 예상 주제

- 공공 빅데이터 플랫폼 구축 시 데이터 수집·통합·개방 단계별 설계 방안
- 가명정보 결합·비식별화 기법 및 개인정보보호법 준수 데이터 거버넌스
- 데이터 레이크하우스(Lakehouse)와 데이터 메시(Data Mesh) 비교 및 도메인 적용 전략

### 단답형 핵심 키워드

- **데이터 레이크하우스(Lakehouse)**: 데이터 레이크의 유연성 + 데이터 웨어하우스의 트랜잭션·스키마 관리 결합 (Delta Lake, Iceberg, Hudi)
- **메달리온 아키텍처**: Bronze(원본) → Silver(정제) → Gold(분석) 계층화
- **데이터 카탈로그**: 메타데이터·계보(lineage)·소유자·품질 등급을 검색·관리하는 시스템
- **가명정보 결합**: 개인정보 일부를 가명화 후 결합전문기관을 통해 다른 기관 데이터와 결합
- **H3 공간 인덱스**: 우버가 공개한 육각형 격자 공간 인덱스, 위치 데이터 집계에 활용
- **데이터 제품(Data Product)**: 도메인 팀이 책임지고 제공하는 검색·접근·이해 가능한 데이터 단위(Data Mesh)

### 비교·대조 포인트

아래는 데이터 웨어하우스, 데이터 레이크, 레이크하우스의 비교다.

| 구분 | 데이터 웨어하우스 | 데이터 레이크 | 데이터 레이크하우스 |
|------|------------------|--------------|--------------------|
| 데이터 형태 | 정형 | 정형·반정형·비정형 | 모두 |
| 스키마 | Schema-on-Write | Schema-on-Read | 둘 다 지원 |
| 트랜잭션 | ACID | 제한적 | ACID(Delta/Iceberg) |
| 비용 | 높음 | 낮음 | 중간 |
| BI 적합성 | 우수 | 부족 | 우수 |
| ML 적합성 | 부족 | 우수 | 우수 |
