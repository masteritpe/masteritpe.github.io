## 1. 뉴스 요약

경향신문·한스경제·글로벌이코노믹·다음·Benzinga 등 다수 매체가 일제히 보도한 바에 따르면 미국 국무부는 **딥시크(DeepSeek)** 가 미국 AI 기업의 모델을 무단으로 **추출(Extraction)·증류(Distillation)** 했다며 전 세계 공관에 경고 전문을 보냈다. 미국은 한·미 정상회담을 앞두고 "중국이 추출 기술로 AI 기술을 약탈했다"며 반도체·AI 모델 통제를 강화하고 있다. 이 사건의 본질은 외교 갈등이 아니라, **API로 노출된 LLM 출력만으로 모델의 지식·능력을 제3자가 복제할 수 있다는 AI 보안의 구조적 취약성**이다. 모델 증류 공격은 학습 데이터·파라미터에 직접 접근하지 않고도 비슷한 성능의 경량 모델을 만드는 기법으로, 정보보안 영역에서 **AI 모델 IP 보호·MLSecOps**의 핵심 주제가 됐다. 기존 OWASP LLM Top 10에서도 **LLM10: Model Theft**로 분류되며 기술사 시험에서 정보보안 출제 비중이 빠르게 늘 영역이다.

## 2. 핵심 개념

### 모델 증류(Knowledge Distillation)와 증류 공격의 차이

지식증류(KD)는 합법적인 모델 경량화 기법이지만, 동일한 원리가 무단으로 사용되면 **공격(Distillation Attack)** 이 된다.

```
┌─ 합법 KD ─────────────────────────────────────────────────┐
│ Teacher(자사 보유) → Soft Label(logits) → Student(자사)   │
│ 목적: 경량화·온디바이스                                   │
│ 권한: 모델 가중치 접근 가능                               │
└───────────────────────────────────────────────────────────┘
┌─ 증류 공격 ───────────────────────────────────────────────┐
│ Victim API(외부) → 대량 질의/응답 수집 → Surrogate 모델   │
│ 목적: 모델 IP 무단 복제                                   │
│ 권한: API 응답만 접근(Black-box)                          │
└───────────────────────────────────────────────────────────┘
```

블랙박스 환경에서도 **수십만~수백만 건의 프롬프트-응답 쌍**으로 충분히 능력을 모사할 수 있다는 점이 위협의 핵심이다.

### AI 모델 공격 분류와 LLM10 위협

| 공격 유형 | 목표 | 대표 기법 | 방어 |
|-----------|------|-----------|------|
| Model Extraction/Stealing | 모델 복제 | API 쿼리 → Surrogate 학습 | 쿼리 제한, 워터마킹 |
| Model Distillation Attack | 능력 전이 | Soft Label·CoT 응답 수집 | 응답 변형, 페이크 토큰 |
| Membership Inference | 학습데이터 추정 | 출력 confidence 분석 | DP-SGD, 출력 평탄화 |
| Model Inversion | 학습데이터 복원 | Gradient·output 역산 | 차분프라이버시 |
| Adversarial Example | 오분류 유도 | FGSM, PGD | Adversarial Training |
| Prompt Injection | 정책 우회 | Jailbreak, Indirect | 입력 필터, 가드레일 |
| Data Poisoning | 학습 오염 | RAG·파인튜닝 데이터 조작 | 출처 검증, 서명 |

OWASP **LLM Top 10**(2025): LLM01 Prompt Injection ~ **LLM10 Model Theft** 까지가 표준 분류이며 NIST **AI RMF·AI 600-1**과 연결된다.

### AI 모델 IP 보호 기술 스택

- **Watermarking**: 모델 가중치·출력에 식별 신호를 삽입(Stanford KGW, Google SynthID-Text)
- **Fingerprinting**: 특정 trigger 입력 시 unique 응답 → 모델 식별
- **API Hardening**: 토큰별 Rate Limit, 분당 쿼리 제한, 비정상 패턴 탐지
- **Output Perturbation**: logits에 noise 추가, top-k만 노출, log-probabilities 차단
- **Differential Privacy**: 학습·응답에 DP 적용, ε-budget 관리
- **Confidential Computing**: TEE(Intel TDX, AMD SEV, NVIDIA H100 Confidential)에서 추론
- **법적 보호**: 영업비밀·AI 기본법·저작권·약관(ToS) 명시

## 3. 기술사 출제 포인트

- **개념 정의와 필요성**: 모델 증류 공격의 정의, 합법 KD와의 구분, AI 모델을 핵심 자산(IP)으로 보호해야 하는 동기
- **위협 모델링**: 공격자 능력(Black/Gray/White-box), 공격 표면(API, 가중치, 학습데이터), STRIDE 매핑
- **OWASP LLM Top 10**: 10가지 위협 분류와 LLM10(Model Theft) 시나리오
- **기존 보안과의 차이**: 전통 SW 취약점(메모리·인증) vs AI 모델 보안(데이터·가중치·추론), 전통 IP 보호와의 차별점
- **방어 아키텍처**: API 게이트웨이(Rate Limit, WAF) → 추론 서버(워터마킹·DP) → 모델 저장소(TEE·암호화) → 모니터링(MLSecOps)
- **표준·프레임워크**: NIST AI RMF, ISO/IEC 27090(AI 보안), MITRE ATLAS, OWASP LLM Top 10, KISA AI 보안 가이드
- **거버넌스·법제**: AI 기본법, 영업비밀보호법, 약관(API ToS), 국가간 수출통제(미국 AI 확산규칙)
- **도입 시 고려사항**: 워터마킹의 강건성·은닉성·용량의 트레이드오프, DP의 모델 효용 저하, TEE의 성능·비용

### 답안 작성 시 핵심 키워드

- **Surrogate Model**: 공격자가 만든 모방 모델, 능력 평가 방법(Benchmark 격차)
- **Knowledge Distillation 손실**: KL Divergence, Temperature Scaling
- **MLSecOps**: DevSecOps의 AI 확장, 모델 SBOM(MLBOM)·서명·검증 파이프라인
- **AI 공급망**: 사전학습 모델(Hugging Face)·데이터셋·라이브러리 신뢰 체인
- **MITRE ATLAS**: AI 시스템 위협 전술·기법 매트릭스
- **워터마킹 평가축**: Robustness, Imperceptibility, Capacity, Verifiability
- **국제 동향**: 美 AI 확산 규제(AI Diffusion Rule), EU AI Act 고위험 시스템

## 4. 관련 토픽 연계

- **AI 보안 표준화**: AI 보안 표준화·에이전틱 AI 위협 대응 ↔ Model Theft가 핵심 위협 카테고리
- **LLM 레드팀**: AI 모델 보안과 LLM 레드팀 자동화 ↔ 증류 공격 시뮬레이션을 레드팀이 수행
- **AI 거버넌스**: 글로벌 AI 기술패권 경쟁 ↔ 모델 IP 보호가 국가 AI 주권의 핵심
- **AI 워터마크**: AI 디지털 워터마크와 딥페이크 인증 ↔ 콘텐츠·모델 워터마킹의 공통 기반
- **데이터 IP**: 지식재산처 AI데이터 IP 통합플랫폼 ↔ 모델·데이터·콘텐츠 IP의 통합 보호
- **양자내성암호**: HNDL 공격·암호 민첩성 ↔ AI 모델 가중치 보관 시 PQC 적용

**결론**: 딥시크 사건은 단순 외교 이슈가 아니라 **API만 열려 있어도 모델 IP가 유출될 수 있다는 AI 보안의 구조적 위협**을 드러냈다. 기술사 수험자는 모델 증류 공격을 OWASP LLM Top 10·MITRE ATLAS·NIST AI RMF의 표준 분류로 정리하고, 워터마킹·DP·TEE·MLSecOps를 결합한 다층 방어 아키텍처와 AI 거버넌스·법제를 함께 답안에 녹여야 한다.
