## 1. 뉴스 요약

과학기술정보통신부가 2026년 4월 '에이전틱 AI 자체 개발 TF(일명 AI 어벤저스)'를 공식 발족했다. 외부 벤더에 위탁하지 않고 과기정통부 소속 젊은 공무원이 직접 에이전틱 AI 시스템을 설계·구현·배포하는 공공부문 최초의 시민개발자(Citizen Developer) 모델이다. 계기는 세종시 7년차 공무원의 'AI 자가개발' 사례가 내부에서 확산되며 업무 자동화 효과를 입증한 점이다. 지디넷코리아·경기일보·v.daum.net·KB Think·네이트 등 5개 이상 매체가 동시 보도했으며, TF는 4월부터 행정 에이전트(보고서 초안, 법령 검색, 민원 분류)를 실무에 투입한다. 기존 공공 SI 방식(요구사항→RFP→계약→외주→검수, 평균 12~18개월)과 달리, 사용자(공무원)가 직접 개발 루프에 참여하는 애자일·저(低)위탁 모델로 공공 SW 개발 패러다임의 전환을 시사한다.

## 2. 핵심 개념

### 시민개발자 기반 에이전틱 AI SDLC

```
[전통 공공 SI 모델]                [시민개발자 에이전틱 AI 모델]
요구사항 분석                       업무 문제 발견(공무원)
   ↓                                   ↓
ISP/제안서(RFP)                     프롬프트·에이전트 설계(본인)
   ↓                                   ↓
입찰·계약(수개월)                   LLM API + 로우코드 툴 조립
   ↓                                   ↓
외주 개발(6~12개월)                  1~2주 PoC → 실무 투입
   ↓                                   ↓
검수·이관                           피드백 반복(Human-in-Loop)
   ↓                                   ↓
유지보수 계약                       사내 에이전트 라이브러리화
```

### 에이전틱 AI(Agentic AI) 4대 요소

- **목표 지향(Goal-oriented)**: 사용자가 최종 목표만 지시하면 AI가 하위 작업으로 분해
- **계획·추론(Plan & Reason)**: ReAct, Plan-and-Execute 등 다단계 추론으로 실행 순서 결정
- **도구 사용(Tool Use)**: 검색·DB 조회·API 호출·코드 실행 등 외부 도구를 능동적으로 활용
- **기억·학습(Memory)**: 단기 컨텍스트 + 장기 벡터 메모리로 업무 맥락을 유지

### 공공 시민개발자 모델의 리스크와 통제

- **보안 리스크**: 개인정보·국가기밀 데이터의 LLM 노출 → 폐쇄형 LLM, PII 마스킹, VPC Endpoint 필수
- **품질 리스크**: 테스트 커버리지·예외 처리 부족 → 공용 테스트 하네스, 코드리뷰, 사내 AI 컴플라이언스 관문
- **섀도우 IT 리스크**: 중복·비표준 에이전트 난립 → 에이전트 레지스트리·표준 프레임워크(관 플랫폼)
- **책임 리스크**: 자동 결정의 책임소재 불명확 → HITL 의무화, 감사로그, 설명가능성(XAI)

## 3. 기술사 출제 포인트

### 논술형 예상 주제

- 공공부문 시민개발자 확산 시 SDLC·품질관리·보안 통제 전략
- 전통 공공 SI와 시민개발자 모델의 비교, 대가산정·감리 체계의 변화
- 에이전틱 AI의 ReAct·Plan-and-Execute 패턴과 신뢰성 검증 방법

### 단답형 핵심 키워드

- **시민개발자(Citizen Developer)**: 현업 사용자가 로우코드·AI 도구로 직접 애플리케이션을 개발하는 개발자
- **에이전틱 AI**: 목표·계획·도구·기억을 갖고 자율적으로 작업을 수행하는 LLM 기반 에이전트
- **ReAct**: Reason-Act 반복 패턴으로 추론과 행동을 교차 수행하는 에이전트 설계 기법
- **HITL(Human-in-the-Loop)**: AI 결정에 사람이 개입해 검증·교정하는 절차
- **로우코드/노코드**: 시각적 빌더와 사전 정의 컴포넌트로 코드를 최소화한 개발 플랫폼

### 비교·대조 포인트

아래는 전통 공공 SI와 시민개발자 모델의 차이점 비교다.

| 구분 | 전통 공공 SI | 시민개발자 에이전틱 AI |
|------|-------------|---------------------|
| 주체 | 외주 SI 업체 | 현업 공무원 |
| 기간 | 12~18개월 | 1~4주 |
| 방식 | 폭포수(RFP→검수) | 애자일·HITL 반복 |
| 비용 | 대가산정 기준 고정 | LLM 토큰·라이선스 |
| 리스크 | 요구사항 표류 | 섀도우IT·보안·품질 |
| 품질관리 | 감리·검수 | 사내 컴플라이언스 게이트 |
