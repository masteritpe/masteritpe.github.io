## 1. 뉴스 요약

UNIST 정임두 교수팀이 눈짓만으로 로봇을 제어할 수 있는 AI 스마트 콘택트렌즈를 개발했다. 렌즈에 내장된 센서가 안구 움직임을 감지하고 AI가 실시간으로 의도를 해석해 로봇 팔 등 외부 장치를 조종하는 방식이다. 동아사이언스, 인공지능신문, mstoday 등 다수 매체가 보도했으며, 초소형 웨어러블 디바이스에서 AI 추론을 수행하는 시스템 아키텍처 설계의 새로운 가능성을 제시한 연구로 평가받고 있다.

## 2. 핵심 개념

### 스마트 콘택트렌즈 시스템 아키텍처

```
┌──────────────────────────────────────────────────────────────┐
│           AI 스마트 콘택트렌즈 시스템 계층 구조                │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  [센서 계층]  안구 운동 센서 → EOG 신호 수집                   │
│       │       가속도/자이로 센서 → 머리 움직임 보정             │
│       ▼                                                      │
│  [엣지 추론 계층]  초경량 AI 모델 → 의도 분류(눈짓 패턴)       │
│       │           TinyML / 양자화 모델 → 수ms 내 추론          │
│       ▼                                                      │
│  [통신 계층]  BLE 5.x / UWB → 저전력 무선 전송                │
│       │       수신 게이트웨이(스마트폰/허브)                    │
│       ▼                                                      │
│  [제어 계층]  로봇 제어 명령 생성 → 액추에이터 구동             │
│              피드백 루프 → 제어 정확도 보정                     │
└──────────────────────────────────────────────────────────────┘
```

### 기존 Edge AI vs 웨어러블 바이오센서 AI 비교

| 구분 | 기존 Edge AI (카메라/차량) | 웨어러블 바이오센서 AI |
|------|--------------------------|----------------------|
| 폼팩터 | 수cm~수십cm | 수mm (콘택트렌즈) |
| 전력 예산 | 수W~수십W | 수μW~수mW |
| 센서 유형 | 카메라, LiDAR | EOG, IMU, 바이오센서 |
| AI 모델 크기 | 수MB~수GB | 수KB~수백KB |
| 추론 방식 | NPU/GPU | MCU 기반 TinyML |
| 지연 허용 | 수십ms | 수ms (실시간 필수) |
| 통신 방식 | Wi-Fi/5G | BLE/NFC |
| 데이터 민감도 | 중간 | 매우 높음 (생체정보) |

### TinyML과 초저전력 추론 아키텍처

```
┌─────────────────────────────────────────────────────┐
│              TinyML 추론 파이프라인                    │
├─────────────────────────────────────────────────────┤
│                                                     │
│  원본 모델 (수MB)                                    │
│       │                                             │
│       ▼  양자화 (INT8/INT4)                          │
│  경량 모델 (수십~수백KB)                              │
│       │                                             │
│       ▼  프루닝 (Pruning)                            │
│  초경량 모델 (수KB)                                   │
│       │                                             │
│       ▼  MCU 배포 (Cortex-M/RISC-V)                 │
│  실시간 추론 (수ms, μW 단위 전력)                     │
│                                                     │
│  ※ 핵심 제약: 메모리 ≤256KB, 연산 ≤수십MOPS          │
└─────────────────────────────────────────────────────┘
```

## 3. 기술사 출제 포인트

### 예상 문제
1. **웨어러블 AI 디바이스의 시스템 아키텍처 설계 시 고려사항을 설명하시오**
   - 초저전력 설계: 에너지 하베스팅, 듀티 사이클링, 이벤트 기반 웨이크업
   - TinyML 모델 최적화: 양자화, 프루닝, 지식 증류(Knowledge Distillation)
   - 실시간 제약: 하드 리얼타임 추론, 결정론적 응답 보장
   - 생체정보 보안: 온디바이스 처리, 전송 시 암호화, 차등 프라이버시

2. **TinyML과 클라우드 AI의 아키텍처 차이를 비교하고, TinyML 적용 전략을 설명하시오**
   - MCU 기반 추론 vs GPU 기반 추론의 제약 조건 비교
   - 모델 경량화 기법: 양자화(Quantization), 프루닝(Pruning), NAS(Neural Architecture Search)
   - Split Computing: 엣지-클라우드 분할 추론 전략
   - 적용 사례: 바이오센서, 산업 IoT, 스마트홈

3. **BCI(Brain-Computer Interface)와 AI 융합 시스템의 아키텍처를 설계하시오**
   - 신호 수집(EOG/EEG) → 전처리 → 특징 추출 → AI 분류 → 제어 명령 파이프라인
   - 실시간 의도 인식 알고리즘 설계
   - 안전성: Fail-safe 메커니즘, 오분류 시 안전 제어

### 답안 작성 키워드
`TinyML`, `EOG(안전도)`, `초저전력 추론`, `BLE 5.x`, `양자화(Quantization)`, `프루닝(Pruning)`, `MCU 기반 AI`, `이벤트 기반 처리`, `에너지 하베스팅`, `Split Computing`, `생체정보 프라이버시`

## 4. 관련 토픽 연계

| 연계 토픽 | 연계 포인트 |
|-----------|------------|
| Edge AI 온디바이스 추론 | TinyML은 Edge AI의 극단적 경량화 형태 |
| IoT 시스템 아키텍처 | 센서-게이트웨이-클라우드 3계층 구조 공유 |
| 실시간 시스템(RTOS) | 웨어러블 AI의 하드 리얼타임 제약 조건 |
| 개인정보보호(PIPA) | 생체정보의 민감정보 분류와 온디바이스 처리 |
| 피지컬 AI | AI가 물리 세계와 직접 상호작용하는 패러다임 |
| 디지털 헬스케어 | 의료용 웨어러블 AI 디바이스 규제(SaMD) |
