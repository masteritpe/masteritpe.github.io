## 1. 뉴스 요약

2026-05-25~26 KBS·동아사이언스·조선일보·중앙일보·데일리안·정보통신신문·인공지능신문·사이언스타임즈 등 10개 이상 매체가 "**KAIST·포스텍·소니AI, 영상 속 물체의 질량·속도 추론해 장면 맞춤형 효과음 생성 AI 'PAVAS' 개발**"을 일제히 보도했다. 핵심은 ① 단순 영상-소리 대응이 아니라 **물체의 질량·속도 등 물리량을 영상에서 추론**해 ② **장면에 물리적으로 그럴듯한 사운드 이펙트를 자동 생성**하는 **Video-to-Audio(V2A)** 멀티모달 생성 모델이며 ③ 영화·게임 **폴리(Foley) 사운드 제작 파이프라인**에 직접 적용 가능하다는 점이다. 구글 'V2A' 등 글로벌 V2A 모델이 시각-의미 단순 대응에 그친 반면, PAVAS는 **물리정보 기반 멀티모달 생성**으로 한 단계 앞섰다. 기술사 AI데이터분석 영역에서 **멀티모달 LLM·생성형 AI·비디오 이해 모델**은 출제 비중이 빠르게 늘어나는 핵심 토픽이며, 본 회차(139회) 응용 사례·아키텍처 출제 가능성이 높다.

## 2. 핵심 개념

### PAVAS 개요

| 항목 | 내용 |
|---|---|
| 명칭 | PAVAS (Physics-Aware Video-to-Audio Synthesis) |
| 주관 | KAIST 전산학부, 포스텍 컴퓨터공학과, 소니 AI |
| 발표 시점 | 2026-05 학술 공개 (SIGGRAPH 계열 추정) |
| 입력 | 무음 비디오 클립 |
| 출력 | 장면에 동기화된 효과음(폴리 사운드) |
| 차별점 | 영상에서 물체 질량·속도·재질 추론 → 물리적으로 일관된 사운드 생성 |
| 응용 | 영화·드라마·게임 폴리 자동화, 영상 콘텐츠 후반작업, AR/VR 몰입 음향 |

### V2A 멀티모달 생성 파이프라인 4단계

| 단계 | 구성요소 | 주요 처리 |
|---|---|---|
| 비전 이해 | 비디오 인코더(ViT/Video Transformer), 광학흐름, 깊이 추정 | 객체 검출·추적, 충돌·접촉 이벤트 추출, 운동학적 특성 추정 |
| 물리 추론 | 물리 파라미터 추정기, 시뮬레이션 사전(prior) | 질량·속도·재질·표면 마찰 추정, 물리적 사운드 캐릭터 분류 |
| 오디오 생성 | 디퓨전(Diffusion) 또는 Flow Matching 기반 오디오 디코더 | 시간 동기 잠재(Latent)에서 멜스펙트로그램·웨이브폼 생성 |
| 동기화·정합 | 오디오-비디오 동기 검증, 시간 정렬 손실 | 시점 정렬, 메트릭 평가(FAD·CLAP·KL·동기율) |

### 글로벌 V2A 모델 비교

| 구분 | Google V2A | Meta MovieGen Audio | KAIST PAVAS |
|---|---|---|---|
| 공개 시기 | 2024~2025 | 2025 | 2026 |
| 핵심 접근 | 시각-텍스트-오디오 잠재 정렬 | 텍스트·비디오 조건 통합 생성 | 물리 정보 기반 멀티모달 생성 |
| 물리 정합 | 약함(의미 대응 중심) | 보통 | 강함(질량·속도·재질) |
| 폴리 적용성 | 제한적 | 보통 | 높음 |
| 평가 지표 | FAD/CLAP | FAD/CLAP/MOS | FAD/CLAP + 물리 정합 지표 |

## 3. 기술사 출제 포인트

### 개념 정의와 필요성

- **V2A(Video-to-Audio)**: 무음 영상으로부터 장면 의미·시간 정합 사운드(효과음·앰비언스)를 자동 생성하는 멀티모달 생성형 AI
- **물리정보 V2A**: 영상에서 추론한 물리량(질량·속도·재질)을 조건으로 사용해 물리적으로 일관된 사운드를 생성하는 차세대 V2A
- **필요성**: ① 영상 콘텐츠 후반작업 비용 절감(폴리 작업 자동화), ② 게임·메타버스·AR/VR의 인터랙티브 사운드 수요, ③ 시각장애인용 사운드 디스크립션, ④ 데이터-부족 도메인의 합성 데이터 생성

### 기존 방식과의 비교

- **수동 폴리 작업**: 폴리 아티스트가 도구·재료로 직접 녹음·합성. 품질 높으나 비용·시간 큼
- **라이브러리 매칭**: 효과음 DB에서 영상 분류로 검색·정합. 자동이나 다양성·물리 정합 부족
- **단순 V2A 모델**: 비전 임베딩-오디오 디퓨전 직결. 의미는 맞아도 충격력·질량 등 물리 정합 약함
- **물리정보 V2A(PAVAS)**: 물리 파라미터 추정→조건부 생성. 물리·시간 정합 모두 강화

### 아키텍처 구성요소

- **비전 백본**: ViT/Video Swin/VideoMAE 등 시공간 인코더, 광학흐름·깊이 추정 보조 신호
- **물리 추정기**: 객체 마스크·궤적에서 질량·속도·접촉 강도 회귀(Regression), 재질 분류
- **잠재 정렬 모듈**: 비전·물리 임베딩을 오디오 잠재공간으로 매핑(Cross-attention)
- **오디오 디코더**: Latent Diffusion 또는 Flow Matching 기반 멜스펙트로그램 생성, Neural Vocoder 웨이브폼 합성
- **시간 정합 모듈**: 비디오 프레임-오디오 윈도우 정렬 손실(Sync Loss)
- **평가·검증**: FAD(Fréchet Audio Distance), CLAP 유사도, MOS(주관 평가), 물리 정합 지표
- **데이터 파이프라인**: VGGSound·AudioSet·자체 라벨링 폴리 코퍼스, 합성 데이터 증강

### 도입 시 고려사항

- **저작권·라이선스**: 학습 데이터(영상·오디오)의 저작권, 폴리 아티스트 일자리·계약 영향
- **품질 평가**: 자동 지표(FAD/CLAP)와 주관 평가(MOS)의 병행, 도메인별 평가 셋 구축
- **추론 비용**: 디퓨전 다단계 샘플링의 GPU 비용, 실시간/배치 처리 모드 분리
- **편향·안전**: 폭력적·공격적 사운드 생성 제어, 워터마킹으로 AI 생성 표식
- **저작권 확장**: AI 생성 사운드의 권리 귀속, AI 워터마크·콘텐츠 출처 표시 의무화 흐름
- **운영(MLOps)**: 모델 카드·데이터시트, 드리프트 모니터링, 점진 학습(continual learning)
