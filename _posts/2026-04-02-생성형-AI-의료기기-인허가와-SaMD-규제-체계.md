## 1. 뉴스 요약

식약처가 국내 최초로 생성형 AI 기반 디지털의료기기를 허가하며 AI 의료기기 규제의 새 장을 열었다. 동시에 복지부는 만성질환자 대상 의료서비스 전주기 AI 전환(AX) 스프린트 사업을 추진하고, AI 의료기기 상용화 지원에 450억 원을 투입한다. 기존 결정론적(Deterministic) AI와 달리 생성형 AI는 출력의 비결정성·환각(Hallucination) 위험이 존재하여, SaMD(Software as a Medical Device) 인허가 체계의 근본적 패러다임 전환이 요구되고 있다.

### AI 의료기기 유형별 규제 비교

| 구분 | 규칙 기반 SaMD | ML 기반 SaMD | 생성형 AI SaMD |
|------|---------------|-------------|---------------|
| 출력 특성 | 결정론적(동일 입력→동일 출력) | 확률적(학습 모델 기반) | 비결정적(매번 다른 출력 가능) |
| 변경 관리 | 버전별 인허가 | 사전결정변경계획(PCCP) | 지속적 모니터링 + PCCP 확장 |
| 검증 방법 | 기능 테스트 | 임상 성능 검증 | 임상검증 + 환각 탐지 + 레드팀 평가 |
| 위험 등급 | Class I~III | Class II~III | Class II~III(강화) |
| 투명성 | 로직 공개 | 모델 성능 지표 공개 | 모델카드 + 데이터시트 + 한계 명시 |
| 사후 감시 | 정기 점검 | 실사용 성능 모니터링 | 실시간 드리프트 감시·자동 알림 |

### 생성형 AI 의료기기 인허가 프레임워크

```
┌─ 사전 단계 ──────────────────────────────────┐
│  ① 의도된 용도(Intended Use) 정의             │
│  ② 위험 등급 분류 (IMDRF 프레임워크)          │
│  ③ GMLP(Good Machine Learning Practice) 적용 │
└──────────────────────────────────────────────┘
          ↓
┌─ 개발 단계 ──────────────────────────────────┐
│  ④ 학습 데이터 품질 관리 (대표성·편향 검증)   │
│  ⑤ 모델 개발 및 내부 검증 (V&V)              │
│  ⑥ 환각(Hallucination) 방지 메커니즘 구현     │
│     - RAG 기반 근거 제시                      │
│     - 불확실성 정량화(Calibration)             │
│     - 안전 가드레일(Safety Guardrails)         │
└──────────────────────────────────────────────┘
          ↓
┌─ 인허가 단계 ────────────────────────────────┐
│  ⑦ 임상 성능 평가 (민감도·특이도·AUC)         │
│  ⑧ 사이버보안 위험 평가 (SBOM 제출)           │
│  ⑨ 인적 요인(Human Factors) 평가              │
│  ⑩ PCCP(사전결정변경계획) 수립                 │
└──────────────────────────────────────────────┘
          ↓
┌─ 시판 후 단계 ───────────────────────────────┐
│  ⑪ 실사용 성능 모니터링 (RWD/RWE 수집)        │
│  ⑫ 모델 드리프트 감시 및 재학습 관리           │
│  ⑬ 이상사례 보고 및 리콜 체계                  │
└──────────────────────────────────────────────┘
```

### GMLP(Good Machine Learning Practice) 10대 원칙

| 번호 | 원칙 | 핵심 내용 |
|------|------|----------|
| 1 | 멀티 분야 전문성 | 임상·데이터·규제 전문가 협업 |
| 2 | SW 엔지니어링 실천 | 재현성, 버전관리, CI/CD 적용 |
| 3 | 대표성 있는 데이터 | 임상 다양성, 편향 최소화 |
| 4 | 독립적 데이터셋 | 훈련·검증·테스트 분리 |
| 5 | 참조 표준 확보 | 임상적 골드 스탠다드 정의 |
| 6 | 모델 설계 적합성 | 의도된 용도에 맞는 모델 선택 |
| 7 | 인적 요인 고려 | 사용자 인터페이스·워크플로우 최적화 |
| 8 | 임상 성능 테스트 | 통계적 유의성 있는 평가 |
| 9 | 보안·복원력 확보 | 적대적 공격 대응, 입력 검증 |
| 10 | 모니터링·재평가 | 배포 후 지속적 성능 추적 |

## 3. 기술사 출제 포인트

### 예상 문제
- **SaMD(Software as a Medical Device)의 개념과 분류 체계를 설명하고, AI/ML 기반 SaMD의 인허가 특수성을 논하시오**
- **생성형 AI 의료기기의 안전성 확보를 위한 GMLP 적용 방안과 환각(Hallucination) 방지 전략을 제시하시오**
- **디지털 헬스케어 분야에서 AI 전환(AX)의 추진 전략과 규제 샌드박스 활용 방안을 논하시오**

### 답안 작성 키워드
- SaMD: 하드웨어 독립적 의료 소프트웨어, IMDRF 위험 분류
- PCCP(Predetermined Change Control Plan): AI/ML 모델 사전 변경 관리
- GMLP: FDA·Health Canada 등 국제 10대 원칙
- TGA(Total Product Lifecycle): 전제품수명주기 관리
- 환각 방지: RAG, 불확실성 정량화, 가드레일

## 4. 관련 토픽 연계

| 연계 토픽 | 연관성 |
|----------|--------|
| AI 거버넌스·AI 기본법 | SaMD 규제의 상위 법적 프레임워크 |
| 디지털 트윈 | 의료 디지털 트윈 기반 시뮬레이션·임상시험 |
| AI 워터마크·딥페이크 탐지 | 생성형 AI 출력물 신뢰성 검증 기술 |
| 마이데이터(MyData) | 의료 데이터 활용·개인정보 보호 균형 |
| XAI(설명 가능한 AI) | 의료 AI 판단 근거 투명성 확보 |
| 소프트웨어 품질(ISO 25010) | SaMD 품질 특성 평가 기준 |
| 사이버보안(SBOM) | 의료기기 소프트웨어 공급망 보안 |
