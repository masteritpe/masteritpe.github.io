## 1. 뉴스 요약

한국전자기술연구원(KETI)이 행정안전부와 협력해 연 1,000만 건 규모의 '안전신문고' 민원 접수·처리 프로세스에 AI 기술을 적용한다. 시민이 신고한 사진·영상·텍스트 민원을 AI가 자동 분류하고 담당 부처·지자체로 라우팅하며, 중복·악성 신고 필터링도 수행한다. 올해 연말 시범 서비스를 개시하고, 이후 단계적으로 확대할 예정이다. 서울STV뉴스·조선비즈·네이트 등 다수 매체가 동시 보도했으며, 공공민원 처리 병목 해소와 공무원 업무 부담 완화를 목표로 한다. 연 1천만 건은 초당 약 0.3건, 피크 시간 수 건/초 수준의 처리율이 요구되며, 다양한 입력 형태(이미지·영상·텍스트·GPS)와 높은 가용성이 요구되는 전형적인 대규모 공공 AI 시스템이다.

## 2. 핵심 개념

### 대규모 공공민원 AI 시스템 아키텍처

```
┌──────────────────────────────────────────────────────────────┐
│            안전신문고 AI 처리 플랫폼 (멀티모달 분류·라우팅)       │
├──────────────────────────────────────────────────────────────┤
│ [접수 계층]                                                   │
│  모바일 앱 / Web / API → API Gateway → 인증(PASS, KAKAO)       │
│                          ↓                                     │
│  메시지 큐 (Kafka/SQS) - 초당 피크 흡수, 백프레셔 제어           │
├──────────────────────────────────────────────────────────────┤
│ [AI 추론 계층] (GPU/CPU 하이브리드)                            │
│  ┌──────────────┬──────────────┬──────────────┐             │
│  │ Vision 모델   │ NLP 분류기    │ 중복·스팸 탐지 │             │
│  │ (YOLO/ViT)   │ (KoBERT)     │ (임베딩+ANN)   │             │
│  └──────────────┴──────────────┴──────────────┘             │
│  → 신고 유형 분류(교통/시설/환경 등 40+ 카테고리)               │

├──────────────────────────────────────────────────────────────┤
│ [라우팅·오케스트레이션]                                         │
│  규칙 엔진 + GIS 매칭 → 지자체·부처별 담당자 자동 배정          │
│  SLA 모니터링 / 재배정 / 에스컬레이션                           │
├──────────────────────────────────────────────────────────────┤
│ [데이터·관측]                                                 │
│  벡터DB(유사 신고) · OLAP(통계) · MLOps(재학습 파이프라인)      │
│  감사로그 / 개인정보 마스킹 / 편향 모니터링                      │
└──────────────────────────────────────────────────────────────┘
```

### 대규모 공공 AI 시스템 4대 설계 원칙

- **확장성(Scalability)**: 수평 확장 가능한 무상태 추론 서버 + 모델 샤딩, 피크 시간대 오토스케일링
- **가용성(Availability)**: 99.9% 이상 SLA, 다중 AZ 배포, 모델 서빙 헬스체크 및 롤링 업데이트
- **신뢰성(Trustworthiness)**: 저신뢰 예측은 사람(휴먼 인 더 루프)에게 위임, 설명가능성(XAI) 제공
- **공정성(Fairness)**: 지역·언어·입력 품질 편향 모니터링, 취약계층(노인·장애인) 접근성 보정

### 처리량 설계 기준 (연 1천만 건)

- 일평균 약 27,400건, 피크 배수 가정 5x → 초당 약 1.6건 지속, 단기 피크 10+ RPS
- 멀티모달 추론 평균 레이턴시 목표: 텍스트 200ms, 이미지 500ms 이내
- 스토리지: 원본 미디어 년 수십 TB, 핫/웜/콜드 계층 분리(객체스토리지)

## 3. 기술사 출제 포인트

### 논술형 예상 주제

- 대규모 공공민원 AI 시스템 설계 시 고려사항(확장성·신뢰성·공정성) 및 아키텍처 제시
- 휴먼 인 더 루프(HITL) 설계: 자동화 수준(L1~L5)별 역할 분담 전략
- 공공 AI 서비스의 편향·오분류 대응 거버넌스 체계

### 단답형 핵심 키워드

- **HITL(Human-in-the-Loop)**: AI 결정의 최종 검증·보정을 사람이 수행하는 반복 루프
- **멀티모달 분류**: 텍스트·이미지·영상·위치 정보를 통합해 분류하는 기법
- **모델 서빙(Model Serving)**: 학습된 모델을 실시간 API로 제공하는 인프라(Triton, TorchServe)
- **피처 스토어**: 학습·추론에 공통 사용되는 피처를 중앙 관리하는 저장소
- **MLOps**: 모델 개발·배포·모니터링·재학습을 자동화하는 운영 체계

### 비교·대조 포인트

아래는 규칙 기반 분류와 AI 분류의 차이점 비교다.

| 구분 | 규칙 기반 분류 | AI 분류 |
|------|---------------|---------|
| 적응성 | 규칙 추가 필요 | 데이터로 학습 |
| 커버리지 | 정의된 경우만 | 신규 유형 일반화 가능 |
| 설명성 | 높음(규칙 추적) | 상대적 낮음(XAI 필요) |
| 운영비용 | 규칙 유지보수 | MLOps·GPU 비용 |
| 오류 유형 | 누락 중심 | 오분류·편향 중심 |
