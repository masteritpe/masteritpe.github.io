**[핵심/키워드]**

&lt;기술요소&gt; 
- 센서이용 Agent 활용기술: 현실 세계의 데이터를 수집, 판단 수행 지능형 에이전트 기술
- IoT컨트롤러 통신: 사물인터넷(IoT) 기기와 제어기 간의 실시간 데이터 송수신 및 연결망
- 시뮬레이션: 가상 모델을 통해 물리적 현상이나 프로세스의 결과를 예측하고 검증하는 기술
- 3D 모델링: 실물의 형태와 구조를 입체적인 디지털 데이터로 시각화하는 기술
- 클라우드: 방대한 양의 트윈 데이터를 저장, 처리, 공유하기 위한 가상화 서버 인프라
- 인공지능: 수집된 데이터를 분석하여 이상 징후를 탐지, 최적의 의사결정을 지원
- Actuator 연결제어: 가상의 분석결과로 실제 기기를 움직이게 하는 구동장치 기술
- 디지털 스레드: 제품의 설계부터 운영, 폐기까지 전 생애주기 데이터를 연결하는  데이터 흐름
&lt;디지털 트윈 기술발전 5단계&gt; : 모관모연자 
- 1단계: 모사(Mirroring) 
- 2단계: 관제(Monitoring) 
- 3단계: 모의(Modeling&amp;시뮬레이션) 
- 4단계: 연합(Federated) 
- 5단계: 자율(Autonomous)
&lt;활용분야&gt;
- 제조, 에너지, 도시, 물류

### [신기술] CXL(Compute Express Link)

**[정의]**

- 중앙처리장치(CPU)와 메모리, 그래픽 처리장치(GPU), 저장장치 등을 보다 효율적으로 활용할 수 있게 해 주는 인터페이스
 - 인터페이스를 하나로 통합해 각 장치를 직접 연결하고 메모리를 공유, 거대한 공용 메모리 풀(pool)을 형성

**[핵심/키워드]**

&lt;필요성&gt;
- 메모리 장벽(Memory Wall) 해결
- 리소스 풀링(Resource Pooling)
- 이기종 컴퓨팅 지원: CPU, GPU, FPGA 등 다양한 가속기 간의 효율적인 데이터 연동
- 고속 데이터 처리: AI, HPC(고성능 컴퓨팅), 빅데이터 분석위한 저지연/고대역폭 인터페이스 제공
&lt;프로토콜&gt;
- CXL.io (IO연결)
- CXL.mem (메모리 확장)
- CXL.cache (캐시 일관성)
&lt;디바이스 타입&gt;
- Type1 (CXL.io + CXL.cache)
- Type2 (CXL.io + CXL.mem + CXL.cache)
- Type3 (CXL.io + CXL.mem)
&lt;버전별 진화&gt;
- CXL 1.0 / 1.1 (내부) : PCIe 5.0 기반, 단일 호스트와 장치 간의 직접 연결 (Point-to-Point)
- CXL 2.0 (외부, 공유) : 스위칭 도입, 메모리 풀링(Pooling) 지원
- CXL 3.0 / 3.1 (공유, Fabric) : PCIe 6.0 기반(대역폭 2배), CXL Fabric 도입
- CXL 4.0 (RAS, PCIe 6.0/7.0, Bundled Port) : PCIe 7.0 대응 준비

### [신기술] HBM(High Bandwidth Memory, 고대역폭 메모리)

**[정의]**

- 고성능 데이터 처리를 위해 DRAM 적층화 및 TSV 배선 기반 데이터 이동 병목 현상을 해소한 DRAM 적층형 고성능 메모리

**[핵심/키워드]**

&lt;적용기법&gt;
- TSV (Through Silicon Via): 메모리 칩을 수직으로 쌓고 칩 사이에 얇은 금속 터널을 만들어 데이터 전송 속도를 높이는 기법
- CoW (Chip on Wafer): 웨이퍼 위에 칩을 붙이는 기법
- TCB (Thermal Compression Bonding): TSV가 적용된 얇은 칩을 정밀하게 쌓아 올리는 패키징 기법
&lt;구성요소&gt;
- 메모리 적층화: TSV, DRAM die, Base(Logic) die, Micro Bump
- 칩셋 구성: SoC, PHY, Silicon Interposer, Package Substrate
&lt;활용분야&gt;
- 고성능 그래픽: 고사양 게임, 가상현실 (VR), 증강현실(AR)등
- 인공지능: 딥 러닝, 머신 러닝 등
- 고성능 컴퓨팅: 데이터 분석, 과학 계산 등

**[핵심/키워드]**

&lt;기술요소&gt;
- LPWIO(Low Power Wide I/O) : 저전력으로 넓은 대역폭을 구현하는 입출력 기술
- TSV 미사용 : 실리콘 관통 전극(TSV) 공정 없이 칩을 직접 연결
- VFO(Vertical Fusion Optimization) : 수직 방향의 접합 및 융합을 최적화
- 적층구조 : 기존 방식보다 더 얇고 정밀하게 칩을 쌓아 올리는 구조

### [신기술] OT(Operational Technology) 사이버 보안 6원칙

**[정의]**

- OT 사이버보안 6원칙 : 물,에너지,교통 인프라와 같은 사회 필수서비스의 운영 기술에 위협이 되는 사이버활동에 대해 국가적 시스템을 안전하게 보호하기 위한 원칙

&lt;6원칙&gt; 
1\. 안전이 최우선이다. 
2\. 비즈니스에 대한 지식이 중요하다. 
3\. OT 데이터는 매우 중요하며 반드시 보호해야 한다.

5\. 공급망이 안전해야 한다. 
6\. 사람이 OT 보안에 필수적이다.

### [신기술] ABAC(Attribute-Based Access Control)

**[정의]**

- 사용자, 자원, 환경 등의 속성(attribute)을 기반으로 접근 허용 여부를 동적으로 판단하는 접근통제 모델

**[핵심/키워드]**

&lt;특징&gt;
- 정적/역할 기반 RBAC보다 더 세밀하고 유연한 통제 가능, 정책 중심, 규칙기반 통제
- 다양한 테스트 기법 : SAST/ DAST/ IAST 등 복합 활용

### [신기술] Ansoff Matrix(안소프 매트릭스)

**[정의]**

- 기업이 성장을 달성하기 위해 제품(Products)과 시장(Markets)이라는 두 가지 요소를 기준으로 네가지 유형의 성장 전략을 제시한 모형

**[핵심/키워드]**

&lt;특징&gt;
- 성장 전략을 4가지 유형으로 체계화
- 위험 수준을 직관적으로 비교 가능
- 경영 전략·사업 포트폴리오 의사결정에 활용
&lt;4가지 유형의 성장전략&gt;
① 시장 침투(Market Penetration) : 기존 제품 × 기존 시장
  -&gt; 기존 시장에서 현재 점유율을 높이는 전략 
② 제품 개발(Product Development) : 신제품 × 기존 시장
  -&gt; 기존 고객층 대상으로 신제품 개발하거나 기존 제품 개선 
③ 시장 개발(Market Development) : 기존 제품 × 신시장
  -&gt; 기존 제품을 신규 시장에 진출 
④ 다각화(Diversification) : 신제품 × 신시장
  -&gt; 새로운 제품을 신규 시장에 진출(고위험·고수익 전략)
&lt;그래프 구성요소&gt;
- 가로축(제품): 기존 → 신제품으로 갈수록 혁신 리스크 증가
- 세로축(시장): 기존 → 신시장으로 갈수록 시장 리스크 증가
=&gt; 즉, 우상단(다각화)로 갈수록 위험은 커지지만 성장 잠재력도 극대화되는 구조

- 저전력 D램인 LPDDR을 집적해 전력 효율성을 높이고, 기존 온보드 방식과 달리 메모리를 탈부착할 수 있는 메모리 모듈

**[핵심/키워드]**

&lt;특징&gt;
- LPDDR 활용, 범용성, 저전력, 탈부착 형식 모듈

&lt;주요사양&gt;
- 기반 메모리 (LPDDR)
- 속도 (9,600 MT/s)
- I/O (694개)
- 표준화 (JEDEC, 범용적)
- 용량 (192GB 모듈)
- 저전력 (매우 향상된 저전력)
- 공급업체 (삼성, 하이닉스, 마이크론 등 다양)

&lt;SOCAMM 2 개선&gt;
- 속도 향상, 용량 증가, 공급업체 다양화, 범용적 표준화, SPD 지원

### [신기술] CBDC(Central Bank Digital Currency)

**[정의]**

- 실물 현금과 동일한 법정통화의 지위를 가지며, 중앙은행이 디지털 형태로 직접 발행하는 화폐

**[핵심/키워드]**

&lt;특징&gt; 
- 법적지위: 실물 현금과 동일한 법정화폐(Legal Tender) 지위 및 1:1 교환 보장
- 직접부채: 상업은행이 아닌 중앙은행의 부채로 리스크 없는 안전 자산
- 안정성: 가상자산(비트코인 등)과 달리 가치 변동성이 없으며 국가가 가치 보증 

&lt;기술 요소&gt;
- 합의 알고리즘: 분산 노드 간 거래 정당성을 검증하는 기술 (PBFT, PoS 등)
- 스마트 컨트랙트: 프로그래밍을 통해 자금에 용도나 조건을 부여하는 자동 집행 기술
- 영지식 증명(ZKP): 신원 노출 없이 거래 적격성만 증명하는 프라이버시 보호 기술
- 오프라인 결제: 인터넷 단절 시에도 기기 간(NFC, 블루투스) 결제를 지원하는 기술
- 상호운용성: 기존 금융망 및 타 국가 CBDC 시스템과 데이터를 주고받는 연동 기술 

&lt;운영유형&gt; 
- 소매용(Retail): 일반 개인과 기업이 상거래 및 일상 결제에 활용
- 도매용(Wholesale): 금융기관 간의 거액 자금 결제 및 증권 정산에 활용
- 직접형: 중앙은행이 모든 고객 계좌를 직접 관리하고 서비스 제공
- 간접형(2단계): 중앙은행은 발행/검증을 담당하고, 유통은 민간 금융기관이 수행 

&lt;기술방식&gt; 
- 원장 구조: 단일 원장(중앙집중형) vs 분산 원장(DLT, 블록체인 기반)
- 참여 권한: 허가형(승인된 노드만 참여) vs 비허가형(누구나 참여)
- 검증 방식: 계좌 기반(신원 확인 방식) vs 토큰 기반(화폐 진위 확인 방식)
- 인프라 형태: 클라우드 기반 구축 또는 온프레미스 혼합형 아키텍처

### [주기동] 기업 LLM 도입 · 운영을 위한 추론 최적화

**[정의]**

- AI 산업 패러다임이 과거의 고비용 학습 중심 AI에서 현재의 고효율 추론 기반 서비스 AI로 변화되는 현상

**[핵심/키워드]**

&lt;기업 LLM 도입 3대 과제&gt; : 선택 · 비용 · 성능 
1\. 선택: 모델 난립에 따른 도입 의사결정 부담 
2\. 비용: 추론 중심 경제성 관리 
3\. 성능: 실시간 응답성(Latency) 확보 

&lt;핵심 기술&gt; : 추론 성능을 높이는 최신 기술 동향 
4\. 모델 경량화(Model Compression): 초대형 모델의 효율적 운용 
 - 지식 증류(Knowledge Distillation) 
 - 매개변수 공유 및 최적화 
 - 양자화(Quantization)

5\. 추론 병목 해결을 위한 알고리즘 혁신 
 - 디코딩 단계 축소 
 - Attention 최적화 
 - KV(Key-Value Cache) 캐시 효율화 

6\. On-Device AI와 엣지 컴퓨팅: 사용자 중심의 추론 
 - 엣지 컴퓨팅의 확산, 전력소모 최적화 

7\. 지능형 반도체(NPU) 및 PIM: 추론 전용 하드웨어의 부상
 - NPU(Neural Processing Unit)와 수직적 통합 
 - PIM(Processing-In-Memory)과 메모리 혁신

### [주기동] 엔터프라이즈 생성형 AI 신뢰성

**[정의]**

- 단순 정확도를 넘어 실제 운영 환경에서의 강건성, 지속적인 운영 가능성 그리고 안정적 동작 능력의 포괄적인 의미

**[핵심/키워드]**

&lt;생성형 AI 시스템 신뢰성 확보 관련 기술&gt;
 1\. 데이터 인프라 기반 신뢰성 확보 기술 
 - 데이터 레이크하우스(data lakehouse) 
 - 데이터베이스의 동시성 제어와 스냅샷 관리 
 - 브랜치(Branch)와 커밋(Commit) 기반 데이터 관리 

 2\. 모델 및 추론 기반 신뢰성 확보 기술 
 - 모델 출력을 통제하는 구조화 출력(structured output)
 - 지식 그래프(knowledge graph)와 실시간으로 대조 
 - 다단계 자기 교정 추론(self-correction) 프로세스 

1\. 실행 아키텍처 기반 신뢰성 확보 
 - 이벤트 기반 실행 아키텍처(event-driven architecture)
 - 프로세스 기반 실행구조(process-oriented AI) 

2\. 운영 단계 기반 신뢰성 확보 
 - 타임 거버넌스(Runtime Governance) 기반의 AI 운영 관리 

&lt;생성형 AI 시스템 신뢰성 검증 관련 기술&gt;
 1\. 모델 출력 기반 평가 기술  
 -  ‘LLM-as-a-Judge’(평가자로서의 LLM) 접근 방식 
 2\. 공격 기반 검증 기술  
 - AI 레드티밍(AI Red Teaming) 
 3\. 추론 과정 검증 기술  
 - 추론 흔적(reasoning rrace)을 단계별로 분해 
 4\. 실행 과정 및 도구 상호작용 검증 기술 
 - RIVA(Robust Infrastructure by Verification Agents) 아키텍처
