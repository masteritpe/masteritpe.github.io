## 1. 뉴스 요약

2026년 4월, IT 프로젝트의 성공률을 높이기 위한 "비용 산정 5단계 전략"이 보도되었다. 보고서는 IT 프로젝트의 70% 이상이 일정 지연·예산 초과를 겪는 가장 큰 원인을 "초기 비용 산정 실패"로 지목하며, ① 범위 정의, ② 산정 기법 선택, ③ 리스크·우발비용 반영, ④ 이해관계자 검증, ⑤ 지속적 재산정의 5단계 절차를 제시했다. AI·클라우드 등 신기술 도입과 SaaS 구독형 비용구조의 확산으로 전통적 LOC 기반 산정의 한계가 부각되면서, 기능점수(FP)·유스케이스 포인트·Delphi 등 복합 기법의 중요성이 다시 강조되고 있다.

## 2. 핵심 개념

### IT 프로젝트 비용 산정 5단계 절차

```
┌──────────────────────────────────────────────────────────┐
│            IT 프로젝트 비용 산정 5단계                     │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  ① 범위 정의 (Scope Definition)                          │
│     └─ WBS, 요구사항 명세, 산출물 식별                   │
│              ↓                                           │
│  ② 산정 기법 선택 (Estimation Method)                    │
│     └─ LOC / FP / UCP / COCOMO II / Delphi               │
│              ↓                                           │
│  ③ 리스크·우발비용 반영 (Risk & Contingency)             │
│     └─ 기술 난이도, 인력 가용성, 외부 의존성             │
│              ↓                                           │
│  ④ 이해관계자 검증 (Validation)                          │
│     └─ PMO, 재무, 사업부 합동 리뷰                       │
│              ↓                                           │
│  ⑤ 지속적 재산정 (Re-estimation)                         │
│     └─ EVM 기반 추적, 변경관리, 베이스라인 갱신          │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 주요 비용 산정 기법 비교

| 기법 | 산정 단위 | 적용 시점 | 장점 | 단점 |
|---|---|---|---|---|
| LOC | 코드 라인 수 | 설계 이후 | 직관적, 측정 용이 | 언어 의존, 초기 적용 곤란 |
| 기능점수(FP) | 사용자 기능 | 분석 단계 | 언어 독립, 표준화(ISO) | 산정자 숙련도 필요 |
| COCOMO II | 규모 + 보정계수 | 설계 단계 | 복합 요인 반영 | 파라미터 튜닝 부담 |
| Use Case Point | 유스케이스 | 분석 단계 | UML 친화적 | 표준화 미흡 |
| Delphi | 전문가 합의 | 초기/불확실 | 신기술 적용 가능 | 주관성, 시간 소요 |

### COCOMO II 핵심 산식

```
Effort(PM) = A × (Size)^B × ∏(EM_i)
  - A: 상수(2.94)
  - Size: KSLOC 또는 FP 환산값
  - B: 1.01 ~ 1.26 (스케일 팩터 5종 합산)
  - EM: 17개 노력승수(Effort Multiplier)
        ※ 제품·플랫폼·인력·프로젝트 속성 반영
```

## 3. 기술사 출제 포인트

- **(논술)** 소프트웨어 비용 산정 기법(LOC, FP, COCOMO II, Delphi)을 비교하고, AI/클라우드 환경에서 기존 기법의 한계와 보완 방안을 논하시오.
- **(논술)** IT 프로젝트의 실패 원인을 비용 산정 관점에서 분석하고, EVM(Earned Value Management)을 활용한 진척·비용 통제 방안을 제시하시오.
- **(약술)** 기능점수(FP) 산정 절차 5단계와 보정 계수(VAF, GSC 14항목)를 설명하시오.
- **(약술)** COCOMO II의 3가지 모델(Application Composition, Early Design, Post-Architecture) 차이를 설명하시오.
- **(약술)** Delphi 기법이 신기술 프로젝트(AI, 양자컴퓨팅 등)에 적합한 이유를 서술하시오.
- **출제 트렌드**: AI·SaaS 구독형 비용 구조, FinOps와 결합된 클라우드 비용 산정, ISO/IEC 20926(FP) 표준 준수.

## 4. 관련 토픽 연계

- **PMBOK 7판 / 프로젝트 관리**: 비용 관리(Cost Management) 지식영역, EVM(PV·EV·AC·CPI·SPI) 지표 체계
- **IT 거버넌스**: COBIT 2019 APO06(예산·비용 관리), ITIL 4 Service Value Chain의 재무관리
- **FinOps**: 클라우드 사용량 기반 동적 비용 산정, Showback/Chargeback 모델
- **소프트웨어공학**: CMMI Level 3의 정량적 프로젝트 관리(QPM), 측정·분석(MA) 프로세스 영역
- **AI 프로젝트 산정**: MLOps 파이프라인 비용, 데이터 라벨링·모델 학습 GPU 비용, 추론 운영비
- **위험관리**: ISO 31000 기반 프로젝트 리스크 식별과 우발비용(Contingency Reserve) 산정
- **계약 모델**: Fixed Price / Time & Material / Cost Plus 계약 유형별 산정 전략
