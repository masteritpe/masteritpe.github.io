## 1. 뉴스 요약

지디넷코리아·직썰·생생비즈플러스 등 보도에 따르면, **금융결제원이 「AI 에이전트 전용 결제 플랫폼」 기술검증(PoC)에 공식 착수**했다. 채병득 금융결제원장은 "AI가 사용자 대신 쇼핑·예약·구독 결제를 수행하는 시대를 겨냥해, **사용자 위임 기반의 에이전트 결제 트랜잭션 표준**을 선도한다"고 밝혔다. 핵심은 ① 사용자가 AI 에이전트에게 결제 권한을 **위임(Delegation)** 하는 구조, ② 에이전트가 결제 요청을 발생시킬 때 **사람과 구분되는 식별·인증 토큰**을 사용하는 표준, ③ 결제망·가맹점·은행 시스템에서 **에이전트 트랜잭션을 인지·승인·취소·분쟁처리**하는 새로운 결제 라이프사이클이다. 비자(Visa)·마스터카드·구글이 각각 **Agentic Commerce Protocol·AP2** 등 글로벌 표준 경쟁을 본격화한 가운데, 한국도 금융결제원 주도로 **공동망 차원의 AI 에이전트 결제 인프라**를 정의하기 시작한 것이다. 시스템구조 관점에서 본 사건은 **사람 중심의 결제 트랜잭션이 자율 에이전트 중심으로 확장되는 패러다임 전환**을 의미하며, 트랜잭션 무결성·인증·권한 위임·감사·결제 취소·이중지불 방지 등 분산 결제 시스템의 모든 핵심 요소를 재설계해야 한다.

## 2. 핵심 개념

### AI 에이전트 결제 트랜잭션 처리 흐름

```
┌──────────────────────────────────────────────────────────────┐
│ ① 위임 등록  : 사용자→에이전트 결제 권한·한도·기간 위임      │
├──────────────────────────────────────────────────────────────┤
│ ② 의도 생성  : 에이전트가 상품·금액·가맹점·약관 제안          │
├──────────────────────────────────────────────────────────────┤
│ ③ 사용자 승인: HITL(Human-in-the-Loop) 또는 자동 승인 조건    │
├──────────────────────────────────────────────────────────────┤
│ ④ 결제 요청  : 에이전트 토큰+사용자 위임증명을 결제망에 전송  │
├──────────────────────────────────────────────────────────────┤
│ ⑤ 망 검증    : 에이전트 식별·위임 한도·거래위험·이중지불 검사 │
├──────────────────────────────────────────────────────────────┤
│ ⑥ 승인·정산  : 가맹점·은행 처리 후 영수증·근거를 사용자 통지  │
├──────────────────────────────────────────────────────────────┤
│ ⑦ 사후 통제  : 분쟁·환불·위임 취소·감사 추적                  │
└──────────────────────────────────────────────────────────────┘
```

### 사람 결제 vs AI 에이전트 결제 시스템 비교

다음 표는 두 결제 모델의 시스템 차이를 정리한다.

| 구분 | 사람 결제 | AI 에이전트 결제 |
|------|-----------|------------------|
| 결제 주체 | 자연인(카드 보유자) | 위임받은 자율 에이전트 |
| 인증 방식 | 카드+OTP+지문 | 위임증명+에이전트 토큰 |
| 의도 검증 | 사용자 직접 입력 | 에이전트 의도+한도 정책 |
| 승인 시점 | 결제 직전 사용자 확인 | 사전 위임·자동 승인 룰 |
| 책임 소재 | 카드 보유자 | 사용자·에이전트 운영사·발급자 |
| 분쟁 처리 | 본인 거래 검증 | 위임 범위·에이전트 행동 검증 |
| 이중지불 위험 | 클릭 실수 수준 | 에이전트 루프·재시도 폭증 |
| 감사 단위 | 개별 거래 | 에이전트 세션·정책·근거 |

### AI 에이전트 결제 플랫폼 핵심 구성요소

- **위임 권한 관리(Delegation Service)**: 한도·기간·가맹점·카테고리·MCC 단위 위임 정책 발급·취소
- **에이전트 ID·토큰**: 에이전트 고유 식별자, 짧은 수명의 위임증명(JWT/VC), 키 회전
- **의도 표현 표준**: 결제 의도(상품·금액·약관·반품정책)를 구조화 메시지로 표현(MCP/AP2 등)
- **위험·사기탐지(FDS)**: 에이전트 행동 패턴, 비정상 반복 결제, 가격·상품 이상치 탐지
- **트랜잭션 무결성**: 멱등키(Idempotency Key)·이중지불 방지·재시도 안전 설계
- **분산 결제망 연계**: 카드사·은행·간편결제·정산 시스템과 표준 API 연동
- **사용자 통지·HITL**: 임계 금액 이상은 사용자 실시간 승인, 영수증·근거 푸시
- **감사·법적 책임**: 에이전트 세션 로그, 위임 증거, 분쟁시 책임 추적 체계

### 도입 시 고려사항

- **표준 경쟁**: Visa Agentic Commerce, MasterCard Agent Pay, Google AP2 등 글로벌 프로토콜 호환성 확보
- **법·제도**: 전자금융거래법상 "이용자 본인 결제" 원칙과 "에이전트 위임 결제"의 법적 정합성 정비
- **소비자 보호**: 잘못된 의도 해석·과다 결제·구독 취소 누락 시 환불·분쟁 절차
- **보안 위협**: 에이전트 탈취·프롬프트 인젝션·악성 가맹점의 에이전트 유도 공격
- **개인정보·결제정보**: 에이전트가 보유하는 카드·인증 정보 최소화, 토큰화 적용
- **트랜잭션 폭증 대비**: 자동화로 인한 TPS 급증, 결제망 용량·QoS·요금 정책 재설계
- **거버넌스**: 어떤 에이전트를 결제망에 등록할지(인증·심사) 기준과 정책 운영
- **상호운용성**: 가맹점·해외 결제망·해외 에이전트와의 토큰·서명 검증 체계 통일

## 3. 기술사 출제 포인트

- AI 에이전트 결제(Agentic Payment)의 정의와 등장 배경
- 사람 결제와 AI 에이전트 결제 시스템의 비교
- 위임 기반 결제 권한(Delegation)·위임증명(VC/JWT) 설계
- 에이전트 식별·인증 표준(MCP, AP2, Agentic Commerce Protocol)
- 분산 결제망에서의 트랜잭션 무결성·멱등성·이중지불 방지 설계
- 에이전트 결제의 사기탐지(FDS)와 행동 기반 이상 탐지
- 휴먼 인 더 루프(HITL) 승인 정책과 자동 승인 임계치 설계
- 에이전트 결제의 분쟁·환불·책임 추적 거버넌스
- 전자금융거래법·소비자보호법과 위임 결제의 법적 정합성
- 에이전트 결제 트랜잭션 폭증에 대비한 결제망 용량·아키텍처 설계
- 프롬프트 인젝션·에이전트 탈취 등 신규 보안 위협 대응
