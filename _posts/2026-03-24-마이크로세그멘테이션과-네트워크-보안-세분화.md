## 1. 뉴스 요약

일루미오(Illumio)가 한국 시장에서 마이크로세그멘테이션(Microsegmentation) 기술의 고속 성장을 전망했다. 제로트러스트 보안 도입이 가속화되는 가운데, 마이크로세그멘테이션은 네트워크 내부의 횡적 이동(Lateral Movement)을 차단하는 핵심 기술로 부상하고 있다. 기존 경계 보안(Perimeter Security)의 한계를 보완하여, 워크로드 단위의 세밀한 접근제어를 구현한다. 동시에 과기정통부는 160억원 규모 사이버보안 펀드 운용사를 모집하며 AI·제로트러스트 분야 투자를 확대하고 있다.

## 2. 핵심 개념

### 마이크로세그멘테이션(Microsegmentation)
- **정의**: 데이터센터·클라우드 환경에서 워크로드(VM, 컨테이너, 프로세스) 단위로 네트워크를 논리적으로 세분화하여 접근을 제어하는 보안 기술
- **목적**: 내부 네트워크의 횡적 이동(East-West Traffic) 차단
- **위치**: 제로트러스트 아키텍처의 핵심 구현 기술

### 기존 네트워크 세그멘테이션 vs 마이크로세그멘테이션

| 구분 | 기존 세그멘테이션 | 마이크로세그멘테이션 |
|------|------------------|---------------------|
| 단위 | VLAN, 서브넷 | 워크로드, 프로세스 |
| 제어 위치 | 네트워크 장비(방화벽) | 호스트/하이퍼바이저 |
| 정책 기반 | IP, 포트 | ID, 라벨, 컨텍스트 |
| 가시성 | North-South 중심 | East-West 포함 |
| 확장성 | 하드웨어 의존 | 소프트웨어 정의(SDN) |

### 구현 방식
1. **에이전트 기반**: 각 워크로드에 경량 에이전트 설치, 커널 수준 트래픽 제어
2. **하이퍼바이저 기반**: VMware NSX 등 가상화 레이어에서 정책 적용
3. **네트워크 기반**: SDN 컨트롤러를 통한 플로우 제어
4. **서비스 메시 기반**: Istio, Envoy 등 사이드카 프록시 활용 (컨테이너 환경)

### 횡적 이동(Lateral Movement) 차단
- **킬 체인 관점**: 초기 침투 → 내부 정찰 → **횡적 이동** → 권한 상승 → 데이터 유출
- 마이크로세그멘테이션은 횡적 이동 단계를 차단하여 공격 확산 범위를 최소화

## 3. 기술사 출제 포인트

### 예상 문제
1. **마이크로세그멘테이션의 개념과 구현 방식을 설명하고, 기존 네트워크 세그멘테이션과 비교하시오**
2. **제로트러스트 환경에서 마이크로세그멘테이션의 역할과 구현 전략을 설명하시오**
3. **클라우드 네이티브 환경에서 워크로드 보안을 위한 마이크로세그멘테이션 적용 방안을 제시하시오**

### 답안 구성 가이드
- 정의 → 필요성(횡적 이동 위협, 내부자 위협) → 기존 방식 비교
- 구현 아키텍처: 에이전트/하이퍼바이저/SDN/서비스메시 방식 비교
- 제로트러스트 원칙과 연계: "Never Trust, Always Verify" + 최소 권한 원칙
- 도입 시 고려사항: 트래픽 가시성 확보 → 정책 설계 → 단계적 적용(모니터링 → 차단)

## 4. 관련 토픽 연계

- **제로트러스트 아키텍처(ZTA)**: NIST SP 800-207 기반 전체 프레임워크
- **SDN/NFV**: 소프트웨어 정의 네트워킹 기반 동적 정책 적용
- **CWPP(Cloud Workload Protection Platform)**: 클라우드 워크로드 보호 플랫폼
- **사이버 킬 체인**: Lockheed Martin 킬 체인, MITRE ATT&CK 프레임워크
- **서비스 메시**: Istio/Envoy 기반 마이크로서비스 간 통신 보안
