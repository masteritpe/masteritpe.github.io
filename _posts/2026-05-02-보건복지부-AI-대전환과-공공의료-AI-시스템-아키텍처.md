## 1. 뉴스 요약

디지털데일리·데일리메디·GNN뉴스통신 등 보도에 따르면, **보건복지부가 「AI 대전환(AX)으로 지역·필수·공공의료 공백을 해소」하는 전략을 발표**했다. 핵심은 ① **지역의료 격차(분만·소아·응급)와 필수의료 인력 부족을 AI 진단 보조·원격 협진·예측 모니터링으로 보완**, ② **공공의료기관(국립대병원·지방의료원·보건소)에 표준화된 AI 인프라(공동 학습·공동 추론)를 보급**, ③ **AI 의료기기 인허가·임상 활용·보험수가까지 전주기 지원체계**를 구축한다는 점이다. 배경에는 **인구절벽·지방 의료 소멸·필수의료 기피·응급실 뺑뺑이 등 구조적 의료위기**가 있다. 같은 흐름으로 **AI특화병원·만성질환 AI 전환·외과수술 AI 추론** 등 의료 AX 사업이 동시에 추진되고 있다. 시스템구조 관점에서 본 사업은 **다기관·다지역에 걸친 분산 AI 인프라**, **민감 의료데이터의 안전한 학습·추론**, **EMR/PACS·전송·결제 시스템과의 연계**, **모델 거버넌스·임상 검증·보험수가 연계**까지 **공공의료용 통합 AI 플랫폼 아키텍처** 설계 과제를 던진다.

## 2. 핵심 개념

### 공공의료 AI 통합 플랫폼 계층 구조

```
┌─────────────────────────────────────────────────────────────┐
│ ⑦ 정책·수가 : AI수가·인허가·근거생성(RWE)                    │
├─────────────────────────────────────────────────────────────┤
│ ⑥ 거버넌스 : 모델·데이터·편향·임상검증 위원회                │
├─────────────────────────────────────────────────────────────┤
│ ⑤ 서비스 : 진단보조·예측·원격협진·환자 챗봇                  │
├─────────────────────────────────────────────────────────────┤
│ ④ AI 플랫폼 : 학습·MLOps·연합학습·모델 레지스트리            │
├─────────────────────────────────────────────────────────────┤
│ ③ 데이터 : EMR·PACS·유전체·웨어러블·청구                    │
├─────────────────────────────────────────────────────────────┤
│ ② 보안·프라이버시 : 가명처리·암호화·접근제어·감사            │
├─────────────────────────────────────────────────────────────┤
│ ① 인프라 : 의료망·국립대병원·지방의료원·보건소·클라우드      │
└─────────────────────────────────────────────────────────────┘
```

### 기존 공공의료 IT vs AI 대전환

다음 표는 두 모델의 차이를 정리한다.

| 구분 | 기존 공공의료 IT | AI 대전환(AX) 의료 |
|------|------------------|---------------------|
| 데이터 활용 | 기관별 EMR·통계 보고 | 다기관 통합·연합학습 |
| 의사결정 | 의료진 단독 판단 | AI 보조 + 의사 최종 판단 |
| 지역 격차 | 의료진 부족=서비스 단절 | 원격 AI 협진·예측 보완 |
| 임상 근거 | 공급자 중심 RCT | 실사용근거(RWE)·연속 학습 |
| 인허가·수가 | 기기 단위·고정 | SaMD 인허가·AI 수가·재평가 |
| 운영 | 기관별 자체 운영 | 공공 공동 플랫폼·표준 API |

### 공공의료 AI 핵심 구성요소

- **데이터 거버넌스**: K-CDM(공통데이터모델)·가명처리·동의관리(IRB·DUA)
- **연합학습(FL)**: 데이터 반출 없이 다기관 공동 학습, 차분 프라이버시 결합
- **다중 모달 모델**: 영상(CT/MRI)·텍스트(EMR)·시계열(바이탈)·유전체 융합
- **MLOps for Health**: 데이터 드리프트·모델 성능 모니터링, 재학습·재인허가
- **임상 통합**: HL7 FHIR·DICOM·SMART on FHIR 기반 EMR 워크플로 연동
- **보안**: 의료망 분리·N2SF·제로트러스트·SBOM·데이터 무결성
- **휴먼 인 더 루프**: AI 추천에 대한 의사 검토·근거 제시(XAI)·책임 추적

### 도입 시 고려사항

- **의료 데이터 편향**: 수도권·대형병원 편중 → 지역 데이터 보강·역가중 학습
- **모델 일반화**: 기관별 장비·프로토콜 차이 → 도메인 적응·표준화
- **인허가 변동성**: 모델 갱신 시 SaMD 재평가 부담 → PCCP(사전 변경 계획) 활용
- **수가·법규**: AI 수가 산정·의료법·개인정보보호법·생명윤리법 동시 충족
- **지속가능성**: 도입 후 운영비·재학습·라이선스 비용 책임 분담
- **재난·사이버위협**: 의료기관 표적 랜섬웨어 대비 BCP·DR·면역화
