## 1. 뉴스 요약

인공지능신문 보도(2026-05-03)에 따르면, **MIT·구글·우스터공대(WPI) 공동 연구진이 「WRING」(Whack-a-mole Reduction by Iterative Neutralization of Groups) 기술**을 발표했다. *"두더지 잡기 딜레마(Whack-a-mole Dilemma)"* 즉 **AI 모델에서 한 그룹의 편향을 줄이면 다른 그룹의 편향이 새로 튀어나오는 현상**을 체계적으로 해결하는 기법이다. 기존 편향 완화 연구는 ① 학습 데이터 재가중(Reweighing), ② 적대적 학습(Adversarial Debiasing), ③ 사후 보정(Post-processing) 등을 사용했지만, **단일 보호 속성(예: 성별)을 다루면 또 다른 속성(예: 인종·연령)에서 편향이 증가**하는 풍선효과가 빈번했다. WRING은 **여러 보호 속성을 반복적·동시적으로 중립화**하여 전체 공정성을 균형 있게 끌어올린다. 소프트웨어공학 관점에서 본 연구는 ① **AI 모델의 비기능 요구사항(공정성·신뢰성)**, ② **AI SW 품질 모델(ISO/IEC 25059)**, ③ **편향 검증·테스트 자동화**, ④ **모델 카드·데이터시트를 통한 산출물 거버넌스**, ⑤ **MLOps 품질 게이트** 까지 통합적으로 설계해야 하는 **AI 소프트웨어 품질·검증 체계** 과제다.

## 2. 핵심 개념

### AI 편향 완화 SW 개발 파이프라인

```
┌──────────────────────────────────────────────────────────────┐
│ ① 요구사항    : 공정성 목표·보호 속성·허용 격차 정의         │
├──────────────────────────────────────────────────────────────┤
│ ② 데이터 분석 : 분포·표본 편향·라벨 편향 진단 (데이터시트)   │
├──────────────────────────────────────────────────────────────┤
│ ③ 모델 학습   : 재가중·적대학습·제약 최적화 (WRING 등)       │
├──────────────────────────────────────────────────────────────┤
│ ④ 공정성 평가 : DI·EOdds·EOpp·Calibration 다지표 측정        │
├──────────────────────────────────────────────────────────────┤
│ ⑤ 회귀·교차   : 다중 보호 속성 동시 검증, 두더지 효과 감지   │
├──────────────────────────────────────────────────────────────┤
│ ⑥ 산출물 관리 : 모델 카드·데이터시트·결정 로그              │
├──────────────────────────────────────────────────────────────┤
│ ⑦ 운영 모니터: 드리프트·공정성 지표 모니터링·재학습          │
└──────────────────────────────────────────────────────────────┘
```

### 기존 편향 완화 vs WRING 방식 비교

다음 표는 두 접근의 차이를 정리한다.

| 구분 | 기존 단일 속성 완화 | WRING (반복 중립화) |
|------|---------------------|---------------------|
| 대상 속성 | 단일 보호 속성 | 다중 보호 속성 동시 |
| 처리 방식 | 1회성 재가중·보정 | 반복·교차 중립화 |
| 부작용 | 풍선효과(다른 그룹↑) | 균형 수렴 |
| 평가 지표 | DI 등 단일 지표 | 다지표 동시 균형 |
| 적용 시점 | 학습 단계 위주 | 학습·검증·운영 전반 |
| 거버넌스 | 모델 단위 보고 | 그룹·속성 매트릭스 보고 |
| 회귀 검증 | 어려움 | 자동화 가능 |
| 운영 안정성 | 드리프트에 취약 | 주기 재학습·재중립화 |

### AI SW 품질·공정성 핵심 구성요소

- **공정성 지표**: 통계적 동등성(DI), 균등 기회(EOpp), 균등 오즈(EOdds), 보정 (Calibration)
- **편향 완화 기법**: 사전(Reweighing·Sampling), 학습 중(Adversarial·Constraints), 사후(Threshold)
- **테스트 자동화**: 메타모픽 테스트, 대조 사례(Counterfactual), 변이 테스트(Mutation)
- **AI SW 품질 모델**: ISO/IEC 25059(AI 품질 특성), ISO/IEC 24029(신경망 견고성)
- **산출물 표준**: 모델 카드(Model Cards), 데이터시트(Datasheets for Datasets)
- **CI/CD for ML**: 데이터·모델·코드 트리플 버전 관리, 공정성 게이트 자동화
- **설명가능성(XAI)**: SHAP·LIME·반사실 설명으로 편향 원인 추적
- **거버넌스 표준**: NIST AI RMF, EU AI Act, 국내 「인공지능법」 신뢰성 의무

### 도입 시 고려사항

- **공정성 정의 합의**: 도메인·이해관계자별 공정성 정의 상충 → 기준 사전 합의
- **다지표 균형**: 불가능 정리(Chouldechova) 인지, 모든 지표 동시 만족 불가
- **데이터 거버넌스**: 보호 속성 수집 시 개인정보 영향평가(DPIA) 동시 진행
- **검증 자동화**: PR 단계에서 공정성 회귀를 차단하는 품질 게이트 구축
- **회귀 데이터셋**: 다중 보호 속성·교차 그룹을 포함한 벤치마크 유지
- **운영 모니터링**: 데이터·콘셉트 드리프트가 공정성 저하 유발 → 주기 재평가
- **표준 준수**: ISO/IEC 25059·ISO/IEC 42001(AIMS)·NIST AI RMF 정합
- **문서화·투명성**: 모델 카드·결정 로그로 외부 감사·이의 제기 대응

## 3. 기술사 출제 포인트

- AI 모델 편향(Bias)의 정의와 발생 원인
- 통계적·개인적·인과적 공정성 지표의 차이
- 사전·학습 중·사후 편향 완화 기법 비교
- 두더지 효과(다중 속성 풍선효과)의 원인과 대응 전략
- AI SW 품질 모델(ISO/IEC 25059) 구성과 품질 특성
- 모델 카드·데이터시트 등 AI 산출물 표준 적용
- AI 회귀 테스트·메타모픽 테스트·반사실 테스트 설계
- MLOps 품질 게이트와 CI/CD for ML 파이프라인
- 설명가능성(XAI)을 활용한 편향 원인 추적
- 운영 단계의 공정성 모니터링·드리프트 대응
- AI 신뢰성 거버넌스 표준(NIST AI RMF·ISO/IEC 42001)
- 공정성·정확도·프라이버시의 트레이드오프 관리
