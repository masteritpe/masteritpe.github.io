## 1. 뉴스 요약

2026-05-19 데일리시큐·전자신문·머니투데이·뉴스1·아주경제·지디넷코리아·비즈니스포스트·데일리안·시선뉴스 등 **10개 이상 매체**가 "**과기정통부, 침해사고 조사 심의위원회 사전 가동**"을 일제히 보도했다. 핵심은 **2026년 10월 시행 예정인 개정 정보통신망법**에 따라 신설되는 "**침해사고 조사 심의위원회**"가 법 시행 전 사전 가동에 들어갔다는 것이다. 초대 위원장에는 **김승주 고려대 교수**가 위촉됐다. 개정 정보통신망법은 ① **중대 사이버 침해사고 발생 시 기업의 신고 여부와 관계없이 정부가 직권으로 조사**할 수 있는 권한을 신설하고, ② **조사 심의위원회가 중대성 판단·조사 범위·재발방지 권고를 심의**하며, ③ **AI 기반 신규 사이버 위협(에이전트형 LLM 공격, 자동화 봇 공격 등)에 선제적으로 대응**하는 체계를 갖춘다. 이는 국내 침해사고 대응 거버넌스의 패러다임이 **자율 신고 중심에서 정부 직권 조사 중심**으로 전환됨을 의미하며, ISMS-P·ISO 27001·NIST CSF 기반 기업 보안체계와 직접 연결되는 정책 변화다. 기술사 시험 정보보안 영역에서 **사이버 침해사고 대응 체계, 정보통신망법, 직권조사권, 보안 거버넌스**는 빈출 주제로 본 회차(139회) 출제 가능성이 매우 높다.

## 2. 핵심 개념

### 침해사고 조사 심의위원회 개요

| 항목 | 내용 |
|---|---|
| 법적 근거 | 정보통신망법 개정안(2026-10 시행) |
| 위원회 구성 | 학계·산업계·법조계 전문가 + KISA·과기정통부 |
| 초대 위원장 | 김승주 고려대 정보보호대학원 교수 |
| 핵심 권한 | 중대 침해사고 직권조사, 조사 범위 심의, 재발방지 권고 |
| 대상 사고 | 중대성·파급력 기준(사회 인프라·다수 이용자 영향) |
| 사전 가동 시점 | 2026-05-19 (법 시행 5개월 전 선제 출범) |

### 기존 신고 중심 체계 vs 직권조사 체계

| 구분 | 기존(자율 신고) | 개정 후(직권조사 병행) |
|---|---|---|
| 조사 개시 | 기업 신고 시에만 가능 | **기업 신고 없어도 정부 직권 조사** |
| 중대성 판단 | 기업·KISA 협의 | **조사 심의위가 객관적으로 심의** |
| 조사 범위 | 신고 내용 한정 | 침해 경로·피해 전반·공급망까지 확장 |
| 재발방지 | 권고 수준 | **공식 권고 + 이행 점검** |
| 미신고 인센티브 | 은폐 유인 존재 | 은폐 시 직권조사로 강제 노출 |
| AI 위협 대응 | 사후 대응 | **선제 위협 인텔리전스 기반 대응** |

### 침해사고 조사·대응 통합 거버넌스 아키텍처

```
┌──────────────────────────────────────────────────────────────┐
│ ① 위협 인텔리전스 계층(CTI)                                  │
│   KISA C-TAS·OSINT·MITRE ATT&CK 매핑·다크웹 모니터링         │
│   AI 기반 이상행위 탐지(에이전트형 LLM 공격 탐지 포함)       │
├──────────────────────────────────────────────────────────────┤
│ ② 탐지·신고 계층(Detection & Reporting)                      │
│   기업 SOC/SIEM·EDR·NDR → KISA 신고                          │
│   직권조사 트리거: 미신고·은폐·중대성 임계 초과              │
├──────────────────────────────────────────────────────────────┤
│ ③ 조사 심의 계층(Investigation Committee)                    │
│   침해사고 조사 심의위원회 → 중대성·범위 심의                │
│   디지털 포렌식·로그 분석·공급망 추적                        │
├──────────────────────────────────────────────────────────────┤
│ ④ 대응·복구 계층(Response & Recovery)                        │
│   격리·차단·복구·증거 보전(Chain of Custody)                 │
│   ISMS-P 통제 항목 기반 재발방지 통제 강화                   │
├──────────────────────────────────────────────────────────────┤
│ ⑤ 거버넌스·법제 계층(Governance)                             │
│   정보통신망법·개인정보보호법·국가사이버안보법(추진중)       │
│   ISO 27001·NIST CSF·ISMS-P 인증과 연계                      │
└──────────────────────────────────────────────────────────────┘
```

### 침해사고 대응 라이프사이클(NIST SP 800-61 기반)

| 단계 | 활동 | 직권조사 체계 연계 |
|---|---|---|
| 준비(Preparation) | IR 플랜·플레이북 수립 | 심의위 가이드라인 반영 |
| 탐지·분석 | SIEM·EDR·CTI 통합 분석 | AI 기반 침해 탐지 |
| 봉쇄·박멸·복구 | 격리·차단·복원 | 증거 보전 후 조치 |
| 사후 활동 | 포렌식·교훈 도출 | 심의위 재발방지 권고 이행 |

## 3. 기술사 출제 포인트

- **침해사고 조사 심의위원회**의 신설 배경, 법적 근거(개정 정보통신망법), 권한
- **직권조사권**의 도입 의의와 자율 신고 중심 체계와의 비교
- **사이버 침해사고 대응 라이프사이클**(NIST SP 800-61)과 단계별 핵심 활동
- **CTI(Cyber Threat Intelligence)** 체계 및 KISA C-TAS·MITRE ATT&CK 연계
- **디지털 포렌식**의 증거 보전(Chain of Custody)과 침해사고 조사 절차
- **ISMS-P·ISO 27001·NIST CSF**와 침해사고 대응 통제 항목 매핑
- **AI 시대 신규 사이버 위협**(에이전트형 LLM·딥페이크·자동화 봇)과 선제 대응
- **공급망(SBOM) 침해**와 광범위 침해사고 조사 범위 확장 이슈
- **침해사고 조사 거버넌스 도입 시 고려사항**(기업 비밀 보호, 영업비밀과 조사권 충돌, 국제 협력)
