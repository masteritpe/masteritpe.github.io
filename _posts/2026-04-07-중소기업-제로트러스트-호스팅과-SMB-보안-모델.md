## 1. 뉴스 요약

2026년 4월, 나루씨큐리티와 비아웹이 중소기업 전용 "제로트러스트 호스팅" 서비스를 출시했고, 모니터랩은 중소기업 클라우드 보안 구축 비용을 75% 절감하는 패키지를 발표했다. 대기업 중심으로 확산되어 온 제로트러스트 아키텍처(ZTA)가 비용·전문인력 부족으로 도입이 어려웠던 중소기업(SMB) 영역으로 본격 확장되는 신호다. 호스팅·MSSP·SASE 사업자가 ZTA 기능을 패키지화·구독화하여 제공하는 흐름은, NIST SP 800-207의 원칙을 SMB 환경에 맞춰 단순화·서비스화하는 새로운 보안 모델로 평가된다.

## 2. 핵심 개념

### 제로트러스트 핵심 원칙(NIST SP 800-207)

```
┌──────────────────────────────────────────────────────────┐
│           Zero Trust Core Tenets (NIST 800-207)          │
├──────────────────────────────────────────────────────────┤
│  1. 모든 데이터·서비스를 자원(Resource)으로 간주          │
│  2. 위치(네트워크)와 무관하게 모든 통신을 보호            │
│  3. 세션 단위로 자원 접근 허용 (No persistent trust)      │
│  4. 동적 정책에 따른 접근 결정(사용자·기기·환경 컨텍스트) │
│  5. 모든 자산의 무결성·보안 상태를 지속적으로 모니터링    │
│  6. 접근 허용 전 모든 인증·인가를 엄격히 시행             │
│  7. 자산·네트워크·통신에 대한 정보를 최대한 수집·활용     │
└──────────────────────────────────────────────────────────┘
```

### SMB용 제로트러스트 호스팅 아키텍처

```
                  ┌──────────────────────┐
                  │   사용자 / 디바이스   │
                  └──────────┬───────────┘
                             │  ID + Posture
                             ▼
                  ┌──────────────────────┐
                  │  Policy Enforcement  │  ← MFA, 디바이스 검증
                  │      Point (PEP)     │
                  └──────────┬───────────┘
                             │
                  ┌──────────▼───────────┐
                  │  Policy Decision     │  ← 동적 정책 평가
                  │      Point (PDP)     │     (사용자·시간·위험)
                  └──────────┬───────────┘
                             │
        ┌────────────────────┼────────────────────┐
        ▼                    ▼                    ▼
  ┌──────────┐         ┌──────────┐        ┌──────────┐
  │ 웹/앱    │         │ DB/파일  │        │ SaaS     │
  │ (격리)   │         │ (암호화) │        │ (브로커) │
  └──────────┘         └──────────┘        └──────────┘
        │                    │                    │
        └────────── 마이크로세그먼테이션 ───────────┘
                             │
                  ┌──────────▼───────────┐
                  │ 통합 로깅·SIEM·UEBA  │
                  └──────────────────────┘
```

### 대기업 ZTA vs SMB 호스팅형 ZTA

| 구분 | 대기업 ZTA | SMB 제로트러스트 호스팅 |
|---|---|---|
| 도입 모델 | 자체 구축(IdP/PEP/PDP) | 호스팅·구독형(MSSP) |
| 초기 비용 | 수억~수십억 원 | 월 구독료(75% 절감) |
| 운영 인력 | 전담 SOC | 외주 SOC, MDR |
| 정책 관리 | 세분화·맞춤 | 템플릿 기반 |
| 확장성 | 글로벌 멀티사이트 | 단일 사이트 중심 |
| 표준 준수 | NIST 800-207 풀셋 | 핵심 7원칙 부분 적용 |

## 3. 기술사 출제 포인트

- **(논술)** 제로트러스트 아키텍처(ZTA)의 7대 원칙(NIST SP 800-207)을 설명하고, 중소기업이 이를 도입할 때의 과제와 해결 방안을 제시하시오.
- **(논술)** 전통적 경계 보안 모델(Castle & Moat)의 한계와 제로트러스트 전환 전략을 단계별(Identity → Device → Network → Application → Data)로 논하시오.
- **(약술)** ZTNA(Zero Trust Network Access)와 VPN의 차이점을 설명하시오.
- **(약술)** SASE(Secure Access Service Edge)와 SSE(Security Service Edge)의 구성 요소를 비교하시오.
- **(약술)** 마이크로세그먼테이션의 구현 방식(에이전트형, 네트워크형, 하이퍼바이저형)을 비교하시오.
- **출제 트렌드**: 클라우드 네이티브 보안, MSSP/MDR 외주화, AI 기반 위험 점수화(Risk-based Authentication).

## 4. 관련 토픽 연계

- **표준/가이드**: NIST SP 800-207(Zero Trust Architecture), CISA Zero Trust Maturity Model 2.0, KISA 제로트러스트 가이드라인 2.0
- **관련 기술**: SASE, SSE, ZTNA, CASB, SWG, FWaaS, DLP
- **인증/접근제어**: FIDO2/Passkey, OAuth 2.1, OIDC, SCIM, JIT/JEA(Just-In-Time/Just-Enough-Access)
- **디바이스 보안**: EDR/XDR, MDM, 디바이스 자세(Posture) 평가
- **네트워크**: 마이크로세그먼테이션, SDP(Software Defined Perimeter), Service Mesh mTLS
- **운영 모델**: MSSP(Managed Security Service Provider), MDR(Managed Detection and Response), SOC-as-a-Service
- **관련 토픽**: SASE 통합 아키텍처, 암호화 트래픽 분석, CSAP 클라우드 보안, BeyondCorp(구글)
