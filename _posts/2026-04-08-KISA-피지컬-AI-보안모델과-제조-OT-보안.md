## 뉴스 요약

2026년 4월 7일, **한국인터넷진흥원(KISA)**이 *"제조 혁신 보안 역량 고도화"*를 목표로 **피지컬 AI(Physical AI) 보안 모델 개발**에 착수했다고 발표했다. 같은 시기 경남도·성남시가 피지컬 AI 인재 양성·생태계 조성 사업을 발표했고, 롯데이노베이트는 NPU 기반 피지컬 AI 인프라 최적화 계획을 공개했다. **물리 세계와 직접 상호작용하는 AI(로봇·자율 설비·휴머노이드)**의 산업 도입이 가속되면서, 정부 차원에서 **보안 레퍼런스 모델 표준화**가 시작된 셈이다.

## 핵심 개념

피지컬 AI는 **인지(Perception) → 판단(Decision) → 작동(Actuation)**의 폐루프(Closed Loop)를 갖는 AI 시스템으로, 기존 사이버 보안과 달리 **물리적 안전(Safety)**까지 보장해야 한다.

- **사이버-피지컬 시스템(CPS)**: 센서·액추에이터·제어기·네트워크의 결합
- **AI 런타임 보호**: 모델 무결성, 적대적 입력(Adversarial Example) 탐지
- **OT 보안 표준**: IEC 62443(산업제어시스템 보안), ISO/SAE 21434(차량), IEC 61508(기능안전)
- **모델 공급망 보안**: 학습데이터·가중치·MLOps 파이프라인 무결성
- **물리 안전 가드레일**: 비상정지(E-stop), 안전 PLC, 토크/속도 제한

## 피지컬 AI 위협과 통제

다음은 KISA 보안 모델이 다룰 것으로 예상되는 핵심 영역이다.

| 위협 영역 | 공격 시나리오 | 통제 방안 |
|---|---|---|
| 센서 변조 | LiDAR 스푸핑, 카메라 패치 공격 | 센서 퓨전, 이상치 탐지 |
| 모델 회피 | 적대적 예제로 오작동 유도 | Adversarial Training, 입력 검증 |
| 펌웨어 변조 | 액추에이터 펌웨어 교체 | Secure Boot, TPM 기반 무결성 |
| 네트워크 침투 | OT 네트워크 측면이동 | IEC 62443 영역·도관(Zone & Conduit) |
| 데이터 오염 | 학습 데이터 포이즈닝 | 데이터 출처 검증, DVC 기반 추적 |

## 기술사 출제 포인트

- **사이버-피지컬 시스템(CPS) 보안 아키텍처**: Purdue Model 기반 계층 분리
- **IEC 62443 SL(Security Level) 1~4**와 SR(System Requirement) 매핑
- **기능안전(IEC 61508) + 보안(IEC 62443) 통합 설계**: Safety-Security Co-engineering
- **AI 모델 보안**: MITRE ATLAS의 적대적 ML 위협 분류체계
- **디지털 트윈 기반 보안 시뮬레이션**과 사고 대응
- **NIS2·CRA(Cyber Resilience Act)** 등 EU 제품 사이버 보안 규제와의 정합성

## 관련 토픽 연계

KISA 사업은 **원전 DX·산업제어시스템 AI 융합(2026-04-07)**, **디지털 트윈과 산업 AI 융합(2026-03-30)**, **국방 AI(2026-04-03)** 흐름과 동일한 축에 있다. 모두 **AI를 물리 시스템에 결합할 때의 안전·보안·신뢰성** 문제를 다룬다. 기술사 시험에서는 *"스마트팩토리 보안 거버넌스"*, *"CPS 보안 아키텍처 설계"*, *"제조 AX와 OT 보안 정합"* 형태로 직접 출제될 가능성이 매우 높다. 특히 **국가 차원의 보안 레퍼런스 모델 개발 착수**라는 사건성은 시사 답안의 핵심 근거로 활용 가치가 크다.
