## 1. 뉴스 요약

뉴스웍스 보도(2026-05-05)에 따르면, **일본 정부가 공무원용 생성형 AI 「겐나이(GENNAI)」를 18만 중앙 공무원에게 보급**한다고 발표했다. 겐나이는 ① **공공 데이터·내부 문서 검색을 위한 RAG(Retrieval-Augmented Generation)**, ② **문서 작성·요약·번역 등 행정 업무 자동화**, ③ **개인정보·기밀 데이터의 정부 클라우드 내 폐쇄형 운영(소버린 AI)** 을 핵심으로 한다. 한국은 행정안전부 「온AI」, 과기정통부의 자체 LLM 모델 개발이 진행 중으로 비교 대상이다. 소프트웨어공학 관점에서 본 사례는 ① **공공 LLM 소프트웨어의 SDLC와 인수·운영 거버넌스**, ② **RAG 파이프라인 설계와 문서 보안**, ③ **공공 SW 도입 모델(자체개발 vs 외산 도입 vs 하이브리드)**, ④ **AI 시스템 품질·신뢰성 검증(ISO/IEC 25059)**, ⑤ **변경관리·교육·운영 SLA** 까지 통합 설계해야 하는 **공공 LLM SW 도입·운영 전략** 과제다.

## 2. 핵심 개념

### 공공 LLM 소프트웨어 SDLC

```
┌──────────────────────────────────────────────────────────────┐
│ ① 요구분석   : 행정 도메인 유스케이스·법적 제약·KPI 정의     │
├──────────────────────────────────────────────────────────────┤
│ ② 도입 모델  : 자체개발/외산도입/하이브리드 비교·결정        │
├──────────────────────────────────────────────────────────────┤
│ ③ 설계       : RAG 인덱스·프롬프트·툴체인·접근통제 설계      │
├──────────────────────────────────────────────────────────────┤
│ ④ 구현       : 모델·벡터DB·게이트웨이·플러그인 통합          │
├──────────────────────────────────────────────────────────────┤
│ ⑤ 검증       : 환각·편향·보안·성능 회귀 테스트              │
├──────────────────────────────────────────────────────────────┤
│ ⑥ 인수·전개  : 표준 인수기준, 단계적 롤아웃, 사용자 교육     │
├──────────────────────────────────────────────────────────────┤
│ ⑦ 운영       : 관측·드리프트·재학습·SLA·감사 로그            │
└──────────────────────────────────────────────────────────────┘
```

### 공공 LLM 도입 모델 비교

다음 표는 도입 옵션의 차이를 정리한다.

| 구분 | 자체개발(소버린) | 외산 SaaS 도입 | 외산 모델 온프레 | 하이브리드 |
|------|-------------------|------------------|---------------------|--------------|
| 데이터 주권 | 매우 높음 | 낮음 | 높음 | 중간 |
| 비용 | 매우 높음 | 사용량 과금 | 라이선스+인프라 | 분리 청구 |
| 도입 속도 | 느림(년 단위) | 매우 빠름 | 보통 | 빠름 |
| 성능(SOTA) | 후행 | 최첨단 | 첨단 모델 가능 | 최적 조합 |
| 보안 통제 | 완전 통제 | 약관 의존 | 정부 클라우드 격리 | 데이터 분리 |
| 커스터마이징 | 자유 | 제한적 | 파인튜닝 | 모듈 단위 |
| 공급망 리스크 | 자국 인재 의존 | 외산 종속 | 라이선스 종속 | 분산 |
| 사례 | 일본 겐나이·한국 자체LLM | 일부 지자체 | 행안부 온AI | 다수 부처 |

### 공공 LLM SW 핵심 구성요소

- **모델 계층**: 한국어·일본어 특화 LLM(소버린), 도메인 파인튜닝 어댑터(LoRA)
- **RAG 파이프라인**: 문서 인덱싱·청크·임베딩·벡터DB·재정렬(Re-ranker)
- **접근 통제**: 부서·직급·문서 등급별 RBAC/ABAC, 마스킹·익명화
- **프롬프트 거버넌스**: 시스템 프롬프트 표준, 사용자 프롬프트 라이브러리
- **품질 검증**: 환각률·정확도·편향 지표, ISO/IEC 25059 AI 품질 모델
- **보안 통제**: 데이터 유출 방지(DLP), 프롬프트 인젝션 방어, 감사 로그
- **SDLC 통합**: 모델·데이터·코드 트리플 버전 관리, MLOps·CI/CD for ML
- **변경관리**: 사용자 교육, 변경 영향 분석, 롤아웃·롤백 계획

### 도입 시 고려사항

- **법적 적합성**: 「개인정보보호법」·「공공기록물법」·문서 보존·이의 제기 대응
- **데이터 거버넌스**: 학습·RAG 인덱싱 대상 문서의 등급·보존주기·열람 권한
- **소버린 AI**: 외산 모델의 정부 데이터 학습 차단, 폐쇄망·온프레 우선
- **표준 인수기준**: 정확도·환각률·응답시간·보안성에 대한 정량 기준 사전 합의
- **공공 SW 대가산정**: AI SW의 공정한 대가 산정·SaaS 과금 가이드라인 적용
- **사용자 교육**: 프롬프트 엔지니어링·결과 검증·금지 사용 사례 교육
- **운영 SLA**: 가용성·응답시간·환각 사고 대응·보안 사고 대응 SLA 명시
- **지속 평가**: 배포 후 환각·편향 모니터링, 사용자 피드백 루프, 분기 재평가
