## 1. 뉴스 요약

KT클라우드가 고가용성(HA) 플랫폼으로 CSAP(클라우드 보안 인증) 상위 등급을 획득하며 공공 클라우드 인프라 시장 공략에 나섰다. 공공기관의 클라우드 전환이 가속화되면서, 단순 기능 충족을 넘어 고가용성·재해복구·데이터 주권을 보장하는 클라우드 보안 인증 체계의 중요성이 커지고 있다. CSAP 인증은 공공 클라우드 도입의 필수 요건으로, 보안 통제 항목부터 시스템 아키텍처 설계까지 포괄적 평가를 수행한다.

## 2. 핵심 개념

### CSAP 인증 체계 구조

```
┌──────────────────────────────────────────────┐
│     CSAP(Cloud Security Assurance Program)    │
│         클라우드 보안 인증 체계                │
├──────────────────────────────────────────────┤
│  ┌──────────┐ ┌──────────┐ ┌──────────────┐ │
│  │  하 등급  │ │  중 등급  │ │  상 등급     │ │
│  ├──────────┤ ├──────────┤ ├──────────────┤ │
│  │ 공개 정보 │ │ 비공개·  │ │ 중요 정보    │ │
│  │ 처리     │ │ 민감 정보 │ │ 시스템       │ │
│  │          │ │ 처리     │ │ (국가 핵심)  │ │
│  │ 14개 분야│ │ 14개 분야│ │ 14개 분야    │ │
│  │ 78개 항목│ │ 109개항목│ │ 130개 항목   │ │
│  └──────────┘ └──────────┘ └──────────────┘ │
├──────────────────────────────────────────────┤
│  평가 분야: 관리체계, 물리보안, 네트워크,     │
│  시스템, 가상화, 데이터, 암호화, 접근통제 등   │
└──────────────────────────────────────────────┘
```

### CSAP 인증 등급별 요구사항 비교

| 구분 | 하 등급 | 중 등급 | 상 등급 |
|------|---------|---------|---------|
| 대상 정보 | 공개 정보 | 비공개·민감 정보 | 국가 핵심 정보 |
| 통제 항목 | 78개 | 109개 | 130개 |
| 가용성 요구 | 99.5% | 99.9% | 99.95% 이상 |
| 이중화 | 주요 장비 | 전체 인프라 | 멀티 리전 DR |
| 물리 보안 | 기본 | 강화 | 최고 수준 |
| 데이터 위치 | 국내 | 국내 특정 리전 | 국내 격리 환경 |
| 침투 테스트 | 선택 | 필수 | 필수(확장) |
| 인증 유효기간 | 3년 | 3년 | 3년(연간 갱신) |

        │  ┌──── AZ-1 ──┐│     │  ┌──── AZ-1 ──┐│

### 고가용성 핵심 설계 원칙

| 설계 원칙 | 구현 방안 | CSAP 연관 항목 |
|----------|----------|---------------|
| 단일 장애점(SPOF) 제거 | 모든 구성요소 이중화·다중화 | 시스템 가용성 |
| 장애 자동 감지·복구 | Health Check, Auto Failover | 운영 관리 |
| 데이터 무손실 보장 | 동기 복제, WAL 아카이빙 | 데이터 보호 |
| 서비스 연속성 | Active-Active, GSLB | SLA 관리 |
| 격리 및 독립성 | AZ/리전 물리적 분리 | 물리 보안 |
| 확장성 | Auto Scaling, Load Balancing | 용량 관리 |

### RTO/RPO 기반 DR 등급 설계

| DR 등급 | RTO | RPO | 구현 방식 | 비용 |
|---------|-----|-----|----------|------|
| Tier 1 (Mirror) | 수 분 | 0 (무손실) | Active-Active 동기 복제 | 최고 |
| Tier 2 (Hot) | 수 분~1시간 | 수 분 | Active-Standby 준동기 | 높음 |
| Tier 3 (Warm) | 수 시간 | 수 시간 | 비동기 복제 + 수동 전환 | 중간 |
| Tier 4 (Cold) | 수일 | 24시간 | 백업 기반 복원 | 낮음 |

## 3. 기술사 출제 포인트

1. **CSAP(클라우드 보안 인증) 제도의 개념과 등급 체계**: 공공기관 클라우드 도입을 위한 CSAP 인증의 목적, 등급별 평가 항목과 차이를 설명하시오.
2. **클라우드 환경의 고가용성(HA) 아키텍처 설계**: 공공 클라우드에서 99.95% 이상 가용성을 확보하기 위한 멀티 AZ·멀티 리전 아키텍처 설계 방안을 논하시오.
3. **RTO/RPO 기반 재해복구(DR) 전략**: 업무 중요도에 따른 DR 등급 분류와 각 등급별 기술적 구현 방안 및 비용 효율성을 비교 설명하시오.
4. **공공 클라우드 데이터 주권과 보안 거버넌스**: 데이터 국지화(Data Localization) 요구사항과 클라우드 보안 거버넌스 체계 수립 방안을 논하시오.

## 4. 관련 토픽 연계

| 연계 토픽 | 연관성 |
|----------|--------|
| N2SF(국가망보안프레임워크) | 공공 클라우드 보안 정책 체계와 CSAP 인증의 연계 |
| 제로트러스트 | CSAP 상위 등급에서 요구하는 지속적 접근 통제 원칙 |
| 데이터베이스 이중화·복제 | HA 아키텍처의 핵심 구성요소로 동기·비동기 복제 기술 |
| SLA/SLO 관리 | 가용성 지표 정의 및 서비스 수준 보장 체계 |
| 클라우드 네이티브 아키텍처 | 컨테이너·마이크로서비스 기반 고가용성 설계 패턴 |
