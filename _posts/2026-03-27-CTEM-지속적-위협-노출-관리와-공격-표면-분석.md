## 1. 뉴스 요약

네트워크 자산 자동 식별을 통해 공격 표면을 줄이는 CTEM(Continuous Threat Exposure Management, 지속적 위협 노출 관리) 기술이 주목받고 있다. 기존의 정기적 취약점 스캔 방식에서 벗어나, 실시간으로 자산을 식별하고 위협 노출도를 평가하며 우선순위 기반 대응을 자동화하는 방향으로 보안 패러다임이 전환되고 있다. Gartner는 2026년까지 CTEM 프로그램을 도입한 조직이 보안 침해를 2/3 이상 줄일 것으로 전망했다.

## 2. 핵심 개념

### CTEM 5단계 프로세스

| 단계 | 활동 | 주요 기법 | 산출물 |
|------|------|-----------|--------|
| Scoping | 비즈니스 자산 및 공격 표면 정의 | 자산 인벤토리, 비즈니스 임팩트 분석 | 보호 대상 목록 |
| Discovery | 자산·취약점·설정 오류 식별 | ASM, 취약점 스캔, OSINT | 노출 목록(Exposure Inventory) |
| Prioritization | 비즈니스 영향도 기반 우선순위 결정 | CVSS + EPSS + 자산 중요도 | 위험 순위 리스트 |
| Validation | 실제 악용 가능성 검증 | BAS, 침투테스트, 레드팀 | 검증된 위협 목록 |
| Mobilization | 대응 조치 실행 및 추적 | SOAR, 티켓팅, 자동 패치 | 조치 완료 보고서 |

### 공격 표면 관리(ASM) 아키텍처

```
[외부 공격 표면]         [내부 공격 표면]         [디지털 공급망]
 - 도메인/서브도메인      - 내부 자산/서비스       - 서드파티 SaaS
 - 공개 API/포트         - IAM 설정 오류          - 오픈소스 의존성
 - 클라우드 인스턴스      - 횡적 이동 경로         - API 연동 지점
        ↓                       ↓                       ↓
     [EASM]              [CAASM/CSPM]             [TPRM/SCA]
        ↘                       ↓                       ↙
              [CTEM 통합 플랫폼 - 위험 상관분석]
                          ↓
              [우선순위 기반 자동 대응(SOAR)]
```

### 기존 취약점 관리 vs CTEM 비교

| 구분 | 기존 취약점 관리(VM) | CTEM |
|------|---------------------|------|
| 범위 | 알려진 자산의 CVE | 전체 공격 표면(Shadow IT 포함) |
| 주기 | 정기 스캔(월/분기) | 지속적·실시간 모니터링 |
| 우선순위 | CVSS 점수 기반 | 비즈니스 맥락 + 악용 가능성(EPSS) |
| 검증 | 스캔 결과 신뢰 | BAS/침투테스트로 실제 악용 가능성 검증 |
| 대응 | 수동 패치 관리 | SOAR 연동 자동화 대응 |
| 관점 | 기술 중심(IT 자산) | 비즈니스 위험 중심 |

### EPSS(Exploit Prediction Scoring System)
- CVSS가 취약점의 **심각도**를 측정하는 반면, EPSS는 향후 30일 내 **실제 악용 확률**을 예측
- 머신러닝 기반으로 CVE 메타데이터, 소셜미디어, 다크웹 정보를 종합 분석
- CVSS High이지만 EPSS Low인 취약점은 우선순위를 낮추는 등 효율적 자원 배분 가능

## 3. 기술사 출제 포인트

### 예상 문제
1. **CTEM(Continuous Threat Exposure Management)의 5단계 프로세스를 설명하고, 기존 취약점 관리(VM)와의 차이점을 비교하시오**
2. **공격 표면 관리(ASM)의 유형(EASM, CAASM)을 구분하고, 클라우드 환경에서의 적용 방안을 설명하시오**
3. **CVSS와 EPSS의 차이를 설명하고, 취약점 우선순위 결정 시 두 지표를 조합하는 방안을 제시하시오**

### 답안 구성 가이드
- CTEM의 5단계를 Scoping → Discovery → Prioritization → Validation → Mobilization 순으로 상세 설명
- VM(Vulnerability Management) → CTEM 전환의 필요성과 기대 효과
- EASM(External Attack Surface Management)과 CAASM(Cyber Asset Attack Surface Management) 구분
- BAS(Breach and Attack Simulation)를 통한 위협 검증 방법론
- SOAR와의 연동을 통한 자동화 대응 워크플로우 설계

## 4. 관련 토픽 연계

- **제로트러스트 아키텍처**: CTEM의 지속적 검증 원칙과 제로트러스트의 "Never Trust, Always Verify" 연계
- **DevSecOps**: CI/CD 파이프라인에 CTEM 프로세스를 통합한 Shift-Left 보안
- **ISMS-P**: 정보보호 관리체계 인증 기준과 CTEM 프로세스의 매핑
- **클라우드 보안(CSPM/CWPP)**: 클라우드 환경의 설정 오류 탐지와 CTEM Discovery 단계 연계
- **위협 인텔리전스(CTI)**: CTEM Prioritization 단계에서 CTI 피드를 활용한 우선순위 결정
