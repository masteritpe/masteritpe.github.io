## 1. 뉴스 요약

보안뉴스·전자신문·디지털타임스·지디넷코리아·뉴시스 등 다수 매체(2026-05-12)는 **과학기술정보통신부**가 **양자기술산업법(가칭) 개정**을 추진하고, **공공기관 양자보안 구축을 단계적으로 의무화**하는 한편 **양자-AI 융합**을 지원하는 정책을 발표했다고 일제히 보도했다. 핵심은 ① **양자 R&D 위주 정책에서 산업화·상용화 정책으로 전환**, ② **공공망·국가 기간 인프라에 양자키분배(QKD)·양자내성암호(PQC) 적용 의무화**, ③ **양자-AI 융합 연구 및 양자 컴퓨팅 활용 지원**의 세 축이다. 네트워크 관점에서 이 정책은 ① **양자 통신 네트워크(QKD)**, ② **포스트양자 암호 전환(PQC TLS·VPN·인증)**, ③ **하이브리드 양자-고전 보안 네트워크**의 국가 단위 도입을 가속화한다. **HNDL(Harvest Now Decrypt Later) 위협**에 대응한 공공 네트워크의 **암호 민첩성(Crypto-Agility)** 확보와 **양자 통신 인프라 의무 표준화**는 기술사 시험에서 네트워크·정보보안 융합 영역의 핵심 주제로 부상한다.

## 2. 핵심 개념

### 양자보안 네트워크 정의

**양자보안 네트워크**는 ① **양자키분배(QKD: Quantum Key Distribution)**를 통해 도청 불가능한 키를 분배하고, ② **양자내성암호(PQC: Post-Quantum Cryptography)**로 양자컴퓨터의 Shor 알고리즘 위협에 대응하며, ③ **양자난수발생기(QRNG)**로 진정한 무작위성을 확보하는 통신 인프라다. 단일 기술이 아닌 **QKD·PQC·QRNG·고전 암호의 하이브리드 결합**으로 구축한다.

### 공공 양자보안 네트워크 아키텍처

```
┌──────────────────────────────────────────────────────────────┐
│ ① 응용 계층: 행정·금융·국방·의료 등 공공 서비스             │
├──────────────────────────────────────────────────────────────┤
│ ② 보안 미들웨어: 암호 민첩성 게이트웨이·키 관리(KMS·HSM)     │
├──────────────────────────────────────────────────────────────┤
│ ③ 전송 보안: PQC TLS 1.3(ML-KEM·ML-DSA), IPsec PQC VPN     │
├──────────────────────────────────────────────────────────────┤
│ ④ 키 분배 계층: QKD 네트워크(BB84·E91)·신뢰노드·QKD-as-Service│
├──────────────────────────────────────────────────────────────┤
│ ⑤ 난수 생성: QRNG(양자난수발생기) ↔ TRNG 결합               │
├──────────────────────────────────────────────────────────────┤
│ ⑥ 물리 계층: 광섬유·자유공간(위성 QKD)·중계기·신뢰노드      │
├──────────────────────────────────────────────────────────────┤
│ ⑦ 거버넌스: 양자보안 의무화 정책·표준·인증(공공기관 적용)   │
└──────────────────────────────────────────────────────────────┘
```

### 고전 암호 네트워크 vs 양자보안 네트워크 비교

다음 표는 고전 암호 기반 네트워크와 양자보안 네트워크의 차이를 정리한다.

| 구분 | 고전 암호 네트워크 | 양자보안 네트워크 |
|------|--------------------|--------------------|
| 키 분배 | Diffie-Hellman·RSA | QKD(물리법칙 기반)·PQC KEM |
| 양자 위협 | Shor로 RSA·ECC 깨짐 | 양자 컴퓨터 내성 |
| 도청 탐지 | 불가능 | QKD는 도청 시 양자상태 교란 탐지 |
| 표준 알고리즘 | RSA·ECC·AES | ML-KEM·ML-DSA·SLH-DSA(NIST PQC) |
| 인프라 비용 | 일반 광·이더넷 | QKD 전용 광로·중계기 필요 |
| 거리 한계 | 수천 km | QKD 100~500km(중계기·위성 보완) |
| HNDL 대응 | 취약 | PQC·QKD 결합으로 대응 |
| 도입 단계 | 광범위 운영 중 | 공공·국방 우선, 단계적 확산 |

### 핵심 구성요소

- **QKD(양자키분배)**: BB84·E91·CV-QKD 프로토콜, 광섬유 QKD·자유공간/위성 QKD(예: 한국 시범망·중국 묵자호·유럽 EuroQCI)
- **QKD 네트워크 구성**: 신뢰노드(Trusted Node) 기반 다구간 중계, 양자중계기(Quantum Repeater) 미래 적용
- **PQC 표준**: NIST FIPS 203(ML-KEM, Kyber), FIPS 204(ML-DSA, Dilithium), FIPS 205(SLH-DSA, SPHINCS+)
- **PQC TLS·VPN**: TLS 1.3 하이브리드 키 교환(X25519+ML-KEM), IPsec IKEv2 PQC, OpenSSL·BoringSSL 적용
- **QRNG**: 광자·진공 요동 기반 양자 난수, KMS·HSM 시드로 활용
- **암호 민첩성(Crypto-Agility)**: 알고리즘·키·인증서를 추상화해 PQC로 무중단 전환, 인벤토리·자동 교체 도구
- **HNDL 대응**: 장기 보존 트래픽·문서에 PQC 우선 적용, 키 수명·재암호화 정책
- **거버넌스·표준**: 양자기술산업법, ITU-T Y.3800 시리즈(QKDN), ETSI QKD, ISO/IEC JTC1 SC27 PQC

### 도입 시 고려사항

- **단계적 의무화**: 국가 기간망·국방·금융 우선, 일반 공공 후속, 민감도·법적 보존기간 기반 우선순위
- **하이브리드 전환**: PQC만/QKD만보다 PQC+QKD+고전 알고리즘 하이브리드로 위험 분산
- **암호 인벤토리**: 알고리즘·키·인증서·라이브러리 가시화, 마이그레이션 로드맵 수립
- **성능·지연**: PQC 키·서명 크기 증가(ML-KEM 키 ~1KB, ML-DSA 서명 ~3KB), MTU·핸드셰이크 지연 영향
- **QKD 인프라 비용**: 전용 광로·중계기·신뢰노드 비용, 위성 QKD 보완
- **표준·인증**: NIST PQC, ITU-T Y.3800, KISA·NIS 가이드라인 준수
- **상호운용성**: 멀티벤더 QKD·PQC 라이브러리 상호운용 시험, KMIP·KMS 통합
- **공급망 보안**: 양자 장비·SW의 신뢰 공급망 확보, 백도어·SBOM 검증
- **인력·역량**: 양자 보안 운영·감사 전문가 양성, 공공기관 의무 교육
- **거버넌스**: 양자기술산업법 시행령·고시, 양자보안 인증제도, 양자-AI 융합 R&D 자금

## 3. 기술사 출제 포인트

- 양자보안 네트워크의 개념 정의(QKD·PQC·QRNG)와 도입 필요성
- 고전 암호 네트워크와 양자보안 네트워크의 구조·성능·위협 비교
- QKD 프로토콜(BB84·E91·CV-QKD)과 신뢰노드·양자중계기 아키텍처
- NIST PQC 표준(ML-KEM·ML-DSA·SLH-DSA) 알고리즘 특성과 활용
- PQC TLS 1.3 하이브리드 키 교환과 IPsec VPN PQC 적용 방안
- HNDL 위협과 암호 민첩성(Crypto-Agility) 전략
- QKD 네트워크(QKDN) 표준(ITU-T Y.3800)과 ETSI QKD 표준
- 공공 네트워크 양자보안 의무화 정책과 단계적 도입 전략
- 양자기술산업법과 산업화·상용화 정책 방향
- 위성 QKD·자유공간 QKD를 활용한 광역 양자 통신 구성
- 양자 키 관리(KMS·HSM)와 암호 인벤토리·마이그레이션 로드맵
- 양자-AI 융합 응용(양자 머신러닝·양자 최적화)과 네트워크 연계
