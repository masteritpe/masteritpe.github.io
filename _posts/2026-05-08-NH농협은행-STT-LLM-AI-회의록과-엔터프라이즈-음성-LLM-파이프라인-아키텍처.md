## 1. 뉴스 요약

일간투데이(2026-05-06) 보도에 따르면, **NH농협은행**이 **국내 은행권 최초로 STT(Speech-to-Text)·LLM 기반 ‘NH AI 회의록’ 시스템**을 도입했다. 이 시스템은 ① **녹취 음성을 STT로 자동 텍스트 변환**, ② **LLM이 화자 분리·요약·결정사항·액션아이템 추출**, ③ **회의 내용을 사내 지식베이스에 저장·검색** 가능하도록 한다. **금융권 특성상 보안·컴플라이언스 요건**(개인정보·금융실명·녹취보존)을 충족하면서 **음성-LLM 파이프라인을 엔터프라이즈에 통합**한 사례다. 소프트웨어공학 관점에서 이 시스템은 ① **음성-LLM 멀티모달 파이프라인 아키텍처**, ② **마이크로서비스 기반 STT·LLM·임베딩·검색 분리 설계**, ③ **금융권 보안·로깅·감사 요구사항 반영**, ④ **휴먼인더루프 검증 워크플로**, ⑤ **품질 측정·평가(Eval) 자동화**가 통합되어야 하는 엔터프라이즈 AI 통합 과제다.

## 2. 핵심 개념

### 음성-LLM 회의록 파이프라인 아키텍처

```
┌──────────────────────────────────────────────────────────────┐
│ ① 음성 수집      : 회의실 마이크·VoIP·녹취 게이트웨이        │
├──────────────────────────────────────────────────────────────┤
│ ② 전처리         : 잡음제거·VAD·세그먼트·언어 감지            │
├──────────────────────────────────────────────────────────────┤
│ ③ STT 서비스     : 음성→텍스트 변환(Whisper·Conformer 등)    │
├──────────────────────────────────────────────────────────────┤
│ ④ 화자 분리       : Speaker Diarization(클러스터링·임베딩)    │
├──────────────────────────────────────────────────────────────┤
│ ⑤ LLM 요약·추출   : 요약·결정사항·액션아이템·태깅            │
├──────────────────────────────────────────────────────────────┤
│ ⑥ 검증·교정       : 휴먼인더루프 교정·이의제기 처리           │
├──────────────────────────────────────────────────────────────┤
│ ⑦ 저장·인덱싱     : 벡터DB·임베딩·검색 인덱스                │
├──────────────────────────────────────────────────────────────┤
│ ⑧ 검색·RAG       : 사내 지식베이스 RAG 질의응답              │
├──────────────────────────────────────────────────────────────┤
│ ⑨ 보안·감사       : 접근제어·DLP·로그·녹취 보존 정책          │
└──────────────────────────────────────────────────────────────┘
```

### 수기 회의록 vs STT·LLM AI 회의록

다음 표는 두 방식의 차이를 정리한다.

| 구분 | 수기 회의록 | STT·LLM AI 회의록 |
|------|-------------|-------------------|
| 작성 주체 | 회의 참석자 | STT+LLM 자동, 사람 검증 |
| 작성 시간 | 회의 후 수 시간~수 일 | 회의 종료 직후 분 단위 |
| 화자 식별 | 수기 메모 | 화자 임베딩·다이어리제이션 |
| 누락 위험 | 높음 | 전체 발언 텍스트화 |
| 검색성 | 낮음(파일 단위) | 벡터 검색·키워드 검색 |
| 결정·액션 추출 | 수기 정리 | LLM 자동 추출·태깅 |
| 지식 자산화 | 사일로 | 사내 RAG 지식베이스 통합 |
| 일관성 | 작성자 편차 | 프롬프트·가이드라인 표준화 |

### 핵심 구성요소

- **STT 엔진**: Whisper·Conformer·전문 도메인 모델, 한국어·금융 용어 어휘 강화
- **화자 분리(Speaker Diarization)**: 화자 임베딩·클러스터링, 회의 참석자 매핑
- **LLM 요약·추출 모듈**: 프롬프트 템플릿, 결정사항·액션아이템·이슈 분류
- **임베딩·벡터DB**: 회의록 청크 임베딩, 시맨틱 검색·RAG 백엔드
- **RAG 검색 인터페이스**: 사내 챗봇·검색 포털 연동, 출처 인용·근거 제시
- **휴먼인더루프 워크플로**: 작성자·승인자·기록관 검증 단계, 정정 이력 관리
- **품질 평가 자동화**: WER(STT)·ROUGE·BERTScore(요약)·정확도(액션 추출)
- **보안·컴플라이언스**: 망분리·DLP·암호화·접근제어, 금융실명·전자금융감독규정 준수
- **MLOps·LLMOps**: 모델 버전·프롬프트 버전·리그레션 테스트, 평가셋(Eval) 자동화

### 도입 시 고려사항

- **금융권 보안**: 망분리(영업망/인터넷망), 암호화·DLP, 접근권한·감사 로그
- **개인정보·민감정보**: 녹취 동의, 개인정보 마스킹, 제3자 LLM API 사용 제한
- **녹취 보존 정책**: 전자금융감독규정·내부통제 기준, 보존 기간·삭제 정책
- **할루시네이션 대응**: 출처 인용 강제, 신뢰도 점수, 휴먼인더루프 검증
- **모델 선택**: 사내 sLLM(보안)·외부 LLM(품질) 하이브리드, 금융 도메인 파인튜닝
- **프롬프트 거버넌스**: 결정사항·액션 추출 프롬프트 표준, 버전 관리, 변경관리
- **품질 평가**: WER·CER(STT), ROUGE·BERTScore(요약), 사용자 만족도 추적
- **운영 SLA**: 처리시간·정확도·가용성·장애 대응 SLA, ITSM 통합

## 3. 기술사 출제 포인트

- 음성-LLM 멀티모달 파이프라인 아키텍처 설계
- STT(Speech-to-Text) 모델 비교(Whisper·Conformer·Transducer)
- 화자 분리(Speaker Diarization) 알고리즘과 임베딩 기법
- LLM 기반 요약·추출 프롬프트 엔지니어링
- RAG(Retrieval-Augmented Generation) 아키텍처와 벡터DB 선택
- 마이크로서비스 기반 AI 파이프라인 분리 설계
- LLMOps·MLOps 운영 모델(평가셋·리그레션·프롬프트 버전)
- 금융권 망분리·암호화·DLP·접근제어 적용
- AI 시스템의 휴먼인더루프 검증 워크플로
- WER·ROUGE·BERTScore 등 품질 측정 지표
- LLM 할루시네이션 대응 기법(인용·신뢰도·검증)
- 엔터프라이즈 AI 통합 거버넌스(프롬프트·모델·데이터)
