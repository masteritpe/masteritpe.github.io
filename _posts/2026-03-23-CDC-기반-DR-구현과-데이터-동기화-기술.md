## 1. 뉴스 요약

지디넷코리아 기고에서 DR(재해복구) 구현의 핵심 기술로 CDC(Change Data Capture) 솔루션 기반 데이터 동기화를 다뤘다. CDC는 소스 데이터베이스의 변경 사항을 실시간으로 감지하고 타깃 시스템에 전파하는 기술로, 전통적인 ETL 방식 대비 낮은 지연시간과 시스템 부하를 제공한다.

## 2. 핵심 개념

### CDC(Change Data Capture)
- **정의**: 데이터베이스의 변경 사항(Insert, Update, Delete)을 실시간으로 캡처하여 다른 시스템으로 전달하는 기술
- **방식 비교**:

| 방식 | 원리 | 장점 | 단점 |
|------|------|------|------|
| 로그 기반 | DB 트랜잭션 로그(WAL/Redo) 감시 | 소스 부하 최소, 실시간 | 로그 접근 권한 필요 |
| 트리거 기반 | DB 트리거로 변경 감지 | 구현 간단 | 소스 DB 부하 증가 |
| 타임스탬프 기반 | 변경 시간 컬럼으로 비교 | 단순, DB 독립적 | 삭제 감지 불가 |
| 스냅샷 비교 | 전체 데이터 비교 | 모든 변경 감지 | 대용량 시 비효율 |

### CDC 기반 DR 아키텍처
```
[운영 DB] → [CDC Agent] → [메시지 큐] → [CDC Agent] → [DR DB]
 (Active)    (로그 캡처)   (Kafka 등)    (적용)      (Standby)
```

### DR 유형과 CDC의 역할

| DR 유형 | RPO | RTO | CDC 적합도 |
|---------|-----|-----|-----------|
| Mirror Site | 0 | 즉시 | 동기식 복제 병행 |
| Hot Site | 수분 | 수시간 | 최적 (비동기 CDC) |
| Warm Site | 수시간 | 수일 | 적합 |
| Cold Site | 수일 | 수주 | 불필요 (배치 동기화) |

### 주요 CDC 도구
- **Debezium**: 오픈소스, Kafka Connect 기반, MySQL/PostgreSQL/MongoDB 지원
- **Oracle GoldenGate**: Oracle 엔터프라이즈 솔루션, 이기종 DB 복제
- **AWS DMS**: 클라우드 기반 데이터 마이그레이션/복제 서비스

## 3. 기술사 출제 포인트

> **예상 문제**: CDC(Change Data Capture)의 개념과 구현 방식을 비교 설명하고, DR(재해복구) 시스템에서의 활용 방안을 제시하시오.

- **RPO/RTO**: 복구 시점 목표와 복구 시간 목표에 따른 DR 전략 수립
- **데이터 정합성**: 동기식 vs 비동기식 복제의 정합성 보장 메커니즘
- **이벤트 소싱과 CDC**: 이벤트 드리븐 아키텍처에서 CDC의 활용
- **데이터 파이프라인**: CDC → Kafka → 데이터 레이크 구성
- **마이크로서비스와 CDC**: Outbox 패턴, Saga 패턴에서의 CDC 활용

## 4. 관련 토픽 연계

- **시스템구조**: 고가용성(HA) 아키텍처, Active-Standby, Active-Active 구성
- **IT경영**: BCP/DRP 수립, 비즈니스 영향 분석(BIA)
- **네트워크**: 데이터센터 이중화, 전용회선 vs VPN
- **소프트웨어공학**: 이벤트 드리븐 아키텍처, 마이크로서비스 데이터 관리
