## 1. 뉴스 요약

2026년 4월, "속도냐 보안이냐"라는 솔라나(Solana) 블록체인의 난제가 업계의 집중 조명을 받고 있다. 솔라나는 초당 수만 건의 트랜잭션 처리(TPS) 성능을 자랑하지만, 네트워크 장애와 보안 취약점이 반복되면서 블록체인의 근본적 트레이드오프인 '트릴레마(Trilemma)'가 재조명되고 있다. 동시에 포스트퀀트랩스가 양자컴퓨터로 블록체인 성능을 높이는 테스트넷을 공개하며, 합의 알고리즘의 성능·보안·탈중앙화 간 균형 설계가 차세대 블록체인의 핵심 과제로 부상하고 있다.

## 2. 핵심 개념

### 블록체인 트릴레마(Blockchain Trilemma)

```
                    보안성
                  (Security)
                     △
                    / \
                   /   \
                  / 트릴 \
                 / 레마   \
                /  영역    \
               /____________\
    탈중앙화                  확장성
 (Decentralization)       (Scalability)

 ※ 비탈릭 부테린 제안: 세 가지를 동시에 완벽히 달성할 수 없음
 → 두 가지를 강화하면 나머지 하나가 희생됨
```

### 트릴레마 속성별 특성

| 속성 | 정의 | 측정 지표 | 강화 시 희생 |
|------|------|----------|-------------|
| 보안성(Security) | 공격에 대한 네트워크 내성 | 51% 공격 비용, 검증자 다양성 | 확장성 또는 탈중앙화 저하 |
| 탈중앙화(Decentralization) | 권한·데이터의 분산 정도 | 노드 수, 지니 계수, 나카모토 계수 | 확장성 저하(합의 오버헤드 증가) |
| 확장성(Scalability) | 트랜잭션 처리 능력 | TPS, 최종성 시간, 가스비 | 보안 또는 탈중앙화 저하 |

### 주요 합의 알고리즘 비교

| 합의 알고리즘 | 대표 체인 | TPS | 최종성 | 보안 모델 | 탈중앙화 수준 |
|-------------|----------|-----|--------|----------|-------------|
| PoW(작업증명) | Bitcoin | 7 | 60분(6블록) | 에너지 기반 51% 내성 | 높음(누구나 채굴) |
| PoS(지분증명) | Ethereum 2.0 | 30~100 | 12분(2 에폭) | 스테이킹 기반 경제적 보안 | 중간(검증자 진입장벽) |
| DPoS(위임지분증명) | EOS, Tron | 3,000+ | 3초 | 위임 검증자 신뢰 | 낮음(소수 검증자) |
| PoH+PoS | Solana | 65,000 | 0.4초 | 시간 증명 + 지분 | 낮음(하드웨어 요구) |
| PBFT 변형 | Hyperledger | 10,000+ | 즉시 | 비잔틴 내결함(f < n/3) | 낮음(허가형) |
| Tendermint BFT | Cosmos | 10,000 | 6초 | BFT(2/3 합의) | 중간(검증자 셋) |

### 트릴레마 극복 전략

| 전략 | 접근 방식 | 대표 기술 | 트레이드오프 |
|------|----------|----------|-------------|
| 샤딩(Sharding) | 네트워크를 병렬 파티션으로 분할 | Ethereum Danksharding, Near | 크로스샤드 통신 복잡성 증가 |
| Layer 2 | 오프체인에서 처리 후 메인체인에 증명 제출 | Rollup(Optimistic/ZK), State Channel | 메인체인 보안에 의존, 탈출 지연 |
| DAG 기반 | 블록 대신 DAG 구조로 병렬 처리 | IOTA Tangle, Hedera Hashgraph | 최종성 보장 복잡, 초기 중앙화 |
| 모듈러 블록체인 | 실행·합의·데이터 가용성 계층 분리 | Celestia, Avail | 계층 간 신뢰 모델 복잡 |
| 하이브리드 합의 | 복수 합의 메커니즘 조합 | Avalanche(Snow + DAG) | 프로토콜 복잡성 증가 |

### 솔라나 사례 분석: 속도 vs 보안

```
솔라나 아키텍처 특징:
┌──────────────────────────────────────────────┐
│  PoH(Proof of History) + Tower BFT           │
│  • 글로벌 타임스탬프로 합의 오버헤드 최소화  │
│  • 65,000 TPS, 0.4초 블록 시간               │
├──────────────────────────────────────────────┤
│  보안·안정성 이슈:                            │
│  • 네트워크 중단 다수 발생(검증자 동기화 실패)│
│  • 높은 하드웨어 요구 → 검증자 진입장벽      │
│  • 검증자 수 ~1,900개(ETH ~90만 vs)          │
│  • MEV 봇에 의한 네트워크 혼잡               │
├──────────────────────────────────────────────┤
│  교훈:                                        │
│  확장성 극대화가 보안·탈중앙화 리스크로 귀결  │
│  → 트릴레마는 설계 선택의 문제                │
└──────────────────────────────────────────────┘
```

## 3. 기술사 출제 포인트

### 예상 문제
- **블록체인 트릴레마(Trilemma)의 개념을 설명하고, 이를 극복하기 위한 기술적 접근 방안을 비교 분석하시오**
- **PoW, PoS, BFT 계열 합의 알고리즘의 보안 모델을 비교하고, 각 알고리즘의 위협 요소와 대응 방안을 논하시오**
- **Layer 2 확장 기술(Optimistic Rollup, ZK-Rollup)의 동작 원리와 보안 모델 차이를 설명하시오**

### 답안 작성 키워드
- 트릴레마: 보안성·탈중앙화·확장성의 동시 달성 불가 원리
- 나카모토 계수: 네트워크 장악에 필요한 최소 독립 주체 수(탈중앙화 지표)
- 경제적 최종성(Economic Finality): PoS에서 스테이킹 슬래싱을 통한 안전 보장
- 데이터 가용성(Data Availability): 블록 데이터의 검증 가능한 접근성 보장
- ZK-Rollup: 영지식 증명으로 오프체인 연산의 정당성을 수학적으로 검증
- 모듈러 블록체인: 실행·합의·DA 계층 분리를 통한 각 계층 독립 최적화

### 합의 알고리즘 선택 기준

| 평가 기준 | PoW 적합 | PoS 적합 | BFT 적합 |
|----------|---------|---------|---------|
| 참여자 신원 | 불특정 다수 | 스테이킹 참여자 | 식별된 검증자 |
| 네트워크 규모 | 대규모 공개 | 대규모 공개 | 소·중규모 |
| 최종성 요구 | 확률적 허용 | 준즉시 필요 | 즉시 필요 |
| 에너지 효율 | 낮음 | 높음 | 높음 |
| 적용 환경 | 공개형(비트코인) | 공개형(이더리움) | 컨소시엄·엔터프라이즈 |

## 4. 관련 토픽 연계

| 연계 토픽 | 연관성 |
|----------|--------|
| 분산 시스템 / CAP 정리 | 블록체인 트릴레마는 CAP 정리의 블록체인 버전으로 해석 가능 |
| 비잔틴 장군 문제(BFT) | 합의 알고리즘의 이론적 기반, f < n/3 내결함성 |
| 영지식 증명(ZKP) | ZK-Rollup, ZK-SNARK/STARK 기반 확장성·프라이버시 |
| 포스트양자암호(PQC) | 블록체인 서명 알고리즘의 양자내성 전환 필요성 |
| 스마트 컨트랙트 보안 | 합의 계층 위의 애플리케이션 보안, 형식 검증 |
| 탈중앙화 신원증명(DID) | 블록체인 기반 자기주권 신원 관리 |
| 디지털 자산 규제 | MiCA 등 글로벌 규제와 기술 설계의 상호작용 |
