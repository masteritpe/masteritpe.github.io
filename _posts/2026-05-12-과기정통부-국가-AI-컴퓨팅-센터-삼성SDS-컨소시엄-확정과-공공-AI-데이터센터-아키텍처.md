## 1. 뉴스 요약

한국미디어뉴스통신(2026-05-11) 보도에 따르면, **과학기술정보통신부**는 **국가 인공지능(AI) 컴퓨팅 센터 구축 사업**의 민간 참여자로 **삼성SDS 연합체(컨소시엄)**를 최종 확정했다. 이 사업은 ① **국가가 운영하는 대규모 GPU 컴퓨팅 인프라**를 구축해 ② **국내 AI 연구·산업계가 GPU 부족 없이 대규모 모델을 학습·추론**할 수 있도록 지원하고, ③ **소버린 AI(주권 AI)·국가 LLM 구축의 기반 인프라**로 활용하는 것이 핵심이다. 시스템구조 관점에서 **국가 AI 컴퓨팅 센터**는 ① 수만 장 단위 GPU 클러스터, ② 초고속 인터커넥트(InfiniBand/NVLink), ③ 병렬 파일시스템·고성능 스토리지, ④ MLOps·스케줄러(SLURM·K8s), ⑤ 에너지 효율 PUE·냉각 설계, ⑥ 멀티테넌시·보안 격리, ⑦ 데이터 주권·민감정보 처리 거버넌스를 통합한 **AI 특화 데이터센터(AIDC) 시스템 아키텍처**로 귀결된다. NVIDIA Blackwell·H200 등 차세대 가속기 도입이 본격화되는 시점에, 국가 단위 AI 컴퓨팅 인프라 설계는 기술사 시험의 핵심 주제로 부상한다.

## 2. 핵심 개념

### 국가 AI 컴퓨팅 센터(National AIDC) 정의

**국가 AI 컴퓨팅 센터**는 국가가 주체가 되어 구축·운영하는 **AI 학습·추론 전용 고성능 컴퓨팅(HPC for AI) 인프라**로, 민간 클라우드에 의존하지 않고 **국가 AI 주권·데이터 주권**을 확보하기 위한 공공 AIDC다. 일반 클라우드 데이터센터와 달리 **GPU 밀도·인터커넥트 대역폭·전력 밀도**가 결정적이다.

### 국가 AI 컴퓨팅 센터 시스템 아키텍처

```
┌──────────────────────────────────────────────────────────────┐
│ ① 컴퓨팅 계층: GPU 노드(H100/H200/Blackwell B200)            │
│   - DGX/HGX 노드(8 GPU) × 수천 노드                           │
├──────────────────────────────────────────────────────────────┤
│ ② 인터커넥트: NVLink(노드 내), InfiniBand NDR/XDR(노드 간)   │
│   - 800Gbps 비차단 토폴로지(Fat-Tree / Dragonfly+)            │
├──────────────────────────────────────────────────────────────┤
│ ③ 스토리지: 병렬파일시스템(Lustre/GPFS/WekaFS)               │
│   - NVMe Tier + 객체스토리지 콜드 Tier                        │
├──────────────────────────────────────────────────────────────┤
│ ④ 스케줄링/오케스트레이션: SLURM·K8s·Run:ai·NVIDIA Base Cmd  │
├──────────────────────────────────────────────────────────────┤
│ ⑤ MLOps 플랫폼: 학습·실험관리·모델레지스트리·서빙            │
├──────────────────────────────────────────────────────────────┤
│ ⑥ 멀티테넌시: 네임스페이스·GPU 파티셔닝(MIG)·격리 네트워크    │
├──────────────────────────────────────────────────────────────┤
│ ⑦ 전력·냉각: 액침/액냉(Direct Liquid Cooling), PUE 1.1~1.3   │
├──────────────────────────────────────────────────────────────┤
│ ⑧ 보안·거버넌스: 망분리·N2SF·데이터 주권·감사·접근통제      │
└──────────────────────────────────────────────────────────────┘
```

### 민간 클라우드 vs 국가 AI 컴퓨팅 센터 비교

다음 표는 일반 클라우드 AI 인프라와 국가 AI 컴퓨팅 센터의 차이를 정리한다.

| 구분 | 민간 클라우드 AI | 국가 AI 컴퓨팅 센터 |
|------|----------------|---------------------|
| 운영 주체 | 빅테크(AWS·Azure·GCP) | 정부·국가 컨소시엄 |
| 자원 할당 | 시장 가격·우선순위 | 국가 R&D·전략 산업 우선 |
| 데이터 주권 | 해외 리전 의존 가능 | 국내 완전 통제 |
| GPU 확보 | 클라우드 종속(할당 한계) | 국가 단위 직접 확보 |
| 비용 모델 | 사용량 종량제(고비용) | 공공 보조·정액 지원 |
| 보안 등급 | 일반 클라우드 보안 | N2SF·CSAP 상급·전용 |
| 모델 학습 규모 | 단일 조직 한계 | 국가 LLM·소버린 AI 학습 가능 |
| 활용 대상 | 기업 일반 | 연구기관·스타트업·중소기업 |
| 운영 표준 | 사업자별 상이 | 표준화·공유 가능 |

### 핵심 구성요소

- **GPU 노드(컴퓨팅 코어)**: NVIDIA H100·H200·Blackwell B200(B300), 노드당 8 GPU + NVSwitch, 노드 간 NDR/XDR InfiniBand
- **고성능 인터커넥트**: NVLink(900GB/s 노드 내), InfiniBand 400~800Gbps(노드 간), Fat-Tree/Dragonfly+ 비차단 토폴로지
- **병렬 파일시스템(PFS)**: Lustre·GPFS·WekaFS·VAST Data — 학습 데이터 페치 IOPS·대역폭 보장
- **계층형 스토리지**: NVMe Hot Tier → SSD Warm Tier → 객체스토리지 Cold Tier
- **워크로드 스케줄러**: SLURM(HPC 전통), K8s + Volcano/Kueue(클라우드 네이티브), Run:ai(MIG/스케줄링 통합)
- **MLOps 플랫폼**: Kubeflow·MLflow·NVIDIA Base Command·NeMo, 모델 레지스트리·실험 추적·재현성
- **멀티테넌시 격리**: K8s 네임스페이스·NetworkPolicy, GPU MIG(Multi-Instance GPU) 파티셔닝
- **전력·냉각**: 랙당 60~120kW(차세대), 액침 냉각(Immersion)·DLC(Direct Liquid Cooling), PUE 1.1~1.3
- **운영 거버넌스**: 자원 할당 정책(국가 R&D 우선), 사용량 보고·감사 로그, 데이터 분류·접근통제
- **보안 아키텍처**: 망분리·N2SF·CSAP, 데이터 주권(국내 보관), AI 모델 유출 방지(IP 보호)

### 도입 시 고려사항

- **GPU 수급 리스크**: NVIDIA 공급 제약 → 국산 NPU(리벨리온·퓨리오사·사피온) 병행 도입 전략
- **인터커넥트 설계**: 학습 잡 통신 패턴(All-Reduce, All-to-All) 분석 → 비차단 토폴로지·RDMA 튜닝
- **스토리지 IO 병목**: 대규모 학습은 데이터 페치 병목 발생 → 노드 로컬 캐시·랭크별 prefetch 최적화
- **에너지 효율**: 차세대 GPU는 랙당 100kW 이상 → 액냉 설계, 신재생·VPP 연계, PUE 목표 1.2 이하
- **멀티테넌시 SLA**: GPU 시간 할당·우선순위 큐, 선점·체크포인트·재개 정책
- **데이터 거버넌스**: 학습 데이터 분류(공개·민감·개인정보), 익명화·합성데이터·차분 프라이버시
- **모델 보안**: 모델 가중치 IP 유출 방지, 추론 결과 모니터링, 오용 차단
- **운영 표준화**: 자원 사용 신청·승인·과금 표준(NIST 800-203, ISO/IEC 5338 등) 적용
- **확장성·연속성**: 단일 센터 의존 회피 → 분산 다지역 AIDC, DR·페일오버
- **국제 정합**: EuroHPC·미국 NAIRR 등 글로벌 국가 AI 인프라 거버넌스와의 호환성
