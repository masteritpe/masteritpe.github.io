## 1. 뉴스 요약

2026-05-27 한국대학신문 "**경기대, CSTS 전국 수석 배출… SW 품질·보안 인재 양성 성과**"와 eduplusnews.com "**경기대, 2026년도 CSTS 자격시험 '전국 수석' 배출… 누적 200건 돌파**"가 동시 보도됐다. 표면적으로는 대학 합격자 뉴스이지만, **CSTS(Certified Software Test Specialist, 공인 SW 테스트 전문가)** 는 **TTA가 운영하는 국가공인 SW 테스트 자격**으로 SW산업진흥법 기반 SW 품질 인력양성 체계의 핵심이다. 동시기 **TTA AI 보안 PG 출범(2026-05)**, **AI SW 품질관리 역량강화 정책**, **공공 AI 혁신조달**과 묶여, **SW 품질·테스트 표준화가 AI 시스템 품질 검증으로 확장**되는 흐름과 직결된다. CSTS는 **ISO/IEC/IEEE 29119(SW 테스트 국제표준), ISO/IEC 25010(품질모델), ISTQB**의 한국형 적용 체계이며, 공공 SW 사업 감리·검수·발주 시 인력 기준으로 활용된다. 기술사 소프트웨어공학 영역에서 **SW 테스트 프로세스·V모델·테스트 레벨·AI/ML 테스트 확장**은 매 회차 단골 출제 토픽이다.

## 2. 핵심 개념

### CSTS 자격 체계

| 항목 | 내용 |
|---|---|
| 명칭 | CSTS (Certified Software Test Specialist) |
| 주관 | 한국정보통신기술협회(TTA) |
| 분류 | 국가공인 민간자격, SW 테스트 전문가 |
| 등급 | Foundation Level, Advanced Level(테스트 매니저·테스트 분석가·기술 테스트 분석가) |
| 근거 | SW산업진흥법, 자격기본법, 국가공인 민간자격 고시 |
| 활용 | 공공 SW 감리·검수 인력 기준, SI/SM 사업 입찰 평가 |
| 국제 연계 | ISTQB(국제소프트웨어테스팅자격위원회) 신디케이트 호환 |

### SW 테스트 표준 체계

| 표준 | 범위 | 핵심 내용 |
|---|---|---|
| ISO/IEC/IEEE 29119 | 테스트 프로세스·문서·기법 | 조직·관리·동적 테스트 프로세스, 시험설계 기법 |
| ISO/IEC 25010 | SW 품질 모델 | 기능적합성·성능·호환성·사용성·신뢰성·보안성·유지보수성·이식성 |
| ISO/IEC 33063 | 테스트 프로세스 평가 | 테스트 프로세스 성숙도 모델 |
| TMMi/TPI | 테스트 성숙도 | 조직 테스트 역량 단계별 평가 |
| ISTQB Syllabus | 테스트 지식체계 | Foundation/Advanced/Expert 단계 |
| TTA CSTS | 국가공인 적용 | ISTQB 기반 한국형 자격 |

### V-모델과 테스트 레벨

| 개발 단계 | 대응 테스트 레벨 | 주요 활동 |
|---|---|---|
| 요구사항 정의 | 인수 테스트(UAT) | 사용자 요구 충족 검증, 알파/베타 |
| 시스템 분석 | 시스템 테스트 | 기능·성능·보안·사용성 |
| 아키텍처 설계 | 통합 테스트 | 모듈 간 인터페이스, 빅뱅/하향식/상향식 |
| 모듈 설계 | 단위 테스트(Unit) | 함수·클래스 단위, TDD, 커버리지 |
| 구현 | 정적 분석·코드 리뷰 | SAST, 린트, 코드 컨벤션 |

### 테스트 설계 기법

| 범주 | 기법 | 적용 |
|---|---|---|
| 블랙박스 | 동등분할, 경계값 분석, 결정테이블, 상태전이, 유스케이스 | 명세 기반 시험설계 |
| 화이트박스 | 구문·결정·조건·경로 커버리지, MC/DC | 코드 구조 기반 |
| 경험기반 | 탐색적 테스트, 오류추정 | 베테랑 시험자 활용 |
| 비기능 | 성능·부하·스트레스, 보안, 사용성, 접근성 | ISO 25010 품질특성 |
| AI/ML | 메타모픽, 데이터 변형, 적대적 평가, 편향·환각 평가 | AI 시스템 신뢰성 |

## 3. 기술사 출제 포인트

### 개념 정의와 필요성

- **SW 테스트**: 정의된 조건에서 SW를 실행하거나 평가하여 명세된 요구사항을 충족하는지, 결함을 식별하는 활동(IEEE 829, ISO/IEC/IEEE 29119)
- **CSTS**: TTA가 운영하는 국가공인 SW 테스트 전문가 자격, ISO 29119·ISTQB 기반 한국형 적용 체계
- **필요성**: ① 공공 SW 감리·검수 인력 기준 충족, ② SW 품질사고·재작업 비용 절감, ③ 인공지능기본법·AI 신뢰성 검증으로 테스트 범위 확장, ④ 자동화·CI/CD 환경에서 테스트 역량 의무화

### 기존 방식과의 비교

- **임시적 테스트 vs 표준 기반 테스트**: 경험·관행 → ISO 29119/CSTS 기반 정형 프로세스, 문서·메트릭 체계
- **수동 테스트 vs 자동화·DevOps**: 회귀시험 자동화, CI/CD 파이프라인 내 시프트레프트(정적분석·단위·통합) + 시프트라이트(카나리·옵저버빌리티)
- **전통 SW 테스트 vs AI/ML 테스트**: 결정적 입력→출력 검증 → 확률적·데이터 의존적 시스템 검증(메타모픽, 적대적 평가, 편향·환각·드리프트)

### 아키텍처/프로세스 구성요소

- **조직 테스트 프로세스**: 테스트 정책, 전략, 자원 관리, 표준 절차(ISO 29119-2 Part 2)
- **테스트 관리 프로세스**: 계획→모니터링·통제→완료, 위험기반 테스트(RBT)
- **동적 테스트 프로세스**: 시험설계→구현→실행→결함 보고, 시험기준·범위 정의
- **테스트 인프라**: 시험관리도구(Jira/TestRail), 자동화 프레임워크(Selenium/Playwright/JUnit), 정적분석(SonarQube), 부하시험(JMeter/k6), AI 평가(MLflow, DeepEval, Garak)
- **메트릭**: 결함밀도(KLOC), 결함제거율(DRE), 커버리지(라인·분기·MC/DC), MTTR, 잔존결함 추정

### 도입 시 고려사항

- **인력·자격**: CSTS Foundation·Advanced 인력 확보, 공공 사업 입찰·감리 인력 기준 충족
- **프로세스 성숙도**: TMMi/TPI로 조직 테스트 성숙도 진단, 단계별 개선 로드맵
- **자동화 ROI**: 회귀시험 자동화 우선, 시각·E2E 자동화는 유지비용 검토
- **AI/ML 시스템 확장**: 데이터셋 시험(데이터 품질·라벨링·편향), 모델 시험(정확도·강건성·공정성), 시스템 시험(통합·드리프트·재학습)
- **공공 SW 적용**: 정보시스템 감리 기준, 검수 기준, GS·BMT 인증 연계
- **시프트레프트·시프트라이트 균형**: 개발 초기 결함 차단(코드리뷰·SAST·단위) + 운영 단계 관찰(옵저버빌리티·카나리·실시간 SLO)
- **AI 테스트 도구체인**: LLM 평가(BLEU·ROUGE·LLM-as-Judge), 적대적·환각·편향 평가, 레드팀 자동화(Garak/PyRIT)
- **법·표준**: SW산업진흥법, 인공지능기본법(고위험 AI 사전평가), ISO 25010/29119/42001 연계
