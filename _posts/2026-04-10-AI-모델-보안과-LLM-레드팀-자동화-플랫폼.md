## 뉴스 요약

2026년 4월 8~9일, 국내 보안 기업 팀스톤이 이스라엘 AI 보안 전문 기업 **DeepKeep**과 국내 파트너십을 체결하고 LLM 보안 시장 공략을 본격화했다. DeepKeep은 AI 모델의 취약점을 자동으로 탐지·분석하는 **AI 레드팀 자동화 플랫폼**을 제공한다. 같은 주 개최된 '닷핵(DotHack) 2026' 컨퍼런스에서는 **생성형 AI가 바꾸는 보안 생태계**를 주제로 AI 기반 자동화 보안 경쟁이 진단되었다. 데일리시큐, 디지털데일리, 네이트 등이 보도했으며, AI 모델 자체를 공격 대상으로 보는 'AI for AI Security' 패러다임이 국내에서도 본격화되는 신호다.

## 핵심 개념

### AI 모델 보안 위협 분류(OWASP Top 10 for LLM)

| 순위 | 위협 | 설명 |
|---|---|---|
| LLM01 | 프롬프트 인젝션 | 악의적 프롬프트로 모델 행위 조작 |
| LLM02 | 안전하지 않은 출력 처리 | 모델 출력의 무검증 실행(XSS, SSRF) |
| LLM03 | 학습 데이터 중독 | 훈련 데이터 조작으로 모델 편향·백도어 삽입 |
| LLM04 | 서비스 거부(DoS) | 과도한 토큰·복잡 쿼리로 자원 고갈 |
| LLM05 | 공급망 취약점 | 사전학습 모델·플러그인·데이터셋의 무결성 미검증 |
| LLM06 | 민감 정보 노출 | 학습 데이터 내 개인정보·비밀정보 유출 |

### AI 레드팀(Red Teaming) 자동화 프레임워크

AI 레드팀은 기존 사이버 보안 레드팀 개념을 AI 모델에 적용한 것으로, 다음 3단계로 구성된다.

1. **위협 모델링(Threat Modeling)**: 대상 AI 모델의 입력·출력·학습 파이프라인에서 공격 표면 식별
2. **자동화 공격 생성(Automated Attack Generation)**: 적대적 프롬프트, 탈옥(Jailbreak) 시나리오, 데이터 중독 벡터를 AI가 자동 생성
3. **취약점 분류·보고(Vulnerability Triage)**: 발견된 취약점의 심각도 분류, 재현 가능성 검증, 완화 방안 제시

DeepKeep과 같은 플랫폼은 이 3단계를 **지속적 보안 테스트(Continuous Security Testing)** 파이프라인으로 자동화하여 CI/CD에 통합한다.

### AI 보안 가드레일(Guardrails) 아키텍처

```
사용자 입력 → [입력 필터링] → [프롬프트 가드] → LLM 추론
                                                    ↓
               [출력 검증] ← [콘텐츠 분류기] ← 모델 출력
                    ↓
              안전한 응답 반환
```

가드레일은 입력단(프롬프트 인젝션 차단, PII 마스킹)과 출력단(유해 콘텐츠 필터링, 할루시네이션 탐지)으로 구분되며, 레드팀 테스트는 이 가드레일의 우회 가능성을 검증한다.

## 기술사 출제 포인트

1. **OWASP Top 10 for LLM Applications** 주요 위협을 분류하고, 각 위협별 기술적 완화 방안을 설명할 것.
2. **AI 레드팀과 기존 사이버 보안 레드팀의 차이점** — 공격 대상(모델 vs 시스템), 공격 벡터(프롬프트 vs 익스플로잇), 성공 기준(정렬 위반 vs 권한 탈취).
3. **적대적 공격(Adversarial Attack)** 유형 — 화이트박스(그래디언트 기반), 블랙박스(쿼리 기반), 전이 공격(Transferability) 비교.
4. **AI 보안 거버넌스 체계** — EU AI Act 고위험 AI 시스템의 보안 요구사항, 국내 AI 기본법과의 연계.
5. **DevSecOps와 AI 보안 통합** — ML 파이프라인(MLOps)에 보안 테스트를 내장하는 **MLSecOps** 방법론.

## 관련 토픽 연계

- **프롬프트 엔지니어링**: 방어적 프롬프트 설계와 시스템 프롬프트 보안.
- **연합학습 보안**: 모델 업데이트 과정에서의 비잔틴 공격(Byzantine Attack) 방어.
- **XAI(설명가능한 AI)**: 모델 판단 근거의 투명성 확보와 보안 감사 연계.
- **SBOM/AI BOM**: AI 모델의 구성요소(데이터셋, 프레임워크, 사전학습 모델) 명세 관리.
- **제로트러스트**: AI 에이전트의 권한 최소화와 지속적 행위 검증.

> AI 모델 보안은 기존 애플리케이션 보안(AppSec)과 본질적으로 다른 공격 표면을 갖는다. 기술사 답안에서는 OWASP LLM Top 10 위협 분류를 기반으로, 레드팀 자동화→가드레일 설계→MLSecOps 통합의 체계적 접근을 서술하는 것이 핵심이다.
