## 뉴스 요약

2026년 4월 12~13일, **국토교통부**는 "동남아 5개국에서 한국 스마트시티 기술 실증"을 본격화한다고 발표했다(네이트, 뉴스핌 등). 교통·재난·물관리 분야에서 한국형 스마트도시 기술을 인도네시아·베트남·태국·필리핀·말레이시아 등지에 적용하여, 국내 검증된 도시 플랫폼을 **K-City Network 프로그램** 확장 형태로 수출한다. 같은 기간 **한국수자원공사(K-water)** 는 베트남 켄동정수장에 AI 정수장 운영기술을 첫 상용 수출했다. 이는 공공 부문이 축적한 **도시 디지털 전환(DX) 자산**을 표준화하여 글로벌 상용화 단계로 전환하는 본격 신호로, 스마트시티가 '개별 솔루션'에서 **플랫폼 수출 모델**로 진화하고 있음을 보여준다.

## 핵심 개념

### 스마트시티 아키텍처 5계층 모델

```
[서비스 계층]
  교통·재난·물관리·환경·에너지·안전 서비스
        ↓ (Open API, GraphQL)
[플랫폼 계층]
  도시 통합운영플랫폼 ─── 디지털 트윈 ─── AI 분석엔진
        ↓ (NGSI-LD, FIWARE)
[데이터 계층]
  도시 데이터 허브 ─── 데이터 레이크 ─── 공간정보(GIS) DB
        ↓ (MQTT, CoAP, OneM2M)
[네트워크 계층]
  5G/LoRaWAN/NB-IoT ─── 엣지 컴퓨팅
        ↓
[센싱 계층]
  IoT 센서 ─── CCTV ─── V2X ─── 기상·환경 측정기
```

### 스마트시티 국제 표준 비교

| 표준/프레임워크 | 제정 기관 | 핵심 내용 | 적용 영역 |
|---|---|---|---|
| ISO 37120 | ISO | 도시 서비스·삶의 질 지표(100개 이상) | 성과 측정·벤치마킹 |
| ISO 37122 | ISO | 스마트시티 특화 지표 | 디지털 전환 성숙도 |
| ITU-T Y.4900 시리즈 | ITU | 스마트시티 ICT 아키텍처 | 기술 참조 모델 |
| NGSI-LD | ETSI | 컨텍스트 데이터 교환 표준 | 플랫폼 상호운용성 |
| FIWARE | EU 오픈소스 | 스마트시티 오픈 플랫폼 | 참조 구현체 |
| U4SSC | UN | Sustainable Smart Cities 이니셔티브 | 지속가능성 평가 |

### 디지털 트윈 기반 도시 운영

| 구분 | 내용 |
|---|---|
| 정의 | 물리 도시의 실시간 동기화 가상 복제물, AI 시뮬레이션 기반 의사결정 지원 |
| 구성 요소 | 3D 공간정보, IoT 실시간 데이터, 물리 엔진, AI 예측 모델 |
| 활용 사례 | 교통 흐름 최적화, 홍수 시뮬레이션, 대기질 예측, 도시 계획 시나리오 분석 |
| 성숙도 5단계 | 가상화 → 연결 → 예측 → 처방 → 자율(Autonomous) |
| 기술 스택 | Unreal/Unity, CesiumJS, Apache Kafka, TensorFlow, OSGeo |

### K-City Network 수출 모델 구조

| 단계 | 활동 | 핵심 산출물 |
|---|---|---|
| 1단계: 국내 실증 | 스마트시티 국가시범도시(세종·부산 에코델타) | 기술 검증, 운영 노하우 |
| 2단계: 해외 F/S | KOICA 연계 타당성 조사 | 현지 수요·규제 분석 |
| 3단계: 해외 실증 | 현지 파일럿 구축·검증 | 테스트베드, 표준화 |
| 4단계: 본사업 | 민관 컨소시엄 수출 | 플랫폼 라이선스·SI 계약 |
| 5단계: 운영 지원 | 원격 유지보수·컨설팅 | 지속적 수익 모델 |

### 공공 SW 해외 수출의 기술·제도 쟁점

| 쟁점 | 내용 | 대응 방안 |
|---|---|---|
| 현지화(Localization) | 언어·법규·기후·문화 차이 | 모듈형 아키텍처, 파라미터화 |
| 데이터 주권 | 현지 데이터 현지 저장 의무 | 국가별 리전 분리, 주권 클라우드 |
| 상호운용성 | 기존 시스템과의 통합 | NGSI-LD 표준 준수, Open API |
| 보안 인증 | 현지 보안 인증 획득 | ISO 27001, SOC2, 현지 인증 병행 |
| 지속가능성 | 운영·유지보수 현지 역량 부족 | 현지 인력 교육, 기술이전 프로그램 |

## 기술사 출제 포인트

1. **스마트시티 참조 아키텍처 5계층** — 센싱·네트워크·데이터·플랫폼·서비스 계층의 구성 요소와 데이터 흐름, 계층간 인터페이스 표준(OneM2M, NGSI-LD)을 설명할 것.
2. **디지털 트윈 성숙도 모델** — 가상화 → 연결 → 예측 → 처방 → 자율 5단계의 기술 요건, 물리 엔진과 AI 예측 모델 결합 방식, 도시 운영 의사결정 지원.
3. **스마트시티 국제 표준** — ISO 37120/37122 성과 지표 체계, ITU-T Y.4900 ICT 아키텍처, FIWARE 오픈 플랫폼의 참조 구현.
4. **공공 SW 해외 수출의 기술 쟁점** — 현지화, 데이터 주권, 상호운용성, 보안 인증의 기술적·제도적 대응 방안.
5. **IoT 연결 기술 비교** — LoRaWAN, NB-IoT, LTE-M, 5G mMTC의 특성·적용 시나리오와 스마트시티 센싱 계층 설계.
6. **도시 데이터 허브와 마이데이터** — 도시 통합 데이터 플랫폼의 개인정보 보호, 가명처리·동형암호 적용, 시민 데이터 주권 설계.

## 관련 토픽 연계

- **AIOps 기반 공공 인프라 운영**: AI 정수장·스마트 교통 등 도시 인프라 예지정비와 자율 운영.
- **K-Digital 플랫폼 수출**: 전자정부·디지털 ID·도시 플랫폼의 통합 수출 전략.
- **지속가능 스마트시티(Sustainable Smart City)**: UN SDG 연계, 탄소중립 도시 설계와 ESG 평가.
- **6G·시맨틱 통신**: 차세대 스마트시티 센싱·초저지연 서비스 기반 네트워크.
- **도시 사이버보안(Urban Cybersecurity)**: 스마트시티 플랫폼의 공격 표면 확장과 제로트러스트 적용.
