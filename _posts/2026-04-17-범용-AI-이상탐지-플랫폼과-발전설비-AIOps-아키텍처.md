## 1. 뉴스 요약

한국남부발전·한전KPS·연세대학교가 '범용(Foundation) AI 이상탐지 기술' 공동 개발 MOU를 체결했다. 기존 설비별 개별 AI 모델은 기종·운영조건이 달라지면 재학습이 필요해 현장 확산이 어려웠는데, 이번 협력은 다양한 발전설비 기종에 공통 적용 가능한 적응형(Adaptive) AI 모델을 개발해 이상탐지·예지보전 범용성을 확보하는 것이 목표다. 연합뉴스·뉴스1·에너지신문·핀포인트뉴스 등 다수 매체가 일제히 보도했으며, 같은 주 정부 7회 과기장관회의에서도 '공공 AX와 산업 AI 확산'이 안건으로 의결되어 국가 차원의 산업 AI 아키텍처 논의가 본격화되고 있다.

## 2. 핵심 개념

### 범용 AI 이상탐지 플랫폼의 계층 구조

```
┌─────────────────────────────────────────────────────────────┐
│      발전설비 범용 AI 이상탐지 플랫폼 (AIOps) 아키텍처          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  [L4: 운영·거버넌스 계층]                                     │
│   모델 레지스트리 · 드리프트 모니터링 · A/B 롤아웃             │
│       ▲                                                     │
│       │  MLOps 파이프라인 (재학습/배포)                        │
│  [L3: 지능 계층 - 클라우드]                                   │
│   Foundation Model (사전학습) ─→ Fine-tuning per 기종         │
│   데이터 레이크 · Feature Store · 추론 서비스                  │
│       ▲                                                     │
│       │  단방향 게이트웨이 / DMZ                                │
│  [L2: 엣지 AI 계층 - 발전소 현장]                             │
│   경량 추론 엔진 (TensorRT/ONNX) · 스트림 처리                 │
│   센서 이상 스코어 · 실시간 알람                                │
│       ▲                                                     │
│       │  OT 네트워크 (Modbus/OPC-UA)                           │
│  [L1: 현장 계층]                                              │
│   진동/온도/전류 센서 · PLC · SCADA · HMI                      │
│                                                             │
│  ※ Foundation Model이 기종 간 공통 특징을 학습해 재학습 최소화    │
└─────────────────────────────────────────────────────────────┘
```

### 개별 AI vs 범용(Foundation) AI 이상탐지 비교

| 구분 | 기존 개별 AI 모델 | 범용(Foundation) AI 모델 |
|------|------------------|------------------------|
| 학습 방식 | 설비 1종당 개별 학습 | 대규모 사전학습 + 소량 Fine-tune |
| 확장성 | 신규 기종마다 재구축 | Few-shot/Zero-shot 적응 |
| 데이터 요구 | 기종별 라벨 데이터 대량 | 자기지도학습(SSL) 기반 |
| 도메인 일반화 | 낮음(분포 이동에 취약) | 높음(공통 표현 학습) |
| 운영 복잡도 | 모델 N개 관리 | 1개 베이스 + 어댑터 N개 |
| 엣지 배포 | 경량 가능 | 증류(Distillation)/양자화 필요 |

### 핵심 구성 기술

- **자기지도학습(SSL)**: 라벨 없는 정상 운전 데이터로 사전학습 후 이상 구간 탐지
- **도메인 적응(Domain Adaptation)**: 동일 아키텍처를 유량/온도 등 다른 센서 도메인으로 전이
- **엣지-클라우드 하이브리드**: 1차 엣지 필터 → 2차 클라우드 정밀 분석 → 재학습 루프
- **드리프트 탐지(Drift Detection)**: 입력 분포 변화 감지로 재학습 트리거
- **AIOps 통합**: 이상탐지 → 근본원인분석(RCA) → 자동 조치 → 티켓 생성

## 3. 기술사 출제 포인트

### 예상 문제
1. **산업 현장 적용을 위한 Foundation Model 기반 AI 이상탐지 시스템 아키텍처를 설명하시오**
   - 계층 구조: 현장(센서) - 엣지(경량 추론) - 클라우드(학습/거버넌스)
   - 자기지도학습 사전학습 → Fine-tuning/Adapter 기반 기종 확장
   - OT/IT 분리를 고려한 단방향 게이트웨이·DMZ 설계
   - 운영 지표: MTBF, MTTR, False Alarm Rate, 드리프트 지표

2. **AIOps 기반 예지보전(Predictive Maintenance) 플랫폼 설계 방안을 논하시오**
   - 데이터 파이프라인(수집→정제→Feature Store→학습→배포)
   - MLOps: Continuous Training, 모델 레지스트리, Shadow/A-B 배포
   - 엣지 최적화: 모델 양자화(INT8), 증류, TensorRT/OpenVINO 활용
   - 신뢰도 기반 의사결정(Confidence Gating)과 운영자 검토 루프

3. **엣지-클라우드 하이브리드 AI 추론 구조의 설계 고려사항을 설명하시오**
   - 지연시간 SLA, 네트워크 단절 시 Fail-operational 설계
   - 데이터 중력(Data Gravity)과 프라이버시 경계
   - 추론-학습 분리, 모델 배포 파이프라인 보안

### 답안 작성 키워드
`Foundation Model`, `AIOps`, `자기지도학습(SSL)`, `도메인 적응`, `엣지-클라우드 하이브리드`, `MLOps`, `모델 레지스트리`, `드리프트 탐지`, `Feature Store`, `Purdue Model`, `예지보전(PdM)`, `양자화(Quantization)`, `Confidence Gating`
