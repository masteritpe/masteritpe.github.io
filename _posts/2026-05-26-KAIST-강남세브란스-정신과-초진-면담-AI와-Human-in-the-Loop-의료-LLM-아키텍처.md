## 1. 뉴스 요약

2026-05-24~25 인공지능신문·이뉴스투데이·지디넷코리아·네이트 등 5개 이상 매체가 "**KAIST-강남세브란스, 정신과 초진 면담 지원 AI 기술 개발**"을 보도했다. 핵심은 ① 환자가 의사 진료 전에 **AI와 사전 면담**을 진행해 ② **핵심 질문(core question)을 자동 생성·정제**하고 ③ 의사가 이를 받아 본격 진료에 활용하는 **Human-in-the-Loop(HITL) 임상 보조 AI** 라는 점이다. 이는 단순 의료 챗봇이 아닌 ① **임상 도메인 NLP/LLM**, ② **임상 의사결정지원(CDSS)** 의 새로운 인터페이스, ③ **소프트웨어 의료기기(SaMD) 인허가**·**의료 데이터 거버넌스** 가 결합한 융합 시스템이다. 동일 시기 **지디넷 "AI 정신과 진료, 아직은 견습생 수준"(2026-05-24)** 보도가 AI 단독 진단의 한계와 의사-AI 협업 모델의 필요성을 동시에 짚었다. 앞서 진행된 **AI 특화병원 시범사업(2026-04-21)**, **생성형 AI 의료기기 SaMD 인허가(2026-04-02)**, **보건복지부 AI 대전환(2026-05-02)** 흐름과 같은 방향으로 수렴한다. 기술사 AI데이터분석 영역에서 **임상 LLM·CDSS·HITL 의료 AI**는 출제 비중이 빠르게 늘어나는 토픽으로, 본 회차(139회) 출제 가능성이 있다.

## 2. 핵심 개념

### KAIST-강남세브란스 정신과 초진 AI 개요

| 항목 | 내용 |
|---|---|
| 주관 | KAIST 전산학부, 강남세브란스병원 정신건강의학과 |
| 시점 | 2026-05 기술 개발·공개 |
| 적용 영역 | 정신과 초진(intake) 면담 |
| 핵심 기능 | 환자 사전 대화 → 증상·맥락 수집 → 핵심 임상 질문 자동 생성 |
| 인간-AI 협업 | AI는 보조(Assistant), 진단·처방 권한은 의사 |
| 데이터 흐름 | 환자 응답 → LLM → 구조화 요약 → 의사 EMR 표시 |

### Human-in-the-Loop 의료 AI 4단계 파이프라인

| 단계 | 구성요소 | 주요 처리 |
|---|---|---|
| 환자 인터랙션 | 음성/텍스트 챗 UI, STT, 다국어 처리 | 안전한 환자 대화, 동의 관리, 이상 신호 즉시 인간 호출 |
| 임상 도메인 LLM | 정신과 가이드(DSM-5-TR, K-ICD-11) RAG | 증상 추출, 자살·위해 위험 신호 분류, 후속 질문 생성 |
| 의사 검토 인터페이스 | 구조화 요약·핵심 질문·근거 출처 표시 | 의사가 가감·수정·승인, EMR 자동 연계 |
| 거버넌스·평가 | 임상 평가·바이어스·환각률 모니터링 | 지속 평가, 라벨러 피드백, 모델 갱신 |

### 일반 의료 챗봇 vs HITL 임상 보조 AI

| 구분 | 일반 의료 챗봇 | HITL 임상 보조 AI |
|---|---|---|
| 사용자 모델 | 환자 단독 사용 | 환자 + 의사 협업 |
| 출력 형태 | 답변/안내 텍스트 | 의사용 구조화 요약·핵심 질문 |
| 의사결정 권한 | 권한 모호, 자체 답변 | 의사 결정 보조, 권한 분리 |
| 규제 분류 | 일반 SW | 의료기기 SaMD 가능성 |
| 평가 지표 | 사용자 만족도 | 진단 시간 단축, 진료 품질, 안전성 |
| 데이터 보호 | 일반 개인정보 | 민감 의료정보(가명·익명 처리, 동의 관리) |

## 3. 기술사 출제 포인트

### 개념 정의와 필요성

- **HITL 임상 보조 AI**: 환자-AI-의사 3자 루프에서 AI가 면담·문서화·요약을 자동화하고 의사가 최종 임상 판단을 수행하는 임상 의사결정지원(CDSS) 모델
- **필요성**: ① 정신과·1차 진료의 초진 시간 부족, ② 환자 자기보고의 일관성·완결성 결여, ③ 의사의 차팅 부담(Burnout), ④ AI 단독 진단의 환각·책임소재 문제 회피, ⑤ AI 의료기기 인허가·법적 책임 프레임 정립

### 기존 방식과의 비교

- **종이/구두 문진**: 시간 소모, 누락·기억 의존, 비정형 기록
- **표준화 설문(PHQ-9 등)**: 정량적이나 정형적, 환자 맥락 반영 한계
- **AI 단독 진단 챗봇**: 환각·책임·규제 위험, 안전 임계 시나리오 미흡
- **HITL 보조 AI**: 환자 맥락을 풍부하게 수집하면서도 의사 권한·책임 명확화

### 아키텍처 구성요소

- **환자 인터페이스**: 다국어·음성·텍스트 채널, 위기상황 즉시 인간 연결(자살·자해 신호)
- **임상 LLM Layer**: 정신의학 가이드라인 RAG, 증상-사건-맥락 그래프, 자해 위험 분류기
- **요약·질문 생성기**: 의사용 구조화 출력(주호소·현병력·과거력·가족력·위험요인·후속 질문)
- **의사 워크플로**: EMR 통합(HL7 FHIR), 한 화면 검토·수정·승인 UI
- **거버넌스**: 모델 카드·데이터시트, 안전 임계값, 레드팀·임상시험 평가
- **로그·감사**: 모든 환자 대화·요약·의사 수정 이력 보관, 책임 추적
- **외부 표준 연계**: SaMD(IMDRF), MFDS 가이드, ISO/IEC 42001 AIMS, HL7 FHIR
- **모델 운영**: 임상 결과 기반 지속 평가(MLOps + ClinOps), 드리프트 탐지

### 도입 시 고려사항

- **규제·인허가**: 단순 보조 vs SaMD 경계 판단, MFDS 인증 트랙·임상시험 설계
- **윤리·동의**: 환자 명시 동의, 거부 시 대체 경로, 미성년·취약 계층 보호
- **안전 게이트**: 자살·자해·폭력 위험 신호 감지 시 즉시 의료진 알림(HITL Escalation)
- **데이터 거버넌스**: 가명화·익명화, 데이터 최소 수집, 의료법·개인정보보호법·AI기본법 동시 준수
- **편향·공정성**: 성별·연령·언어·문화권별 모델 편향 정기 평가
- **임상 효과 검증**: 진료 시간·진단 정확도·환자 만족도·재방문율 등 다지표 RCT 평가
- **책임 분배**: 의사의 최종 책임, AI 제조자·의료기관의 부분 책임 명확화
- **임상 통합**: EMR/PACS·예약·청구 시스템과 매끄러운 연계, 의사 워크플로 부담 최소화
