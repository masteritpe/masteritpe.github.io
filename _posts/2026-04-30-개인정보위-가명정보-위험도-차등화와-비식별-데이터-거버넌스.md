## 1. 뉴스 요약

네이트(연합뉴스)·머니투데이 등 보도에 따르면, **개인정보보호위원회가 「가명정보 처리 가이드라인」을 개정해 「위험도 차등화 모델」을 도입**한다. 핵심은 그간 **「가명정보 = 동일 수준의 보호」** 라는 일률적 접근에서 벗어나 ① **저위험 가명정보**(반복 갱신·집계형)는 안전조치 의무를 완화하고 ② **고위험 가명정보**(미세 위치·생체·의료 등 민감정보 결합)는 추가 안전조치(차분프라이버시·합성데이터화·재식별 평가 강화)를 요구한다는 점이다. 이는 **AI 학습 데이터 활용 문턱 완화**와 **재식별 사고 예방**을 동시에 달성하기 위한 조치로, 「데이터 3법」 시행 이후 누적된 **모호성·과잉 규제** 비판을 반영한 변화다. 함께 발표된 내용에는 **결합전문기관 절차 간소화**, **가명정보 결합 후 재식별 위험성 평가(Re-identification Risk Assessment)의 정량 지표화**, **AI 학습용 가명데이터의 「공익적 연구」 범위 명확화** 등이 포함된다. 데이터베이스 관점에서 이는 **「데이터 일생 주기(Lifecycle) 기반 거버넌스」 와 「위험 기반 비식별화 기법 선택」** 이라는 흐름을 가속화한다.

## 2. 핵심 개념

### 비식별화 기법 스펙트럼

```
[원본]──마스킹──일반화──k-익명성──l-다양성──t-근접성──차분프라이버시──합성데이터──[완전비식별]
   ↑            ↓                                          ↑
   유용성 高     ───── 재식별 위험 감소 ─────                유용성 低
```

### 위험도 차등화 모델 매트릭스

| 위험도 | 데이터 특성 | 적용 기법 | 안전조치 |
|------|-----------|---------|---------|
| 저위험 | 통계·집계·코호트 | 일반화 + k≥5 | 표준 접근통제 |
| 중위험 | 거래·이용 로그 | k-익명성 + l-다양성 | 결합전문기관 경유 |
| 고위험 | 위치·생체·의료 | 차분프라이버시(ε≤1) + 합성 | 정량 RRA + 감사로그 |
| 초고위험 | 결합·식별성 결합 | 합성데이터 + Federated | 사전심의 + 사후 모니터링 |

### k-익명성 vs 차분프라이버시 비교

| 구분 | k-익명성 | 차분프라이버시 |
|------|---------|--------------|
| 보호 대상 | 개체(레코드) | 응답·쿼리 결과 |
| 보장 방식 | k개 이상 동일 그룹 | 잡음(ε,δ) 추가 |
| 결합공격 내성 | 약함 | 강함 |
| 활용성 | 높음(평탄) | 중간(잡음 영향) |
| 표준 | ISO 20889 | NIST SP 800-226 |

### 재식별 위험성 평가(RRA) 프로세스

1. **시나리오 모델링**: 동기·자원·외부지식을 가진 가상 공격자 정의
2. **준식별자 추출**: 우편번호·생년월·성별 등 유사 식별자 식별
3. **정량 지표 산정**: 평균 재식별 확률, 최대 그룹 크기, 정보엔트로피
4. **임계치 비교**: 위험도 등급별 허용 임계치(예: P_re ≤ 0.04)
5. **잔여위험 통제**: 추가 마스킹·접근통제·감사

### AI 학습데이터·DB 거버넌스 고려사항

- **데이터 카탈로그**: PII 자동 분류·민감도 라벨링·검색
- **권한·감사**: 행·열 단위 RBAC/ABAC, 동적 마스킹, 쿼리 감사
- **합성데이터 파이프라인**: GAN·디퓨전·LLM 기반 합성과 충실도(Fidelity)·프라이버시 양립
- **결합 거버넌스**: 결합전문기관 → 정량 RRA → 보호조치 → 폐기절차

## 3. 기술사 출제 포인트

- **개인정보·가명정보·익명정보의 정의와 법적 효력 차이**: 개정 보호법·시행령 기준
- **비식별화 기법 비교**: 마스킹·일반화·k-익명성·l-다양성·t-근접성·차분프라이버시 트레이드오프
- **차분프라이버시(ε,δ) 메커니즘**: 라플라스·가우시안·지수 메커니즘 적용 사례
- **재식별 위험성 평가(RRA) 프로세스와 정량 지표**: 평균·최대 재식별 확률, 엔트로피
- **위험도 차등화 거버넌스 도입 시 고려사항**: 분류 기준, 책임 소재, 감사·증적
- **합성데이터(Synthetic Data) 생성 기법과 충실도-프라이버시 트레이드오프**
- **데이터베이스 단의 프라이버시 강화 기술**: 동적 마스킹, 행/열 보안, TDE, 쿼리 재작성
- **개인정보 결합전문기관 운영 모델과 결합 절차**: 결합·반출 분리, 안전구역, 폐기
- **AI 학습데이터 거버넌스**: 출처·라이선스·민감도·재식별 위험·데이터셋 카드(Datasheet)

## 4. 관련 토픽 연계

- **AI 모델 증류 공격과 LLM 지식재산 보안**: 학습데이터·모델 양면 보안
- **지식재산처 AI데이터 IP 통합플랫폼과 AI데이터 지식재산 거버넌스**: 데이터 권리·결합
- **공공기관 AI 챌린지와 국가 AI 인재양성 거버넌스**: 데이터 활용·인재 정책
- **AI SW 품질관리 역량강화와 AI 시스템 신뢰성 검증 체계**: 데이터 품질·편향 검증
- **글로벌 AI 기술 수출 통제와 AI 기술 보호주의 거버넌스**: 데이터·모델 국경 이동
- **제주관광 빅데이터 플랫폼 개편과 공공 도메인 데이터 아키텍처**: 가명·결합 활용 사례
