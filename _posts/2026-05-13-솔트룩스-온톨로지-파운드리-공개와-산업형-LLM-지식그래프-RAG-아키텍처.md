## 1. 뉴스 요약

전자신문·아이뉴스24·디지털타임스·아주경제·머니투데이 등 다수 매체(2026-05-11~12)는 **솔트룩스**가 **온톨로지(Ontology)와 LLM을 결합한 '온톨로지 파운드리(Ontology Foundry)'를 공개**하고 **산업형 AI 시장 본격 공략**에 나섰다고 일제히 보도했다. 핵심은 범용 LLM이 가진 ① 환각(Hallucination), ② 도메인 지식 부정확성, ③ 추론 근거 부재 문제를 **온톨로지·지식그래프(Knowledge Graph)와 결합**해 ④ 검증 가능한 사실 기반 응답, ⑤ 산업 도메인 정합성, ⑥ 추론 경로 설명을 제공한다는 점이다. AI데이터분석 관점에서 이 발표는 **벡터 기반 RAG(Vector RAG)에서 그래프 기반 RAG(GraphRAG)·신경-기호(Neuro-Symbolic) AI로 진화**하는 산업 트렌드의 국내 대표 사례이며, 기술사 시험에서 **지식그래프·온톨로지·GraphRAG**가 새로운 핵심 출제 주제로 부상하는 신호다.

## 2. 핵심 개념

### 온톨로지·지식그래프 기반 LLM 정의

**온톨로지(Ontology)**는 도메인의 **개념·속성·관계·제약**을 형식 언어(RDF/OWL/SKOS)로 정의한 의미 모델이며, **지식그래프(Knowledge Graph)**는 그 위에 사실(트리플 `<주체-술어-객체>`)을 채운 그래프 데이터다. **온톨로지+LLM 결합 시스템**은 LLM의 생성 능력에 **검증 가능한 구조화된 지식**을 결합해 **환각 억제·도메인 정합성·설명 가능성**을 동시에 제공하는 **신경-기호 융합(Neuro-Symbolic) 아키텍처**다.

### 산업형 LLM 지식그래프 RAG 아키텍처

```
┌──────────────────────────────────────────────────────────────┐
│ ① 데이터 원천: 문서·DB·매뉴얼·도메인 표준·외부 KB           │
├──────────────────────────────────────────────────────────────┤
│ ② 지식 추출: NER·관계추출·이벤트추출(LLM·룰 결합)            │
├──────────────────────────────────────────────────────────────┤
│ ③ 온톨로지 모델링: 클래스·속성·관계·SHACL 제약 정의          │
├──────────────────────────────────────────────────────────────┤
│ ④ 지식그래프 저장: 그래프DB(Neo4j·GraphDB·Neptune)·RDF Store │
├──────────────────────────────────────────────────────────────┤
│ ⑤ 하이브리드 인덱스: 벡터DB(임베딩) + 그래프 인덱스 결합     │
├──────────────────────────────────────────────────────────────┤
│ ⑥ GraphRAG 파이프라인: 의도분석·서브그래프 검색·경로 추론   │
├──────────────────────────────────────────────────────────────┤
│ ⑦ LLM 생성·검증: 그래프 근거 주입·SHACL 검증·인용 출처      │
├──────────────────────────────────────────────────────────────┤
│ ⑧ 거버넌스: 출처·버전·신뢰도·편집 워크플로우·감사            │
└──────────────────────────────────────────────────────────────┘
```

### 벡터 RAG vs GraphRAG·온톨로지 결합 비교

다음 표는 벡터 기반 RAG와 그래프 기반 RAG의 차이를 정리한다.

| 구분 | 벡터 RAG | 그래프 RAG(온톨로지 결합) |
|------|----------|-----------------------------|
| 검색 단위 | 텍스트 청크 | 개체·관계·경로(서브그래프) |
| 검색 방식 | 의미 유사도(코사인) | 그래프 탐색·SPARQL/Cypher |
| 환각 억제 | 부분적(컨텍스트 의존) | 강함(제약·검증) |
| 다단계 추론 | 약함 | 강함(경로·관계 활용) |
| 설명 가능성 | 청크 출처 표시 | 추론 경로·근거 그래프 |
| 도메인 정합성 | 임베딩 품질 의존 | 온톨로지 제약으로 보장 |
| 갱신 비용 | 재임베딩 | 트리플 단위 증분 갱신 |
| 적합 과제 | 일반 질의응답 | 산업 규제·진단·의사결정 |
| 구축 난이도 | 비교적 낮음 | 도메인 모델링 필요(높음) |

### 핵심 구성요소

- **온톨로지 모델링**: OWL·RDFS·SKOS·SHACL, 도메인 전문가-AI 협업 편집기, 표준 어휘(FIBO·SNOMED·ISO) 매핑
- **지식 추출**: LLM 기반 개체명 인식(NER)·관계 추출(RE)·이벤트 추출, 신뢰도 점수 부여
- **지식그래프 저장**: 그래프DB(Neo4j·TigerGraph·Neptune), RDF Triplestore(GraphDB·Stardog·Virtuoso)
- **하이브리드 검색**: 벡터 유사도 + 그래프 탐색 결합, 1-홉·N-홉 서브그래프 검색
- **추론 엔진**: OWL 추론기(Pellet·HermiT), 룰 기반(SWRL), 그래프 알고리즘(중심성·커뮤니티)
- **검증·제약**: SHACL Shape으로 응답 사실 검증, 모순 탐지, 누락 탐지
- **LLM 통합**: 그래프 근거를 자연어 컨텍스트로 직렬화, 인용 의무화, 신뢰도 표시
- **거버넌스**: 출처·버전·편집 이력, 마스터 데이터 관리(MDM) 연동, 데이터 카탈로그

### 도입 시 고려사항

- **도메인 모델링 비용**: 온톨로지 설계·검증에 전문가 투입 필수, ROI 산정 필요
- **표준 어휘 활용**: 산업 표준(SNOMED·FIBO·DCAT 등) 채택으로 상호운용성 확보
- **지식 추출 품질**: LLM 추출의 오류 검증·휴먼-인-더-루프(HITL) 큐레이션
- **저장소 선택**: 속성그래프(Neo4j·LPG) vs RDF Triplestore 트레이드오프
- **확장성**: 수십억 트리플 처리, 분산 그래프DB 또는 샤딩 전략
- **갱신·동기화**: 원천 변경 → 그래프 증분 갱신 CDC 파이프라인
- **추론·검증 비용**: 실시간 OWL 추론 비용 큼 → 사전 머터리얼라이즈 전략
- **보안·접근통제**: 그래프 수준 ABAC, 민감 트리플 마스킹·감사
- **LLM 결합 패턴**: 그래프 → 자연어 직렬화 길이, 컨텍스트 윈도우 한계
- **거버넌스·MDM**: 마스터 데이터 일관성, 충돌 해결 정책, 변경 워크플로우

## 3. 기술사 출제 포인트

- 온톨로지·지식그래프의 개념 정의(RDF/OWL/SHACL)와 LLM 결합 필요성
- 벡터 RAG와 GraphRAG의 구조·검색·환각 억제 측면 비교
- 신경-기호(Neuro-Symbolic) AI 아키텍처와 산업형 AI 적용 분야
- 지식 추출(NER·RE·이벤트) 파이프라인과 LLM 활용
- 속성그래프(LPG) vs RDF Triplestore의 트레이드오프
- SHACL Shape 기반 응답 사실 검증과 환각 탐지
- 하이브리드 검색(벡터 + 그래프)과 서브그래프 추출 전략
- OWL 추론·룰 추론(SWRL) 활용과 사전 머터리얼라이즈
- 도메인 표준 어휘(FIBO·SNOMED·DCAT) 활용과 상호운용성
- 지식그래프 거버넌스(출처·버전·MDM·접근통제)
- 산업·규제 도메인(금융·의료·제조)에서 설명 가능 AI 요구사항
- 그래프 증분 갱신 CDC 파이프라인 설계
