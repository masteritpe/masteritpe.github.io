## 1. 뉴스 요약

2026-05-18 조선비즈·이데일리·농민신문은 정부와 LG전자가 주도하는 **차세대 한국형 휴머노이드 로봇(K-휴머노이드) 국가과제**가 본격 착수됐다고 일제히 보도했다. 핵심 골자는 ① **산업통상자원부 주도**로 한국형 휴머노이드 핵심기술 개발·실증·양산 전 주기를 묶은 대형 국가과제를 출범하고, ② **LG전자가 주관기관**으로 **20대의 휴머노이드를 산업·물류·돌봄 현장에 투입**해 작업 능력을 실증하며, ③ **부품·구동기·인지AI·VLA(Vision-Language-Action) 모델**까지 국산화 로드맵을 동시에 추진하는 것이다. 이는 NVIDIA GR00T, Figure 02, Tesla Optimus 등 글로벌 휴머노이드 경쟁에 대한 **국가 차원의 Physical AI 산업화 대응**으로, ① 물건 이송·환자 돌봄 등 **노동력 부족 분야의 자동화**, ② **반도체·자동차에 이은 차세대 한국 산업 기반**, ③ **AI 모델(LLM·VLA)+로보틱스+센서 융합 신기술**이 결합된 대표 사례다. 기술사 관점에서 K-휴머노이드는 **Physical AI / Embodied AI 시스템 아키텍처**, **VLA 멀티모달 모델**, **로봇 OS·디지털 트윈**, **AI 신뢰성·안전 인증** 등 신기술융합 영역의 최우선 출제 후보다.

## 2. 핵심 개념

### Physical AI / Embodied AI 정의

**Physical AI**는 디지털 공간의 LLM을 넘어 **물리적 신체(로봇·자율시스템)**를 가진 AI가 환경을 인식·추론·행동하는 패러다임이다. **Embodied AI**는 그중에서도 **신체(embodiment)를 통한 상호작용·학습**을 강조한다. K-휴머노이드는 ① **휴머노이드 폼팩터(이족보행·양팔 매니퓰레이터)**, ② **VLA 모델 기반 일반화 제어**, ③ **시뮬레이션-실세계 학습 루프(Sim-to-Real)**, ④ **현장 실증 데이터 수집**을 통합한 Embodied AI 시스템이다.

### VLA(Vision-Language-Action) 모델

**VLA**는 ① 카메라·LiDAR로 환경을 인식(**Vision**)하고, ② 자연어 명령을 해석(**Language**)하며, ③ 관절 토크·그리퍼 동작 등 행동(**Action**)을 생성하는 **멀티모달 파운데이션 모델**이다. 대표 사례는 Google RT-2, NVIDIA GR00T N1, OpenVLA 등이며, K-휴머노이드도 국산 VLA 모델 학습을 핵심 과제로 삼는다.

### K-휴머노이드 시스템 아키텍처

```
┌──────────────────────────────────────────────────────────────┐
│ ① 인지(Perception) 계층                                      │
│   - 멀티카메라(RGB-D) + LiDAR + IMU + 촉각 센서             │
│   - SLAM·6DoF Pose Estimation·3D Object Detection           │
├──────────────────────────────────────────────────────────────┤
│ ② 인지지능(Cognition) 계층 - VLA 모델                       │
│   - 멀티모달 LLM(시각+언어) → 행동 계획(Task Planning)      │
│   - Chain-of-Thought + Tool Use(로봇 액션 API)              │
├──────────────────────────────────────────────────────────────┤
│ ③ 행동(Action) 계층                                          │
│   - Whole-Body Controller(이족보행 균형, MPC/RL)            │
│   - Manipulation Policy(파지·이송, Diffusion Policy)         │
├──────────────────────────────────────────────────────────────┤
│ ④ 실행(Embodiment) 계층                                      │
│   - 액추에이터(BLDC·하모닉 드라이브) + 배터리 + 컴퓨트       │
│   - Onboard GPU(Jetson Thor) + 통신(5G·Wi-Fi 6E)            │
├──────────────────────────────────────────────────────────────┤
│ ⑤ 학습·운영(MLOps for Robotics) 계층                        │
│   - 시뮬레이션(Isaac Sim·MuJoCo) + 실세계 데이터 플라이휠   │
│   - Digital Twin 기반 안전 검증 + OTA 정책 업데이트         │
└──────────────────────────────────────────────────────────────┘
```

### 기존 산업용 로봇 vs 휴머노이드 비교

| 구분 | 산업용 로봇 | 협동로봇(Cobot) | 휴머노이드 |
|---|---|---|---|
| 폼팩터 | 고정형 매니퓰레이터 | 6/7축 팔 | 이족보행 + 양팔 |
| 제어 | 사전 프로그래밍 | 직관 티칭 | VLA 자연어 명령 |
| 작업 환경 | 격리된 셀 | 공유 작업장 | 인간 환경 그대로 |
| 학습 방식 | 결정적 코드 | 일부 학습 | 대규모 데이터·RL |
| 시장 | 자동차·전자 제조 | 중소 제조 | 물류·돌봄·서비스 |

## 3. 기술사 출제 포인트

- **Physical AI / Embodied AI 개념 정의와 등장 배경** (LLM의 디지털 한계 → 물리세계 확장)
- **VLA 모델 아키텍처와 멀티모달 LLM과의 관계** (Vision Encoder + Language Model + Action Head)
- **휴머노이드 시스템 5계층 아키텍처** (인지 → 인지지능 → 행동 → 실행 → 학습·운영)
- **Sim-to-Real 학습 전략** (Domain Randomization, Digital Twin, 데이터 플라이휠)
- **휴머노이드 안전·신뢰성 요구사항** (ISO 13482 서비스 로봇 안전, AI 신뢰성 인증)
- **국가 R&D 정책 관점의 도입 시 고려사항** (핵심 부품 국산화, 데이터 주권, 표준화, 인력)
- **산업용 로봇·협동로봇과의 차이점**과 융합 시 시너지

## 4. 관련 토픽 연계

- Physical AI의 기반이 되는 멀티모달 LLM은 [멀티모달 LLM과 Transformer MoE 응용 아키텍처](/tech-engineer-blog/posts/출제예상-139회-멀티모달-LLM과-Transformer-MoE-응용-아키텍처/)에서 다룬 Vision-Language 통합 모델과 직결된다.
- 휴머노이드의 인지지능 계층은 [Agentic AI 멀티에이전트 아키텍처와 MCP 보안 통합 설계](/tech-engineer-blog/posts/출제예상-139회-Agentic-AI-멀티에이전트-아키텍처와-MCP-보안-통합-설계/)에서 본 Tool Use·Task Planning 패턴을 물리세계로 확장한 형태다.
- 산업 현장 분산 에이전트 사례는 [KERI 자율제조 AI 멀티에이전트와 산업 분산 에이전트 시스템 아키텍처](/tech-engineer-blog/posts/KERI-자율제조-AI-멀티에이전트와-산업-분산-에이전트-시스템-아키텍처/)에서 다룬 산업형 자율 에이전트 구조와 연계된다.
