**[핵심/키워드]**

&lt;특징&gt; 
외부 환경 요인 분석, 거시환경분석, 전략 수립 도구 
PEST 분석: 4가지 요인(정치, 경제, 사회, 기술)
-&gt; Political(정치적), Economical(경제적), Social(사회적), Technological(기술적)
PESTEL 분석 :  PEST 분석을 확장한 모델로, 환경(Environmental)과 법률(Legal) &lt;/font&gt;요인을 추가
-&gt; Environmental(환경) : 탄소중립, 기후 리스크, 탄소세, ESG, RE-100, CF100, 전력소비/탄소 배출 규제

### [경영/컨설팅] ALE(Annual Loss Expectancy)

**[정의]**

특정 위험 사건이 발생할 경우 연간 예상되는 손실의 평균적인 금액을 계산하는데 사용하는 정량적 위험평가를 위한 지표

**[핵심/키워드]**

&lt;특징&gt; 
  - 위험 우선순위 결정
  - 위험 관리 전략 수립
  - 자원 할당
  - 모니터링 
&lt;계산 공식&gt; 
ALE = SLE×ARO(1회 예상 손실 금액 X 연간 발생 횟수)&lt;/font&gt;
 - SLE(Single Loss Expectancy): 한번 사고에서 발생하는 예상 손실 금액
  ㄴ SLE = AV × EF
     ㄴ AV(Asset Value): 자산 가치(금전적 가치)
     ㄴ EF(Exposure Factor): 위험 발생 시 자산 손상 비율(0~1)
 - ARO(Annualized Rate of Occurrence): 위험이 연간 발생 예상 횟수

* 시장분석과 전략수립을 위한 시장규모 추정 프레임워크

**[핵심/키워드]**

&lt;시장규모&gt; 전유수&lt;/font&gt;
* PAM &gt; TAM &gt; SAM &gt; SOM &gt; LAM
- PAM : Potential Available Market
- TAM : Total Addressable Market. 제품/서비스와 카테고리 영역이 포함되는 전체 비즈니스 도메인 크기
- SAM : Service Available Market. 전체 시장영역(TAM) 중에 스타트업이 추구하는 비즈니스 모델이 차지하는 유효시장
- SOM : Service Obtainable Market. 유효시장(SAM) 내에 초기단계에 확보가능한 거점/수익 시장규모
- LAM : Launch Addressble Market. 초기 거점 시장

1) Go-Back-N&lt;/font&gt; : 오류 데이터부터 다시 전송
2) Selective-Repeat&lt;/font&gt; : 에러가 발생한 프레임만 다시 전송
3) Hybrid ARQ&lt;/font&gt; : FEC 방식 + 재전송
- CC-HARQ&lt;/font&gt; : Chase Combining 기법으로 모든 재전송에 동일한 정보 포함
- IR-HARQ&lt;/font&gt; : Incremental Redundancy 기법으로 이전 정보와 다른 비트 세트를 이용하여 모든 재전송마다 수신자는 새로운 정보를 얻음

CC-HARQ:(같은 데이터를 여러 번 받아서 합쳐서 더 선명하게 보이게 만드는 방식)
[데이터] → 실패
[데이터] → 실패
[데이터] → 성공
(같은 것 반복, 합쳐서 신호 품질 개선)

IR-HARQ:(매번 새로운 오류정정 정보를 추가로 받아서 점점 더 정확해지는 방식)
[데이터 + 패리티1] → 실패
[패리티2]         → 실패
[패리티3]         → 성공
(다른 정보가 계속 추가됨)

### [테스트] 2-Value 경계값 분석 (2-Value Boundary Value Analysis, 2-BVA) 테스트

**[정의]**

중간값 테스트를 제외하고 절대적인 경계값만 확인하여 빠르게 검증을 수행하기 위해, 최소값과 최대값의 경계만을 테스트하는 경계값 분석기법

**[핵심/키워드]**

&lt;특징&gt;
- 최소한의 테스트 케이스로 경계값 검증 가능
- 중간 값 검증을 생략하여 테스트 효율성을 높임
- 빠른 테스트가 필요할 때 유용하지만, 극단적인 오류 탐지에는 한계가 있음
&lt;절차&gt; 입력값 범위 분석 → 최소·최대값 선정 → 테스트 실행 → 결과 검증
&lt;경계값&gt;
하한 경계 : min-1, min
상한 경계 : max, max+1

### [경영/컨설팅] Ansoff Matrix(안소프 매트릭스)

**[정의]**

- 기업이 성장을 달성하기 위해 제품(Products)과 시장(Markets)이라는 두 가지 요소를 기준으로 네 가지 유형의 성장 전략을 제시한 모형

**[핵심/키워드]**

&lt;특징&gt; 
- 성장 전략을 4가지 유형으로 체계화
- 위험 수준을 직관적으로 비교 가능
- 경영 전략·사업 포트폴리오 의사결정에 활용
&lt;안소프 매트릭스&gt;
① 시장 침투(Market Penetration) : 기존 제품 × 기존 시장
-&gt; 기존 시장에서 현재 점유율을 높이는 전략
② 제품 개발(Product Development) : 신제품 × 기존 시장
-&gt; 기존 고객층 대상으로 신제품 개발하거나 기존 제품 개선
③ 시장 개발(Market Development) : 기존 제품 × 신시장
-&gt; 기존 제품을 신규 시장에 진출
④ 다각화(Diversification) : 신제품 × 신시장
-&gt; 새로운 제품을 신규 시장에 진출(고위험·고수익 전략)

- 가로축(제품): 기존 → 신제품으로 갈수록 혁신 리스크 증가
- 세로축(시장): 기존 → 신시장으로 갈수록 시장 리스크 증가
- 즉, 우상단(다각화)로 갈수록 위험은 커지지만 성장 잠재력도 극대화되는 구조

- Multicast, Unicast, Anycast

- 게이트웨이 관점
- 주소변환기를 사용하여 상호 연동

### [테스트] 콘콜릭 실행 / 테스트 (Concolic Execution / Testing)

**[정의]**

Concrete(구체적 실행)+Symbolic(기호적 실행) 기법을 결합하여 테스트 하는 방법
- 실제 입력 값으로 실행하면서 기호 실행을 병행하여 새로운 테스트 케이스를 자동 생성
* 구조기반. 화이트박스 테스트

**[핵심/키워드]**

&lt;특징&gt;
- Concrete Execution의 실제 테스트 환경 반영 + Symbolic Execution의 자동 경로 탐색 기능 결합
- 자동 경로 탐색 가능 → 기존 Concrete Testing보다 높은 커버리지 제공
- 경로 폭발 완화 → 필요할 때만 Symbolic Execution 사용
- SMT Solver 활용 → 효율적인 테스트 케이스 생성

[ 구성요소 ]
- Concrete State : 실제 입력값 저장 및 사용
- Symbolic State : 변수를 기호로 대체
- Path Condition : 조건문 만나면 논리식으로 변환
- Test Case
- SMT Solver : 제약조건 해결
- Execution Engine

[ 절차 ]
초기 Concrete Input으로 테스트 실행 > 실행 중 경로 조건 수집 > Symbolic Excution 적용 > 새로운 테스트 입력 > 반복 실행 및 커버리지 확장 > 버그 탐지 및 검증

### [경영/컨설팅] 인포노믹스(Infonomics)의 데이터 가치평가모델

**[정의]**

- 데이터의 가치를 평가하기 위해 품질 관리 체계 측면의 가치와 재무적 측면의 가치를 함께 고려하여 6가지 항목을 통해 데이터 가치를 평가하는 가트너 사에서 개발한 데이터 자산 가치 평가 모델 

- 데이터를 경제적 자산으로 간주하고, 이를 평가, 관리, 활용하는 방식을 연구하는 학문적, 실무적 접근법

&lt;6가지 평가지표&gt;
* 기본 지표 (Fundamental)
1) 내재적 가치(IVI) : 데이터의 정확성 및 완전성, 배타성 기준에 따른 평가 지표
2) 비즈니스 가치(BVI) : 데이터와 사업 목표의 합목적성, 일치성 기준에 따른 평가
3) 성과 가치(PVI) : 데이터가 핵심 사업 모델 확장 전략에 영향을 미치는 정도
* 재무 지표 (Financial)	
4) 비용 가치(CVI) : 데이터를 일부 유실·손실되었을 경우 소모되는 비용 수준
5) 시장 가치(MVI) : 데이터를 시장에 판매·거래하여 얻을 수 있는 획득 가치
6) 경제적 가치(EVI) : 데이터를 통해 창출 가능한 수익과 손실 기여 정도

- GRE(Generic Route Encapsulation) : 레이어 프로토콜 범위 캡슐화. VPN 구성을 위해 PPTP와 함께 사용

### [네트워크] NaaS(Network as a Service)

**[정의]**

**[핵심/키워드]**

&lt;특징&gt;
- 대역폭 지능적, 유연성, 확장 가능성. 사용한 만큼 비용 지불 

[ 주요기술 ]
- SDN, NFV, Cloud Infrastructure, Orchestration, API

&lt;NaaS에서 제공하는 주요 서비스&gt;
- 가상 네트워크 (Virtual Network) : VLAN, VPN, SD-WAN 등 가상 네트워크 인프라를 제공.
- 네트워크 보안 (Network Security) : 방화벽, DDoS 방지, 암호화 등 네트워크 보안 기능.
- 대역폭 관리 (Bandwidth Management) : 네트워크 대역폭 할당 및 조정을 통해 품질 보장(QoS).
- 트래픽 라우팅 (Traffic Routing) : 네트워크 트래픽을 최적 경로로 전송하여 지연 시간 감소.
- 인터커넥션 (Interconnection) : 다양한 클라우드 서비스 또는 데이터 센터 간의 연결.
- 모니터링 및 분석 (Monitoring and Analytics) : 네트워크 성능, 장애, 보안 위협을 실시간으로 감지 및 분석. 
- 원격 N/W 접근 : 원격 재택근무 등 업무 시 안전하게 네트워크 접근, 원격 액세스 제공
- 웹 보안 게이트웨이 : 웹 인터페이스에서 게이트웨이 보안 설정 등 관리 기능 제공
- ADC(Application Delivery Controller) : L4~L7 계층의 리다이렉션, 로드밸런싱, Fail-Over 등 지원(Switch), 서버 오프로딩, TCP 연결 다중화, 데이터 압축, NAT 등 기능 제공
- MANO : 가상화 인프라/VNF 관리, NFV 오케스트레이션
- 멀티클라우드 N/W 접근·제어 : 다양한 클라우드 내 정의된 네트워크에 대해 접근 및 제어 관리 수행

&lt;서비스 예시&gt; 
- 하이브리드 클라우드 및 멀티 클라우드 환경 : 여러 클라우드 서비스와의 연결을 통합적으로 관리.
- 원격 근무 지원 : VPN 및 SD-WAN을 통해 직원들이 안전하게 원격으로 접속 가능

수입경로 → 자동화 Pipeline → 수입연결

■ RevOps 수입경로 : 마케팅, 세일즈, 고객채널, 재무
■ RevOps 자동화를 통한 수익 관리 강화

5단계 - 기술 인프라 구축 : CRM, 분석 도구, 자동화 플랫폼 등 통합
6단계 - KPI 설정 : 목표와 전략에 적합한 메트릭 선정
7단계 - 결과 모니터링 : 데이터 관리 모범 사례 문서화 및 개선 사항 파악

&lt;SMART 프레임워크&gt;
Specific : 구체성 - “매출을 늘린다” → “6개월 내 매출 20% 증가”
Measurable : 측정 가능성 - “매출 증가” → “월 매출 20% 증가”
Achievable : 달성 가능성 - 팀 규모와 예산을 고려하여 목표 설정
Relevant : 관련성 - 매출 증가는 회사 성장 전략과 연계되어야 함
Time-bound : 시간 제한 - “6개월 내 목표 달성”

### [테스트] 뮤테이션 테스트 (Mutation Test)

**[정의]**

(= 돌연변이 테스트)
* 의도된 결함, 테스트 데이터 효과성

**[핵심/키워드]**

[ 방식 ]

테스트 결과에 대한 신뢰감

뮤턴트 스코어&lt;/font&gt;로 측정
Mutant score = (Dead mutant(발견) / Total mutant(전체)) X 100
-&gt; Mutantscore 100 : 모든 돌연변이를 테스트 케이스가 검출
-&gt; 살아남은(mutant survived) 돌연변이는 테스트가 부족함을 의미

원본 코드 &gt; 돌연변이 코드 &gt; 테스트 케이스 실행 &gt; 돌연변이 분석 &gt; 뮤테이션 점수 계산 &gt; 테스트 케이스 개선

### [경영/컨설팅] 프로토타이핑(Prototyping)

**[정의]**

제품, 서비스의 초기 버전을 만들어 보고, 시험하고, 수정하는 과정에서 이해관계자 들과의 의사소통을 향상시키고, 요구 사항을 명확하게 이해하고 더 나은 디자인과 기능을 개발하기 위한 방법

**[핵심/키워드]**

&lt;절차 및 산출물&gt;
1) 계획수립: 시스템정의서, 프로젝트 계획서
2) 요구분석: 요구분석서, 요구사항정의서
3) 프로토타입 식별
4) Prototype 설계/개발 : 프로토타입 설계서, 스토리보드
5) 평가: 프로토타입 평가서
6) 구현: 테스트 계획서, 테스트 결과서, 실행파일

&lt;유형&gt;
1️⃣ 재사용 기준 : Experimental Prototyping(실험적), Evolutionary Prototyping(지속적)
2️⃣ 매체 기준 : Paper Prototyping(종이에 그림), Digital Prototyping(도구나 소프트웨어 이용)
3️⃣ 최종결과물 유사성 기준 : Low-fidelity Prototyping(간단, 낮은 Quality의 인터페이스),
Mid-fidelity Prototyping(대략적인 인터랙션 디자인),High-fidelity Prototyping(완성도가 높도록 제작)
4️⃣ 기능 범위 및 정도 기준 : Horizontal Prototyping(전반적으로 예상 가능), Vertical Prototyping(세부적인 부분까지 디테일하게 구현), Scenario Prototyping(지정된 기능과 경로를 완전히 실행)

&lt;장점&gt;커뮤니케이션 오류 감소
&lt;단점&gt;개발 측에서 모델 제시, 불필요한 비용 발생

### [네트워크] DNSSEC(Domain Name System Security Extensions)

**[정의]**

DNS 응답에 디지털 서명을 추가하여, 응답이 변조되지 않았음을 검증할 수 있도록 확장한 DNS 보안 기술

**[핵심/키워드]**

&lt;특징&gt;
- 무결성 보장(변조 방지), 출처 인증(서버 확인), 전자서명 기반(공개키 구조),
- 신뢰 체인 구조(루트 → TLD(Top-Level Domain) → 도메인까지 공개키를 체인으로 검증)
- 공개키 기반 구조를 이용하여 DNS 응답의 무결성과 진위를 검증. 전자서명 기반 검증, RRSIG 검증으로 위·변조 방지

&lt;구성요소&gt;
- ZSK(Zone Signing Key):특정 도메인 영역(zone)의 레코드에 서명하는 키
- KSK(Key Signing Key) : ZSK의 공개키에 서명하는 상위 키, 상위 DNS 서버로부터 검증됨
- RRSIG(Resource Record Signature) : 리소스 레코드에 대한 디지털 서명 데이터
- DNSKEY : 공개키를 저장하는 레코드
- DS(Delegation Signer)        : 자식 도메인의 DNSKEY를 상위 서버에 등록하기 위한 연결 정보
- NSEC / NSEC3 : 존재하지 않는 도메인임을 증명하는 부재 증명 레코드

* 공격유형
- DNS 캐시 포이즈닝 공격
- 패킷 가로채기 공격
- DNS 반사(Reflection)
- DNS 리소스 소진

### [테스트] 비버깅(Bebugging)

**[정의]**

의도적으로 오류코드를 삽입하여 잔존 오류 및 테스트의 효율성을 측정하는 방법

**[핵심/키워드]**

[ 방식 ] 
의도적 오류코드 삽입

[ 목적 ] 
잔존 오류 추정

[ 사례 ] 
의도적 오류100개 삽입, 의도 오류10개, 미의도 오류 20개 발견
의도적 오류발견율 10%로 90%는 찾지못함
의도적 오류로 테스트 능력 측정
→ 발견율 10%
실제 발견한 오류 20개를
→ “전체의 10%”로 보고 전체 오류 추정
→ 20 ÷ 0.1 = 200개
이미 찾은 20개를 빼면
→ 아직 180개의 잔존 오류가 있다고 판단

### [테스트] 퍼즈 테스트(Fuzz Test)

**[정의]**

SW에 랜덤 데이터를 입력함으로써 S/W의 보안 취약점, 예외, 오류 등을 동적으로 분석하고 테스트하는 기법

**[핵심/키워드]**

[ 분류 ]
- 블랙박스 퍼징 : 내부 코드 없이 무작위 입력 제공. Crash, Hang, 예외 처리 검증 목적. 완전 자동화 가능
- 화이트박스 퍼징 : 소스코드 분석 기반으로 구조적 입력 생성. 로직 오류, 메모리 취약점 탐색 목적. 부분 자동화 가능

[ 절차 ]
대상 분석 > 입력 데이터 생성 > 환경 구축 > 테스트 실행 > 취약점 탐색 > 수정 및 재테스트 > 최종 보고서 작성

[ 활용 ] 
알려지지 않은 취약점 탐지, 버그/취약점 사전점검

[ 보안 취약점 점검 커버리지 ]
Direct Physical Attack, Indirect Physical Attack, Local / Remote Wireless Attack
