### [인공지능] 임베딩(Embedding)

**[정의]**

텍스트, 이미지, 음성 등의 객체 데이터를 연속 벡터 공간의 점으로 표현하는 수단

**[핵심/키워드]**

토큰화(Tokenization) , 벡터 변환(Vector Transformation),  문맥화(Contextualization)

### [인공지능] Word2Vec

**[정의]**

- 의미기반 워드 임베딩 기법
- 단어자체가 가지는 의미를 다차원 공간에서 벡터화하는 인공신경망 알고리즘 
- 텍스트 문서(말뭉치)를 통해 학습, 한 단어에 대한 유사도를 측정, 관련 단어로서 인공신경망을 학습시킴

**[핵심/키워드]**

비지도학습
범주형 > (원핫인코딩) > 연속형 > (word2vec) > 수치적 의미부여

→고차원 희소 데이터 문제를 해결하고 데이터의 잠재적 관계를 포착하는 효과적인 방법

### [인공지능] GRU(Gated Recurrent Unit)

**[정의]**

- LSTM과 유사하지만 출력 게이트를 생략한 형태이며, 항상 메모리에서 결과를 출력하는 알고리즘

**[핵심/키워드]**

입력(input)게이트와 망각(forget)게이트가 업데이트 게이트(Z)로 통합
출력게이트 없음, 메모리에 결과 출력
- 성능 면에서는 LSTM과 비슷하지만, 학습할 파라미터가 더 적은 것이 장점

### [인공지능] 프롬프트 인젝션(Prompt Injection)

**[정의]**

공격자가 악의적인 지시사항을 포함한 프롬프트를 입력하여 AI가 본래 의도된 지침을 
무시하고 변경된 동작을 수행하게 만드는 공격

**[핵심/키워드]**

직접 인젝션: 사용자가 직접 명령을 주입하는 것.
간접 인젝션: 외부 소스(웹, 메일)를 통해 명령이 유입되는 것.
프롬프트 리킹 (Prompt Leaking) : 기업이 숨겨둔 프롬프트 설계도를 훔쳐내는 공격

### [인공지능] 제로클릭 공격(Zero-Click Attack)

**[정의]**

사용자의 직접적인 상호작용 없이, 공격자가 원격으로 시스템 취약점을 이용하여  악성코드를 실행하거나 비인가 접근을 달성하는 공격 기법

**[핵심/키워드]**

공격 수단은 ‘가짜 GIF’
페가수스 공격의 배후로 알려진 NSO :  가짜 GIF로 코어그래픽스 PDF(CoreGraphics PDF) 파서의 취약점 겨냥

### [인공지능] 적대적 공격(Adversarial Attack)

**[정의]**

악의적인 목적으로 조작한 데이터를 활용하여 AI시스템이 잘못된 판단 또는 오동작을 
하도록 유도하는 공격

**[핵심/키워드]**

회피 공격 (Evasion Attacks), 중독 공격 (Poisoning Attacks), 모델 추출 공격 (Model stealing/extraction attacks)

### [인공지능] 인공지능 가드레일(AI Guardrail)

**[정의]**

AI가 잘못된 정보, 유해한 콘텐츠, 보안 위험요소를 출력하지 않도록 제한하는 안전 규칙

**[핵심/키워드]**

입력 필터링 , 출력 검증 , 맥락 제약 , 보안 가드레일

### [인공지능] 에이전틱 인공지능(Agentic AI)

**[정의]**

단순히 질문에 답하는 수준을 넘어, 주어진 목표를 달성하기 위해 스스로 계획을 세우고 도구를 사용하며 실행하는 AI

**[핵심/키워드]**

자율성(Autonomy) , 추론 및 계획(Reasoning & Planning) , 반복 및 수정(Iteration)

### [인공지능] 데이터 라벨링(Data Labeling)

**[정의]**

AI가 원천데이터를 학습에 활용하도록  사람이 직접 태그나 정답(Label)을 달아주는 과정

**[핵심/키워드]**

지도 학습의 핵심 
* 데이터 어노테이션: 라벨링보다 넓은 의미로 사용

### [인공지능] 컨텍스트 엔지니어링(Context Engineering)

**[정의]**

- LLM에 제공되는 다양한 정보 소스(정보 페이로드)를 체계적으로 설계하고 조합하여, AI의 추론 및 응답 품질을 극대화하는 통합 프레임워크
- 답변을 생성하는 데 필요한 맥락을 어떻게 구성할지를 다루는 기술

**[핵심/키워드]**

컨텍스트=정보설계+수집+처리+통합

&lt;프롬프트 엔지니어링 vs 컨텍스트 엔지니어링&gt;
- 프롬프트 엔지니어링: 질문 설계 → "어떻게 묻느냐"
- 컨텍스트 엔지니어링: 환경 설계 → "무엇을 알려주고 묻느냐"

&lt;5단계 수행 절차&gt;
[1]사용자 입력 전처리 → [2]컨텍스트 수집(정적정보, 동적정보, 대화이력) → [3]컨텍스트 처리·통합(요약, 필터링, 중요도 정렬) → [4]프롬프트 생성, LLM 호출 → [5]응답 후처리 결과 전달

### [인공지능] 하네스 엔지니어링(Harness Engineering)

**[정의]**

AI 에이전트(LLM)가 안전하고 안정적으로 작업을 수행하도록 감싸는, 스캐폴딩(구조)과 피드백 루프를 설계하는 기술

**[핵심/키워드]**

구성 요소: 파일 시스템 및 저장소, 코드 실행 샌드박스, 컨텍스트 관리, 서브 에이전트, 훅(Hook) 및 백프레셔(backpressure) 메커니즘.
핵심 목적: AI 에이전트가 단독으로 처리하기 힘든 복잡한 작업을 실패 없이 완수하도록 통제하고 보호하는 것.
발전 흐름: 프롬프트 엔지니어링 
 컨텍스트 엔지니어링 
 하네스 엔지니어링 (AI 에이전트 중심의 개발 환경으로 전환)

### [인공지능] MCP(Model Context Protocol,모델컨텍스트 프로토콜)

**[정의]**

- AI 모델이 외부 애플리케이션, 도구, 데이터 소스 등과 안전하고 유연하게 상호작용할 수 있도록 표준화된 프로토콜
- LLM의 입력/출력 컨텍스트를 표준화하고 구성요소 간 상호작용을 정의

**[핵심/키워드]**

LLM↔외부도구/데이터소스 표준 연동 프로토콜

&lt;특징&gt;
- LLM 성능 향상 기법: MCP를 통해 외부 도구·데이터 소스와 연동
- 표준화된 통신: AI 모델과 외부 시스템 간 일관된 인터페이스 제공

&lt;보안 취약점&gt;
- 인증·채널: 인증 미흡, 약한 암호화, MITM, 세션 하이재킹
- 공급망: 악성 MCP 서버 배포·변조, 검증되지 않은 서버 사용
- 권한 과다: MCP 서버·툴이 과도한 파일/네트워크/프로세스 권한 보유
- 프롬프트/툴 인젝션: 불신 데이터가 모델을 속여 위험 툴 호출(Tool-Poison-Attack)
- 자원 남용(DoS): 과다 호출·무한 루프로 서비스 지연·비용 증가

### [인공지능] AI 영향평가 가이드라인

**[정의]**

인공지능사업자가 고영향 인공지능을 이용한 제품 또는 서비스를 제공하는 경우, 사전에 해당 제품 또는 
서비스가 사람의 기본권에 미치는 영향을 평가하는 절차

**[핵심/키워드]**

* 평가 시기
- 신규 인공지능 제품 또는 서비스 제공 전  
* 국내 유사 평가제도와의 관계
 개인정보보호위원회, “개인정보 영향평가” (｢개인정보 보호법｣ 제33조)
: - 평가 항목 일부 중복 가능성 존재, 양 제도 간 적용 범위 (평가주체)가 상이(민간사업자 vs. 공공기관)하여 사업자 대상 직접적인 중복규제 논란은 없을 것으로 판단

### [인공지능] Vanishing Gradient(베니싱 그래디언트), 기울기 소멸문제

**[정의]**

- 역전파 알고리즘을 학습을 통해 가중치를 수정할 때, 은닉층으로 오차가 거의 전달되지 않는 문제 
- 편미분을 통한 기울기를 측정할 수 없는 문제

**[핵심/키워드]**

* 발생원인
- Data set 부족 - 컴퓨터 파워 부족- 초기 가중치 값 오류- 활성함수 선택 오류
* 해결방안
- Drop out, GPU 활용, ReLU 활성화 함수 적용, 사전학습, 데이터 스케일링

### [인공지능] Hopfield(홉필드) 신경망

**[정의]**

- 지역 최소점 문제 해결
- 연상 기억이나 최적화 문제를 병렬적으로 푸는 망 
- 라우팅 프로토콜
- 모든 뉴런이 양방향으로 연결

**[핵심/키워드]**

연상 기억에 있어서는 일정한 범용 패턴들을 연결 강도로 저장했다가 미지의 입력 패턴이 주어질 때 이와 가장 유사한 패턴을 찾음

### [인공지능] 역전파 알고리즘(Back Propagation)

**[정의]**

- 가중치들 각각에 대해 별도로 경사하강을 관찰하고 누적오차 감소에 가장 큰 기여를 하는 가중치 한 개 값을 이동시키는 방법을 반복하여 해를 찾아가는 기법
- 지역 최소점 문제 발생

**[핵심/키워드]**

출력층의 오차를 역전파하여 은닉층을 학습

* Chain Rule : 역전파과정에서 오차를 편미분하여 각 가중치의 기여도를 도출하여 최적의 기울기 얻기위한 수학적 원리

### [인공지능] 다층 퍼셉트론(MLP, Multilayer Perceptron)

**[정의]**

- 퍼셉트론의 XOR의 한계를 개선한 여러 개의 퍼셉트론을 층 구조로 구성한 신경망 모델
- 경사하강법(Gradient Descent)과 오차역전파법(Backpropagation) 원리가 사용

**[핵심/키워드]**

Hidden Layer의 역할: 앞 단계에서 받은 데이터를 필터링해서 좀 더 구체화 한 후 다음 단계층으로 전달

### [인공지능] HMM(Hidden Markov Model)

**[정의]**

- 시스템이 은닉된 상태와 관찰 가능한 결과의 두가지 요소로 이루어졌다고 보는 통계 기반의 모델 
* 상태가 숨겨져 있어 확률로 계산
* 상태(은닉변수)를 직접 볼 수 없고 상태들로부터 야기된 결과(관측변수)만을 관찰

**[핵심/키워드]**

- 은닉변수: 관측 가능한 상태
- 은닉변수 상태공간: 은닉변수의 이산적인 집합
- 상태 전이 확률: 상태가 변화를 가지는 확률
- 출력 확률: 관측변수들의 분포를 제어하는 집합
- 출력확률 분포도: 마르코프 체인에 기반한 상태들의 확률 분포

### [인공지능] AI 리터러시(Literacy)

**[정의]**

- EM알고리즘을 사용하여 각 가우시안 분포의 파라미터(평균, 공분산, 가중치)를 최적화

**[핵심/키워드]**

- 최적화 모수 집합 (θ): 혼합 계수 (πₖ), 평균 벡터 (μₖ), 공분산 행렬 (Σₖ)
- 모수 최적화 과정: EM 알고리즘  

&lt;최적화 과정&gt; 
 - 초기 데이터 분포 분석 -→ K개의 새로운 가우시안 분포로 군집화 -→ 
EM 알고리즘을 이용해 최적의 분포(군집) -→ K개의 새로운 가우시안 분포로
데이터 모델링

### [인공지능] EM(Expectation Maximization)

**[정의]**

- 확률 분포(군집)의 매개변수(평균, 분산, 가중치)를 최적화하여 최적의 군집을 찾게 하는 방법
  - 확률적 모델을 기반으로 군집을 형성하는 기법
  - 관측되지 않는 잠재변수에 의존하는 확률모델에서 최대가능도(maximum like-hood)나 최대사후확률(maximum a posteriori)을 갖는 매개변수를 찾는 반복적 알고리즘 (군집 알고리즘에 활용 가능)

**[핵심/키워드]**

1) 초기화: K개 군집개수 설정 
2) Expectation Step(최대가능도): 클러스터 모델 재생성, 평균/분산값 재계산 
3) Maximization Step(최대사후확률): 데이터간 확률값 재계산 
4) 수렴 확인(조건검사 및 종료): Log Likehood 계산, 포화시 종료 또는 E-step 이동

* 군집이 적절한 데이터분포를 찾음
- 초기 > E-Step > M-Step > 수렴? > 종료
* 장점 
- 통계적 확률계산을 통한 합리적 접근, 다양한 형태 데이터 적용가능
* 단점 
- 연산 복잡도 높음, 처리시간/처리필요공간 상대적 저효율

### [DB] Locking

**[정의]**

- 트랜잭션이 데이터베이스 객체(예: 레코드, 테이블, 페이지 등)에 동시에 접근하는 것을 제어하기 위해 Lock(잠금)을
설정하여, 데이터의 일관성과 무결성을 보장하는 동시성 제어 기법

**[핵심/키워드]**

- 공유Lock(Shared lock, S-lock) - 트랜잭션 읽기만 가능
- 베타Lock(Exclusive lock, E-lock) - 해당 트랜잭션만 읽고/쓰기 가능. 타 트랜잭션 읽기불가
<락 양립성(LockCompatibility)>
- 두 개 이상의 트랜잭션이 같은 데이터 항목에 동시에 락을 요청할 때,그 락들이 서로 충돌 없이 공존할 수 있는지 여부를 판단하는 기준
- 공유(S)-배타(X) 또는 배타(X)-공유(S): 불가능
→읽는 중에 누군가 쓰려 하거나,쓰는 중에 누군가 읽으려 하면 충돌
- 배타(X)-배타(X): 불가능

### [DB] 2PL(2Phase Locking)

**[정의]**

- 트랜잭션 내에서 모든 락을 설정하는 확장(확보) 단계와, 한 번이라도 락을 해제한 이후에는 더 이상 락을 설정할 수 없는 축소(해제) 단계로 구분하여 직렬 가능성을 보장하는 동시성 제어 기법

**[핵심/키워드]**

- 단계: 확장단계, 차단단계, 축소단계 
- 기법: Strict 2PL, Rigorous 2PL, Static 2PL
→Strict2PL(Rollback안전성 강화) : 쓰기 락만 종료시까지 해제 금지
→Rigorous2PL(격리 수준 강화) : 읽기/쓰기 락 모두 종료시까지 해제 금지
→Static2PL(Deadlock예방 목적) : 필요한 모든 락을 미리 확보

### [DB] 낙관적검증

**[정의]**

- 트랜잭션 간 충돌이 드물다는 가정 하에,락 없이 실행한 후 종료 직전에 검증(Validation)을 통해 충돌 여부를 확인하고
문제가 없을 경우 커밋하고,충돌이 있으면 롤백하는 동시성 제어 기법
- 3가지 조건 중 하나만 만족하면 트랜잭션 간섭없음 판정

**[핵심/키워드]**

판독단계(R: Read Phase) 
확인단계(V: Validation Phase) : 데이터베이스에 반영 전 직렬 가능성의 위반여부 확인
기록단계(W: Write Phase)
- 타임스탬프: Start(Ti), Validation(Ti), Finish(Ti)

마지막에 들어간 데이터가 가장 먼저 나오는 후입선출(Last In First Out,LIFO)방식의 선형 자료구조

- 재귀처리 적합(함수 호출), 간단 구현(배열 기반), 직접 접근(O(1) 인덱스로 즉시 접근

* 구성요소
- Stack 저장공간, Top 포인터, Size/Capacity

* 연산

**[정의]**

**[핵심/키워드]**

* 특징
- 순서보장(입력 순서 유지), 인덱스 접근가능(배열 기반),
동적 확장(유연한 크기), 다양한 구현(배열/연결 리스트)

* 종류

- 연결 리스트 : 노드 단위로 연결된 구조, 포인터로 연결. 삽입/삭제에 유리함. 접근은 느림

### [알고리즘] 큐(Queue)

**[정의]**

데이터를 먼저 집어넣은 순서대로 꺼내는 FIFO(First In, First Out) 방식의 선형자료 구조

**[핵심/키워드]**

* 특징
- FIFO방식, 양방향 처리(한쪽 삽입, 반대쪽 삭제),
순차적 처리(입력 순서 유지)
- 배열, 연결 리스트, 덱(이중큐) 중 하나로 큐 구현

* 구성요소
- Front 포인터, Rear 포인터, 저장공간, Size

* 연산
- Enqueue, Dequeue, Peek/Front, isEmpt/isFull

* 종류
- 선형 큐(LinearQueue) : 배열 기반 선형구조. 공간 낭비
- 원형 큐(CircularQueue) : 배열 끝을 처음과 연결한 순환 구조. 선형 큐 공간 낭비 해결
- 우선순위 큐(PriorityQueue) : 우선순위 정렬 구조. 정렬 비용 발생 
- 이중 큐(Deque,덱) : 양쪽 끝 삽입/삭제 모두 가능
- 원형 이중 큐(CircularDeque) : 원형 + 양방향 입출
- 연결 큐(LinkedQueue) : 연결리스트 기반 큐. FIFO(선입선출) 방식

&lt; 시간복잡도&gt;

*  우선순위 큐 O (E log V) or O (E + V log V)

&lt; 절차 &gt;
* 출발/종료 노드 설정 > 초기화 > 미방문 노드 중 가장 짧은 노드 방문 > 최단거리 테이블 업데이트 > 방문-업데이트 반복

&lt; 동작 &gt;
* 간선 정렬 > 간선 선택 (비용 작은 순) > 사이클 확인 > 간선 추가 및 무시 > 반복

### [알고리즘] 연결 리스트 (Linked List)

**[정의]**

데이터를 저장하는 노드들이 포인터를 통해 순차적으로 연결된 선형 자료구조

**[핵심/키워드]**

* 동적 확장(O(1)) 
* 삽입/삭제 유리(O(1))
* 접근 느림(O(n))
* 포인터 필요

### [알고리즘] 분할정복알고리즘(Divide and Conquer)

**[정의]**

큰 문제를 작은 부분 문제로 나눈 뒤,각 부분을 독립적으로 해결하고,그 결과를 합쳐서 전체 문제를 해결하는 기법

**[핵심/키워드]**

① 분할(Divide)
② 정복(Conquer)
③ 결합(Combine)

### [알고리즘] KMP(Knuth-Morris-Pratt) 탐색

**[정의]**

패턴의 접두사-접미사 정보를 미리 계산한 실패함수(π배열)을 이용해,불일치 시 비교를 되돌리지 않고 점프함으로써 선형 시간에 문자열을 탐색하는 알고리즘

**[핵심/키워드]**

* 접두사접미사 구조 활용
* 텍스트 인덱스 되돌림 없음
* 중복 비교 제거
* 패턴 전처리 기반

### [알고리즘] 백트래킹(Backtracking)

**[정의]**

가능한 모든 경우를 탐색하면서 조건에 맞지 않으면 즉시 되돌아가는(가지치기) 방식의 완전 탐색 알고리즘

**[핵심/키워드]**

N-Queen 문제
스도쿠(Sudoku) 풀이

### [알고리즘] 이진 트리(BinaryTree)

**[정의]**

각 노드가 최대 두 개의 자식 노드 (왼쪽, 오른쪽)를 가지는 트리구조

**[핵심/키워드]**

&lt; 특징 &gt;
* 최대 자식 2개, 순서 있느 계층, 순회 방식 다양 (전위, 중위, 후위), 깊이 중요, 재귀적 구조
&lt; 시간 복잡도 &gt;
* 균형 O (log n), 불균형 O (n)
&lt; 유형 &gt;
* 포화이진트리, 완전이진트리, 정이진트리, 편향이진트리, 균형이진트리, 이진탐색트리, 힙트리
&lt; 순회 &gt;
* 전위 (루트, 왼쪽, 오른쪽), 중위 (왼쪽, 루트, 오른쪽), 후위 (왼쪽, 오른쪽, 루트), 레벨 (위에서 아래, 왼쪽에서 오른쪽)
&lt; 연관 &gt;
* 이진탐색트리, 힙트리

### [알고리즘] B-Tree

**[정의]**

데이터를 정렬된 상태로 저장하면서, 검색, 삽입, 삭제를 모두 빠르게 처리할 수 있도록 설계된 트리구조

**[핵심/키워드]**

&lt; 특징 &gt;
* M원트리, 다중 분기, 균형 유지, 정렬 유지, 빠른 탐색, 효율적 디스크 접근
&lt; 구성 &gt;
* 데이터 구성 (키, 데이터 포인터, 데이터)
* 트리 (트리 포인터, 루트 노드, 내부 노드, 리프 노드)
* 조건 (자식 수 제한, 키 수 제한, 루트 노드 조건, 트리 균형 조건)
&lt; 연산 &gt;
* 분할, 차용 / 재분배, 병합
&lt; 활용 &gt;
* DB 인덱스

### [알고리즘] 솔린(Soline) 알고리즘

**[정의]**

각 단계에서 모든 정점을 연결하는 포리스트를 만들고 이를 연결하는 최소 신장 트리

**[핵심/키워드]**

1) 간선 중심 : 크루스칼 방식 
2) 노드 중심 선택: 프림방식
3) 솔린 : 컴포넌트 중심

### [알고리즘] 퀵정렬(Quick Sort)

**[정의]**

기준 값(피봇) 을 중심으로 작은 요소는 왼쪽, 큰 요소는 오른쪽으로 분할(partition) 하여
재귀적으로 정렬을 수행하는 알고리즘

**[핵심/키워드]**

* pivot, 분할과 정복
 * 시간복잡도 : O(n^2)
 * 공간복잡도 : O(log n)
 * pivot : 첫번째, 중간, 랜덤 선택
