## 뉴스 요약

2026년 4월 13일, **한전KDN**은 'N2SF 기반 차세대 사이버보안 체계와 안전한 AI 국가 전략' 워크숍을 개최하고, 국가 에너지 인프라에 **N2SF(National Network Security Framework, 국가 망보안 프레임워크)** 와 **제로트러스트(ZTA)** 를 접목하는 방향을 공유했다(부산일보, 베타뉴스, 소비자경제, 이로운넷, IT조선 등 다수 매체 보도). 같은 날 전자신문은 "최악 해킹 사고 1년…AX 시대 '제로트러스트' 보안 패러다임 전환 서둘러야"라는 분석을 통해, 국가기반시설에 대한 망분리 중심 구조의 한계와 제로트러스트로의 전환 필요성을 지적했다. 이는 AI·클라우드 도입 가속화 속에서 **에너지·전력 OT(운영기술) 환경을 보호**하기 위한 정책 흐름의 전환을 보여준다.

## 핵심 개념

### N2SF 프레임워크 계층 구조

N2SF는 국가정보원이 2025년 공식화한 **국가 망보안 프레임워크**로, 기존 획일적 망분리 체계를 데이터 중요도 기반 3계층(C/S/O)으로 재설계한다.

| 계층 | 데이터 구분 | 연결 원칙 | 주요 통제 |
|---|---|---|---|
| C(Classified) | 비밀자료·국가기밀 | 인터넷 완전 차단, 폐쇄망 | 물리적 분리, 출입통제, 암호화 저장 |
| S(Sensitive) | 민감정보·업무자료 | 제한적 연결, 보안 게이트웨이 경유 | 제로트러스트 인증, DLP, CDR |
| O(Open) | 공개자료·업무지원 | 인터넷 직접 연결 허용 | WAF, SASE, EDR, 로그 감사 |

### IT-OT 융합 환경의 보안 위협

에너지 인프라는 IT와 OT가 융합되면서 공격 표면이 확장된다.

| 영역 | IT 환경 | OT 환경 |
|---|---|---|
| 목적 | 정보 처리·분석 | 물리 프로세스 제어 |
| 우선순위 | CIA(기밀성 > 무결성 > 가용성) | AIC(가용성 > 무결성 > 기밀성) |
| 장비 수명 | 3-5년 | 15-25년 (레거시 PLC·HMI) |
| 프로토콜 | HTTP, TLS | Modbus, DNP3, IEC 61850 |
| 패치 주기 | 상시 | 연 1-2회 (가동 중지 최소화) |
| 주요 위협 | 랜섬웨어, 데이터 유출 | SCADA 조작, 물리 피해, 정전 |

### 에너지 OT 제로트러스트 적용 아키텍처

```
[현장 계층 - Level 0/1]
  센서/액추에이터 ─── PLC/RTU
         ↓ (IEC 62443 Zone 분리)
[제어 계층 - Level 2/3]
  SCADA/HMI ─── 엔지니어링 워크스테이션
         ↓ (Unidirectional Gateway, Data Diode)
[DMZ 계층 - Level 3.5]
  보안 중계 서버 ─── Historian ─── 원격 접속 프록시
         ↓ (제로트러스트 PEP: 정책 집행 지점)
[IT 계층 - Level 4/5]
  ERP/MES ─── AI 분석 플랫폼 ─── 경영 시스템
         ↓
[거버넌스 계층]
  PDP(정책결정) ─── ID/디바이스 신원 ─── 동적 위험 평가
```

### 제로트러스트 7대 원칙(NIST SP 800-207)과 OT 적용

| 원칙 | IT 환경 구현 | OT 환경 적용 방안 |
|---|---|---|
| 모든 데이터·서비스 자원화 | API Gateway, 마이크로세그먼트 | PLC·HMI를 자원 단위로 등록·인벤토리 |
| 모든 통신 암호화 | TLS 1.3, mTLS | 레거시 프로토콜에 보안 게이트웨이·터널링 적용 |
| 세션별 접근 권한 | JWT, SAML, 동적 토큰 | JIT(Just-in-Time) 권한, 엔지니어 작업 시간 제한 |
| 동적 정책 기반 인증·인가 | ABAC, Context 기반 | 근무 시간·위치·작업 계획과 연계 |
| 자산 무결성·보안 상태 모니터링 | EDR, XDR | OT 이상 징후 탐지(아노말리 검출), 펌웨어 무결성 |
| 동적 인증·권한 갱신 | MFA, Continuous Auth | 엔지니어링 세션 실시간 재검증 |
| 지속적 정보 수집·개선 | SIEM, SOAR | OT SOC, 사이버 위협 인텔리전스 연계 |

### IEC 62443 vs N2SF 통합 적용

| 표준/체계 | 범위 | 핵심 통제 | 에너지 OT 적용 |
|---|---|---|---|
| IEC 62443 | 산업 자동화 제어시스템(IACS) | Zone/Conduit 분리, SL(Security Level) 등급 | 발전소·변전소 구역별 SL 설정 |
| N2SF | 국가 공공망 전반 | C/S/O 계층, 데이터 중요도 기반 | 전력거래소·계통운영 데이터 C계층 |
| NIST CSF 2.0 | 범용 위험 관리 | Govern·Identify·Protect·Detect·Respond·Recover | 에너지 사이버 거버넌스 수립 |
| 제로트러스트 | 접근 통제 모델 | 기본 불신, 지속 검증 | 원격 유지보수·벤더 접근 통제 |

## 기술사 출제 포인트

1. **N2SF 3계층(C/S/O) 체계 설계 원칙** — 기존 망분리(물리/논리)와 N2SF의 차이, 데이터 중요도 기반 연결성 정책, C계층의 폐쇄망 기본 원칙과 예외 기준을 서술할 것.
2. **IT/OT 보안의 우선순위 차이** — CIA vs AIC(가용성 우선), 레거시 프로토콜(Modbus/DNP3) 취약성, 패치 불가 환경에서의 보상 통제(Compensating Control) 설계.
3. **제로트러스트 NIST SP 800-207 7대 원칙** — PEP/PDP 구조, ABAC 기반 동적 정책, OT 환경에서의 적용 한계와 보완 방안(브로커리지, Unidirectional Gateway).
4. **IEC 62443 Zone/Conduit 모델** — 산업 제어 시스템 보안 구역화, SL(Security Level) 1-4 등급 산정, Purdue 참조 모델과의 연계.
5. **에너지 인프라 침해사고 시나리오** — Stuxnet, Ukraine 전력망 공격(Industroyer), Colonial Pipeline 사례 분석과 대응 통제, RE(Resilience Engineering) 설계 원칙.
6. **AI 보안 분석 수요와 OT SOC** — 오탐 감소·탐지 시간 단축을 위한 UEBA, AI 기반 침해 예측(AISPM), 공급망 공격 대응.

## 관련 토픽 연계

- **SASE·SSE 아키텍처**: 제로트러스트를 클라우드·원격 접속에 확장하는 통합 보안 모델.
- **CRA(Cyber Resilience Act)**: EU의 OT·IoT 제품 보안 의무화 규정과 공급망 보안.
- **공급망 보안(SBOM)**: 에너지 OT 소프트웨어 자재명세서 관리와 취약점 추적.
- **AI 기반 침해 대응(AI-Driven SOC)**: 정부의 AI 사이버 침해대응 시스템 전환과 연계.
- **디지털 트윈 보안**: 에너지 디지털 트윈 환경에서의 데이터 무결성·조작 방지 설계.
