## 1. 뉴스 요약

뉴시스·해사신문·부산일보·네이트 등 다수 매체가 동시 보도한 바에 따르면, **부산항만공사(BPA)가 HD현대삼호와 「AI 기반 항만하역장비 고도화」 공동연구·사업화 협력**에 본격 착수한다. 컨테이너 크레인·야드 트랙터·AGV 등 대형 하역장비에 ① **센서·CCTV·진동·전류 데이터를 상시 수집**, ② **엣지·클라우드 양단의 AI 모델로 고장 징후를 사전 탐지**, ③ **운영 스케줄·정비 일정과 자동 연동**해 **다운타임을 줄이고 처리 효율을 높이는** 것이 골자다. *"크레인이 고장을 미리 안다"* 는 보도 표현처럼, **사후 정비(Reactive Maintenance) → 예지보전(Predictive Maintenance, PdM)** 으로 운영 패러다임을 전환하는 사업이다. 시스템구조 관점에서 본 사업은 ① **수만 개 산업 센서의 데이터 수집·전처리**, ② **현장 엣지 추론 + 중앙 학습의 분산 AI 아키텍처**, ③ **OT(운영기술)·IT(정보기술) 융합과 보안**, ④ **디지털 트윈 기반 운영 시뮬레이션**, ⑤ **고가용성·내결함성을 갖는 산업용 인프라** 까지 종합적으로 다뤄야 하는 **산업 AIoT 시스템 아키텍처** 과제다.

## 2. 핵심 개념

### 항만 AI 예지보전 시스템 아키텍처

```
┌──────────────────────────────────────────────────────────────┐
│ ① 현장 센서 계층 : 진동·전류·온도·CCTV·LiDAR (수만 채널)     │
├──────────────────────────────────────────────────────────────┤
│ ② OT 게이트웨이 : OPC UA·Modbus 수집, 시각 동기화            │
├──────────────────────────────────────────────────────────────┤
│ ③ 엣지 추론 노드: 이상 탐지·임계치 경보, 저지연 응답         │
├──────────────────────────────────────────────────────────────┤
│ ④ 데이터 허브  : 시계열 DB·데이터 레이크, 스트리밍 파이프    │
├──────────────────────────────────────────────────────────────┤
│ ⑤ 중앙 AI 플랫폼: 모델 학습·재학습·디지털 트윈 시뮬레이션    │
├──────────────────────────────────────────────────────────────┤
│ ⑥ 운영 연계   : 정비 워크오더·스케줄러·SCADA·터미널 운영계   │
├──────────────────────────────────────────────────────────────┤
│ ⑦ 보안·관제   : OT 보안·세그멘테이션·침입 탐지·SOC          │
└──────────────────────────────────────────────────────────────┘
```

### 사후 정비 vs AI 예지보전 비교

다음 표는 두 운영 모델의 차이를 정리한다.

| 구분 | 사후 정비(Reactive) | AI 예지보전(Predictive) |
|------|---------------------|------------------------|
| 정비 시점 | 고장 발생 후 | 고장 징후 사전 탐지 |
| 데이터 활용 | 정기 점검 기록 | 실시간 다채널 센서 |
| 분석 방법 | 작업자 경험·체크리스트 | ML 이상탐지·잔존수명(RUL) |
| 다운타임 | 길고 예측 불가 | 짧고 계획 가능 |
| 정비 비용 | 응급 부품·인력 비용↑ | 계획 정비·재고 최적화 |
| 운영 효율 | 처리량 변동 큼 | 처리량 안정·예측 가능 |
| 인프라 | 분산·기록 위주 | 통합·실시간 스트리밍 |
| 의사결정 | 사람 중심 | AI 추천 + 사람 승인 |

### 산업 AIoT 시스템의 핵심 구성요소

- **현장 센서 계층**: 진동·전류·온도·압력·CCTV·LiDAR 등 멀티모달 센서, 시각 동기화
- **OT 게이트웨이**: OPC UA·Modbus·PROFINET 프로토콜 통합, 데이터 정규화
- **엣지 컴퓨팅**: 저지연 이상탐지(<100ms), 모델 양자화·경량화, 오프라인 운용
- **데이터 파이프라인**: Kafka·Flink 기반 스트리밍, 시계열 DB(InfluxDB·TimescaleDB)
- **AI 모델**: 이상탐지(Autoencoder·Isolation Forest), 잔존수명(LSTM·Transformer)
- **디지털 트윈**: 장비·터미널 가상 모델, what-if 시뮬레이션, 운영 최적화
- **MLOps**: 모델 레지스트리·드리프트 감지·재학습·A/B 배포
- **OT 보안**: IT/OT 망 분리, 마이크로세그멘테이션, 산업제어시스템(ICS) 침입 탐지
- **고가용성 설계**: 이중화·페일오버·재해복구, 24/7 무중단 운영

### 도입 시 고려사항

- **OT/IT 융합**: 폐쇄망 OT와 IT 망 연결 시 보안 경계·데이터 일방향 게이트 검토
- **엣지·클라우드 분담**: 저지연 추론은 엣지, 모델 학습은 중앙으로 책임 분리
- **데이터 품질**: 센서 결측·노이즈가 모델 정확도 좌우 → 자동 품질 게이트
- **모델 신뢰성**: 오탐·미탐 비용 차이 큼(미탐=장비 파손, 오탐=불필요 정비)
- **현장 운영 통합**: 예측 결과가 정비 워크오더·SCADA와 자동 연계되어야 가치 발생
- **OT 보안**: KISA 피지컬 AI 보안모델·NIST CSF·IEC 62443 등 표준 준수
- **표준 준수**: ISO 23247(디지털 트윈), ISO 13374(상태 기반 정비) 등
- **인력·역량**: 데이터 사이언티스트·OT 엔지니어·도메인 전문가 협업 체계
