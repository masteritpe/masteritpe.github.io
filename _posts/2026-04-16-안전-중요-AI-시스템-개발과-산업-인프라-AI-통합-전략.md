## 1. 뉴스 요약

코레일과 카카오모빌리티가 AI 기반 철도 안전 강화를 위한 기술협력을 발표했다. AI를 활용한 실시간 위험 감지, 운행 최적화, 예지보전 등을 공동 개발한다. 같은 시기 남부발전은 4족 보행 로봇을 활용한 '피지컬 AI' 기반 발전소 운영 혁신 발표회를 개최하며, AI 로봇이 위험 구역을 자율 순찰하고 설비 이상을 탐지하는 시스템을 시연했다. 이러한 사례들은 안전-중요(Safety-Critical) 영역에 AI를 통합할 때의 소프트웨어 공학적 과제를 부각시킨다.

## 2. 핵심 개념

### Safety-Critical AI 시스템 개발 생명주기

```
┌──────────────────────────────────────────────────────────────┐
│         Safety-Critical AI 개발 V-모델 확장                    │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  안전 요구분석 ◄──────────────────────► 안전 검증(V&V)        │
│       │                                      ▲               │
│       ▼                                      │               │
│  AI 시스템 설계 ◄────────────────► 시스템 통합 시험            │
│       │                                  ▲                   │
│       ▼                                  │                   │
│  ML 모델 설계 ◄──────────────► 모델 검증/시험                 │
│       │                              ▲                       │
│       ▼                              │                       │
│  데이터 수집·학습 ◄──────► 데이터 품질 검증                    │
│              │                                               │
│              ▼                                               │
│      모델 학습·최적화                                          │
│                                                              │
│  ※ 각 단계에서 안전 분석(HAZOP/FMEA) 및 추적성 확보 필수       │
└──────────────────────────────────────────────────────────────┘
```

### 기존 소프트웨어 V&V vs AI 시스템 V&V 비교

| 구분 | 기존 소프트웨어 V&V | AI/ML 시스템 V&V |
|------|-------------------|-----------------|
| 결정론성 | 동일 입력 → 동일 출력 | 확률적 출력 (비결정론적) |
| 테스트 커버리지 | 코드 커버리지 측정 가능 | 데이터 커버리지 정의 필요 |
| 결함 원인 | 코드 로직 오류 | 데이터 편향, 분포 이동(Drift) |
| 검증 기준 | 명세 기반 Pass/Fail | 성능 지표(정확도, F1) + 안전 경계 |
| 설명 가능성 | 코드 추적 가능 | XAI 기법 별도 적용 필요 |
| 형상 관리 | 소스코드 버전 관리 | 코드 + 데이터 + 모델 버전 관리 |
| 인증 표준 | IEC 61508, DO-178C | EASA AI 가이드라인, ISO/PAS 8800 |


## 3. 기술사 출제 포인트

### 예상 문제
1. **Safety-Critical 시스템에 AI/ML을 적용할 때의 소프트웨어 공학적 과제와 대응 방안을 설명하시오**
   - AI 시스템의 비결정론적 특성과 안전성 보증의 충돌
   - V-모델 확장: 데이터 품질 검증, 모델 V&V, ODD(Operational Design Domain) 정의
   - 안전 분석 기법 적용: AI-FMEA, AI-HAZOP
   - 런타임 모니터링: 입력 분포 이동 감지, 신뢰도 기반 안전 제어

2. **AI 기반 예지보전(Predictive Maintenance) 시스템의 개발 방법론과 MLOps 전략을 설명하시오**
   - 데이터 파이프라인: 센서 데이터 수집 → 전처리 → 특징 추출 → 모델 학습
   - CT/CI/CD: Continuous Training, 모델 재학습 트리거 조건
   - 모델 거버넌스: 모델 레지스트리, A/B 테스트, 롤백 전략
   - OT 환경 특수성: 네트워크 분리, 엣지 배포, 실시간 제약

3. **IEC 61508 기능안전과 AI 시스템 인증 프레임워크를 비교하시오**
   - SIL(Safety Integrity Level) 등급과 AI 모델 신뢰도 매핑
   - EASA AI 로드맵: Learning Assurance, AI Safety Assessment
   - ISO/PAS 8800(자동차 AI 안전) 주요 요구사항

### 답안 작성 키워드
`Safety-Critical AI`, `기능안전(IEC 61508)`, `V-모델 확장`, `AI V&V`, `ODD(Operational Design Domain)`, `예지보전(PdM)`, `MLOps`, `AI-FMEA`, `런타임 모니터링`, `Fail-safe`, `Purdue Model`, `데이터 드리프트`

## 4. 관련 토픽 연계

| 연계 토픽 | 연계 포인트 |
|-----------|------------|
| 소프트웨어 품질보증(SQA) | AI 시스템의 비결정론적 특성에 대한 품질 보증 방법론 확장 |
| DevOps/MLOps | Safety-Critical 환경의 CI/CD 파이프라인 설계 제약 |
| 설명가능한 AI(XAI) | 안전 인증을 위한 모델 판단 근거 설명 필수 |
| OT/ICS 보안 | 산업 인프라 AI 통합 시 Purdue Model 기반 보안 설계 |
| 피지컬 AI | 로봇·자율시스템의 물리 세계 상호작용과 안전 설계 |
| 디지털 트윈 | 안전-중요 시스템의 시뮬레이션 기반 테스트 환경 |
