## 1. 뉴스 요약

네이트 등(2026-05-17)이 보도한 바에 따르면, **AI 학습 목적의 개인정보 활용을 허용하는 「개인정보 보호법」 일부 개정안**이 국회 상임위를 통과했다. 보도에 따르면 개정안은 ① **공익·연구·산업 R&D 목적의 AI 학습데이터 구축**에 한해 ② **가명처리·안전성 확보 조치를 전제로** ③ **정보주체 동의 없이도 개인정보를 학습데이터로 활용**할 수 있는 **특례**를 신설하되, ④ **사전 영향평가·전송요구권·자동화 결정 거부권** 등 **이용자 보호장치를 강화**한다. 이는 **일본 APPI(개인정보보호법) 개정(2017·2020)** 의 가명정보·익명가공정보 활용 모델을 벤치마킹한 것으로, 한국이 **"제한적 옵트아웃 + 강한 안전조치"** 형태의 **'규제 완화-안전장치 강화'** 균형형 모델을 채택하는 신호다. 데이터베이스 관점에서 이번 입법은 **AI 학습 데이터셋의 라이프사이클(수집→가명→저장→학습→파기) 전 단계에 걸친 데이터 거버넌스·DB 설계 의무**를 새롭게 부과한다.

## 2. 핵심 개념

### 가명정보·AI 학습 특례의 정의와 필요성

**가명정보**는 추가정보 없이는 특정 개인을 알아볼 수 없도록 처리한 정보로, 식별 가능성을 낮추되 **분석·결합 가치는 유지**하는 중간 상태다. 이번 특례는 가명정보의 **AI 학습 활용을 명시적으로 허용**하되, **재식별·결합·유출 위험을 차단**하는 ① 기술적 조치(암호화·DP·격리), ② 관리적 조치(가명정보 명세서·접근통제·감사), ③ 절차적 조치(영향평가·전송요구권·자동화 결정 거부)를 부과한다. 필요성은 ① **국가 AI 산업 경쟁력 확보**(데이터 부족 해소), ② **공익·연구 데이터 활용 활성화**, ③ **무단 수집·재식별 위험 차단**, ④ **글로벌 데이터 흐름 호환성**(GDPR·APPI·CCPA) 확보다.

### AI 학습데이터 라이프사이클·DB 거버넌스 아키텍처

```
┌──────────────────────────────────────────────────────────────┐
│ ① 수집·동의 : 동의·정당이익·공익 근거, 수집 출처 메타데이터│
├──────────────────────────────────────────────────────────────┤
│ ② 가명처리 : 식별자 제거·일반화·k-익명성·교란·합성·DP        │
├──────────────────────────────────────────────────────────────┤
│ ③ 저장·격리: 추가정보 분리 저장, 키 관리(KMS·HSM), 암호화    │
├──────────────────────────────────────────────────────────────┤
│ ④ 결합·반출: 결합전문기관·반출 심사·결합 키 격리             │
├──────────────────────────────────────────────────────────────┤
│ ⑤ 학습·이용: 학습 환경 격리(VPC·전용존), 차분 프라이버시     │
├──────────────────────────────────────────────────────────────┤
│ ⑥ 모니터링 : 재식별·멤버십 추론·모델 반전 공격 탐지         │
├──────────────────────────────────────────────────────────────┤
│ ⑦ 권리행사 : 전송요구·자동화 결정 거부·이의제기 인터페이스   │
├──────────────────────────────────────────────────────────────┤
│ ⑧ 파기·감사: 보유기간·자동 파기, 감사 로그·DPIA 결과         │
└──────────────────────────────────────────────────────────────┘
```

### 기존 옵트인 동의 모델 vs AI 학습 특례 비교

다음 표는 기존 동의 모델과 AI 학습 특례 모델의 차이를 정리한다.

| 구분 | 기존 옵트인 동의 모델 | AI 학습 특례(개정안) |
|------|----------------------|----------------------|
| 처리 근거 | 정보주체 사전 동의 | 가명처리 + 안전조치 + 영향평가 |
| 정보 형태 | 식별정보 | 가명·합성·익명 데이터 |
| 활용 범위 | 동의 목적 한정 | 공익·연구·R&D + 영향평가 통과분 |
| 결합 | 별도 동의·결합전문기관 | 결합전문기관·반출 심사 강화 |
| 정보주체 권리 | 열람·삭제 | 전송요구·자동화 결정 거부 추가 |
| 안전조치 | 일반 의무 | 분리저장·KMS·DPIA·로그 의무화 |
| 위반 시 책임 | 과징금·형사 | 가중 책임·시정명령·이용 정지 |
| 국제 표준 | GDPR 일반조항 | APPI 익명가공·GDPR 16조 변형 |
| 거버넌스 | DPO·내부관리계획 | DPO + AI 책임자 + 영향평가위원회 |

### 핵심 구성요소

- **가명처리 기법**: 식별자 제거·일반화·라운딩·교란(noise), k-익명성·l-다양성·t-근접성, 차분 프라이버시(DP), 합성 데이터(SDV)
- **데이터 저장 구조**: 추가정보(매핑 키) 별도 DB 저장, 키 관리 시스템(KMS·HSM), 컬럼 단위 암호화(TDE·필드 토큰화)
- **결합·반출 통제**: 가명정보 결합전문기관, 반출 심사위원회, 결합 키 단방향 해시·시드 격리
- **차분 프라이버시 학습**: DP-SGD·PATE, ε(엡실론) 예산 관리, 모델별 프라이버시 회계
- **재식별 위험 평가**: HIPAA Expert Determination·k-익명성 측정, 멤버십 추론·모델 반전 공격 시뮬레이션
- **권리행사 인프라**: 전송요구권 API, 자동화 결정 거부 UI, 설명 요구 응답 시스템
- **DPIA·AIIA 통합**: 개인정보 영향평가(DPIA) + AI 영향평가(AIIA), 영향평가 자동화 도구
- **감사·로그**: 누가·언제·무엇을·왜 접근했는지 불변 감사 로그, SIEM·DAM(Database Activity Monitoring) 연동

### 도입 시 고려사항

- **법령 준수**: 개정안 통과 후 시행령·고시 모니터링, 분야별 가이드라인(의료·금융·교육) 추적
- **DB 스키마 분리**: 식별정보(원본)·가명정보·추가정보(키) 3계층 분리, 물리적·논리적 격리
- **키 관리**: 매핑 키의 KMS·HSM 보관, 키 회전·접근통제·이중 승인
- **DP 학습 채택**: 의료·금융 등 민감 도메인 우선, ε 예산 정책 수립
- **재식별 모니터링**: 멤버십 추론·반전 공격·연결 공격 정기 시뮬레이션, 결합 출력물 검증
- **글로벌 정합**: GDPR·APPI·CCPA·중국 PIPL 데이터 이동 요건 매핑
- **영향평가 거버넌스**: DPIA + AIIA 통합 절차, 영향평가위원회 설치
- **이용자 권리 UX**: 전송요구권·자동화 결정 거부 UI 표준화, 응답 SLA
- **AI 모델 보안**: 모델 가중치 보호, 모델 추출·증류 공격 대응
- **분야별 특례**: 의료(보건의료데이터활용가이드라인), 금융(신용정보법), 교육(개정 교육데이터법) 충돌 정리
- **사고 대응**: 가명정보 유출·재식별 시 통지·신고 의무, 영향 범위 평가
- **인력·교육**: DPO·AI 책임자 지정, 가명처리 실무자 자격·교육 의무화

## 3. 기술사 출제 포인트

- 가명정보·익명정보·식별정보의 개념 정의와 비교
- k-익명성·l-다양성·t-근접성·차분 프라이버시 원리
- 차분 프라이버시 학습(DP-SGD·PATE)과 ε 예산 관리
- AI 학습데이터 라이프사이클(수집·가명·결합·학습·파기) 단계별 통제
- 가명정보 결합전문기관·반출 심사 절차
- 멤버십 추론·모델 반전·연결 공격 등 재식별 위험과 대응
- DPIA·AIIA 통합 영향평가 프레임워크
- 합성 데이터(SDV)와 가명·익명 처리 비교
- DB 분리저장·KMS·필드 토큰화·TDE 기반 데이터 보호
- 한국·일본 APPI·EU GDPR·미국 CCPA·중국 PIPL 비교
- 자동화 결정 거부권·전송요구권 등 정보주체 권리 인프라
- AI 책임자(DPO + AI Officer) 거버넌스 모델
