## 1. 뉴스 요약

전체 인터넷 트래픽의 90% 이상이 암호화되는 시대에 접어들면서, 복호화 없이 암호화 트래픽 내 위협을 탐지하는 기술이 보안 업계의 핵심 과제로 부상하고 있다. 기존의 SSL/TLS 복호화 방식은 프라이버시 침해, 성능 저하, 인증서 관리 부담 등의 한계가 있어, 메타데이터 분석·머신러닝 기반의 암호화 트래픽 분석(ETA: Encrypted Traffic Analysis) 기술이 차세대 네트워크 보안의 핵심으로 주목받고 있다.

## 2. 핵심 개념

### 암호화 트래픽 위협 탐지 접근법 비교

| 구분 | SSL/TLS 복호화 방식 | ETA(비복호화 방식) |
|------|-------------------|--------------------|
| 원리 | 중간자(MITB) 방식 트래픽 복호화 후 검사 | 메타데이터·패턴 분석으로 위협 추론 |
| 프라이버시 | 침해 우려 (평문 노출) | 보호 (콘텐츠 미확인) |
| 성능 영향 | 높음 (CPU 집약적 복호화) | 낮음 (메타데이터만 처리) |
| TLS 1.3 대응 | 제한적 (eSNI/ECH 미지원) | 완전 대응 가능 |
| 규제 준수 | GDPR 등 프라이버시 규제 충돌 | 규제 친화적 |
| 탐지 정확도 | 높음 (전수 검사) | 중간~높음 (AI 모델 의존) |
| 인증서 관리 | 필요 (CA 인증서 배포) | 불필요 |

### ETA 핵심 분석 요소

```
[암호화 트래픽 분석(ETA) 다차원 특징 추출]
├── 플로우 메타데이터(Flow Metadata)
│   ├── 패킷 크기 분포(Packet Length Distribution)
│   ├── 패킷 간 도착 시간(Inter-Arrival Time)
│   ├── 세션 지속 시간(Session Duration)
│   └── 바이트 전송량 비율(Byte Ratio)
├── TLS 핸드셰이크 분석
│   ├── JA3/JA3S 핑거프린트
│   ├── 암호 스위트 목록(Cipher Suite List)
│   ├── TLS 확장 필드(Extensions)
│   └── 인증서 체인 메타데이터
├── DNS 연관 분석
│   ├── 도메인 엔트로피(Domain Entropy)
│   ├── DGA 패턴 탐지
│   ├── DNS-over-HTTPS 행위 분석
│   └── FQDN 생성 시점·빈도
└── 네트워크 행위 분석
    ├── 비콘 패턴(Beaconing Detection)
    ├── C2 통신 주기성 분석
    ├── 데이터 유출(Exfiltration) 패턴
    └── 수평 이동(Lateral Movement) 흔적
```

### JA3/JA3S 핑거프린팅 원리

```
┌─────────────────────────────────────────────┐
│          JA3 핑거프린트 생성 과정            │
├─────────────────────────────────────────────┤
│  Client Hello 패킷에서 추출:                │
│  ┌───────────────────────────────────────┐  │
│  │ TLS 버전 | 암호 스위트 | 확장 필드   │  │
│  │ | 타원곡선 | 포인트 형식              │  │
│  └───────────────────────────────────────┘  │
│           ↓ MD5 해시                        │
│  JA3 = "769,47-53-5-10-49161,0-10-11,23,0" │
│           ↓                                 │
│  JA3 Hash = e7d705a3286e19ea42f587b344ee6865│
├─────────────────────────────────────────────┤
│  활용: 악성코드 C2 통신, 봇넷, 피싱 사이트  │
│  식별을 위한 TLS 클라이언트 핑거프린팅       │
└─────────────────────────────────────────────┘
```

### AI/ML 기반 ETA 탐지 모델 아키텍처

| 모델 유형 | 적용 기법 | 탐지 대상 | 장점 |
|----------|----------|----------|------|
| 지도학습 | CNN, LSTM, Transformer | 알려진 악성코드 패밀리 | 높은 정확도 |
| 비지도학습 | Autoencoder, Isolation Forest | 제로데이·이상 트래픽 | 미지 위협 탐지 |
| 준지도학습 | GAN, Self-supervised | 레이블 부족 환경 | 적은 학습 데이터 |
| 연합학습 | Federated Learning | 다기관 협력 탐지 | 프라이버시 보존 |

## 3. 기술사 출제 포인트

1. **암호화 트래픽 분석(ETA)의 개념과 필요성**: 네트워크 트래픽 암호화 확산에 따른 보안 사각지대 문제와 ETA 기술의 동작 원리를 설명하시오.
2. **JA3/JA3S 핑거프린팅 기법**: TLS 핸드셰이크 메타데이터를 활용한 클라이언트·서버 핑거프린팅의 원리와 위협 인텔리전스 연동 방안을 논하시오.
3. **TLS 1.3 환경에서의 보안 모니터링 과제**: TLS 1.3의 보안 강화 특성(eSNI, 0-RTT, 축소된 핸드셰이크)이 기존 보안 모니터링에 미치는 영향과 대응 방안을 설명하시오.
4. **프라이버시 보존형 네트워크 위협 탐지**: 복호화 없는 위협 탐지와 프라이버시 규제(GDPR, 개인정보보호법) 준수를 동시에 충족하는 기술적 방안을 논하시오.

## 4. 관련 토픽 연계

| 연계 토픽 | 연관성 |
|----------|--------|
| 제로트러스트 아키텍처 | 암호화 트래픽 환경에서의 지속적 신뢰 검증 체계 |
| NDR(Network Detection & Response) | ETA를 핵심 엔진으로 활용하는 차세대 네트워크 탐지 솔루션 |
| SASE/SSE | 클라우드 기반 보안 서비스에서의 암호화 트래픽 처리 |
| AI 기반 보안 | 머신러닝·딥러닝 모델을 활용한 위협 탐지 고도화 |
| TLS 1.3 프로토콜 | 암호화 강화에 따른 보안 가시성 변화와 대응 기술 |
