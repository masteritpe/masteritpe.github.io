### [프로젝트관리] 역량삼각형 (Talent Triangle)

**[정의]**

프로젝트 관리자에게 필요한 기술에 대해 3가지 측면에서의 역량 개발 프레임워크

**[핵심/키워드]**

* 프로젝트 관리자의 핵심 역량
* 3가지 역량 (기술적 프로젝트 관리, 리더십, 전략 및 비즈니스 관리)
* 상호 연결, 균형, 지속적 개발

### [네트워크] 소스 코딩 (Source Coding)

**[정의]**

데이터의 효율적 전송 위해 데이터의 중복 제거하여 압축하는 기술

**[핵심/키워드]**

* 효율, 압축
* 아날로그 → 디지털, Shannon의 1정리
* ASCII, JPEG, MPEG, Huffman, RLC, LZW, PCM, DM

### [프로젝트관리] 리더십 (Leadership)

**[정의]**

조직의 비즈니스 목표를 달성할 수 있도록 팀을 이끌고 동기부여하며 지시하기 위해 필요한 지식, 기술 및 행동

**[핵심/키워드]**

* 유형 (자유방임, 거래, 섬김, 변신, 카리스마, 상호작용)
레빈(전제, 민주, 방임형)
브레이크와 모턴(인간관계, 과업)
허시와 블랜차드 모형(지시, 판매, 참여, 위임)
하우스 목표-경로이론(지시, 지원, 참여, 성취)
* 역량 삼각형

### [네트워크] 채널 코딩 (Channel Coding)

**[정의]**

전송하는 과정에서 발생하는 오류를 감지하고 수정하기 위해 원래 정보에 여분의 정보를 추가하는 기술

**[핵심/키워드]**

* 오류 감지, 오류 수정, 잉여 비트
* 디지털 → 디지털, Shannon 2정리
* Parity, Checksum, CRC, Hamming, RS, LDPC, Polar, Convolution, Turbo

### [프로젝트관리] PMBok 7th

**[정의]**

PMBok 6th의 프로젝트 관리에서 일반적으로 수용되고 실행되는 원칙과 전반적인 프로젝트의 결과에 초점을 맞춘 프로젝트 관리 지침 안내서

**[핵심/키워드]**

* 원칙, 결과, 성과 중심, 애자일 확장
* 가치전달시스템 (전략, 포트폴리오, 프로그램, 프로젝트, 운영)
* 8가지 성과관리 (이해관계자, 팀, 생애주기, 기획, 성과, 인도, 측정, 불확실성)
* 12가지 관리원칙(스튜어드십, 팀, 이해관계자, 가치, 시스템사고, 리더십, 조정, 품질, 복잡성, 위험, 회복력, 변화)

### [네트워크] 다중화 (Multiplexing)

**[정의]**

전송로 하나에 데이터 신호 여러 개를 중복시켜 고속 신호 하나를 만들어 전송하는 방식

**[핵심/키워드]**

* 여러 개 통합, 하나의 통신채널
* 구성 (Mux, Demux)
* 물리계층 (SDM, FDM, TDM, CDM, WDM, OFDM, PDM, GFDM)
** OSI 7 Layer (5계층 → 4계층 → 3계층)

### [프로젝트관리] 범위 관리

**[정의]**

프로젝트를 성공적으로 완료하기 위해 반드시 필요한 작업만을 빠짐없이 프로젝트에 포함시키기 위해 필요한 프로세스

**[핵심/키워드]**

* 반드시 필요한 작업 포함
* 프로세스 (계획, 수집, 정의, WBS, 확인, 통제)
* 벙뮈관리 기준선

### [네트워크] 다중접속/다원접속 (Multiple Access)

**[정의]**

서로 다른 사용자들이 같은 통신 채널이나 전달 매체를 공유하여 통신하는 기술

**[핵심/키워드]**

* 여러 사용자 공유
* Random Access (ALOHA, CSMA)
* Channel (SDMA, FDMA, TDMA, CDMA, OFDMA, SC-FDMA, NOMA)

### [프로젝트관리] 요구사항 수집

**[정의]**

프로젝트 목표 달성에 필요한 사항과 이해관계자의 요구사항을 판별하고 문서화하고 관리하는 프로세스

**[핵심/키워드]**

* 유형 (외부, 기능, 성능, 비기능, 제약사항, 속성)
* 기법 (인터뷰, 브레인스토밍, 포커스, 설문, 문서, 투표, 독단, 다수결, 친화도, 아이디어, 명목집단, 관찰, 워크샵, 프로토타입, 연관도, 제품분석)
* 요구사항 정의서, 요구사항 명세서

### [네트워크] 후진 오류 수정 (Backward Error Correction, BEC)

**[정의]**

수신측에서 오류 발견 시 송신측에 재전송을 요청하여 오류를 정정하는 방식

**[핵심/키워드]**

* 송신측 재전송, ARQ
* 오류 검출 (Parity, CRC, Checksum)
* ARQ (Stop & Wait ARQ, Go-back-N ARQ, Selective ARQ, Hybrid ARQ, Adaptive ARQ)

### [프로젝트관리] 요구사항 명세서(Software Requirement Specification, SRS)

**[정의]**

프로젝트의 목표, 기능, 성능, 제약 조건 등 소프트웨어가 충족해야 할 요구사항을 체계적으로 정리한 문서

**[핵심/키워드]**

* 요구사항 정리 문서
* 원칙 (명확, 완전, 일관, 추적가능, 검증가능, 변경용이, 우선순위, 이해가능)
* 항목 (프로젝트 개요, 이해관계자 정의, 기능적 요구사항, 비기능적 요구사항, 시스템 인터페이스, 제약 조건, 변경 관리, 수락 기준)
* 요구사항 추적 매트릭스, IEEE 830, ISO/IEC/IEEE 29148

### [네트워크] Token Bucket Algorithm

**[정의]**

네트워크 트래픽이나 API 요청의 처리 속도를 제어(rate limiting)하는 데 사용되는 알고리즘

**[핵심/키워드]**

* 토큰 생성, 토큰 소모, 평균속도 제어
* 구성 (토큰, 버킷), 버스트 트래픽 허용
* 동작 (토큰 생성, 요청 처리)
* Traffic Policing, Traffic Shaping

### [프로젝트관리] 작업분해도(Work Breakdown Structure, WBS)

**[정의]**

프로젝트 팀에서 실행할 전체 작업범위를 계층구조로 세분해 놓은 계통도

**[핵심/키워드]**

* 작업 범위 계층구조
* 구성 (Work Package, Planning Package, Control Account, Decomposition, Code of Account, WBS Dictionary)

### [프로젝트관리] 주공정경로법 (Critical Path Method, CPM)

**[정의]**

가장 오래 걸리는 경로를 찾아 프로젝트 최소 완료 시간을 계산하는 방법

**[핵심/키워드]**

**[정의]**

* 아날로그 → 디지털
* 표본화, 양자화, 부호화, 재생, 복호화, 재구성
* 표본화(나이퀴스트 샘플링 / 나이퀴스트-새넌 표본화 정리)
* 소스코딩, 변조, 새논 1정리
* DM, DPCM

### [프로젝트관리] 주공정연쇄법(Critical Chain Method, CCM)

프로젝트 일정의 불확실성에 대비하기 위해 버퍼를 두고 관리하는 일정 관리 기법

**[핵심/키워드]**

* 자원 제약 고려, CPM 개선
* 버퍼 (프로젝트, 피딩, 자원)

### [네트워크] OFDM (Orthogonal Frequency Division Multiplexing)

고속의 송신 신호를 다수의 직교하는 협대역 반송파로 다중화 시키는 기술

* LTE / 5G
* DFT-s-OFDM, FBMC

### [프로젝트관리] 획득가치관리(Earned Value Management, EVM)

* Traffic Shaping

* 체크시트, 히스토그램, 파레토 차트, 원인결과도(특성요인도), 산점도, 층별(분류) 분석, 관리도

### [네트워크] CSMA/CA (Carrier-Sense Multiple-Access with Collision Avoidance)

**[정의]**

주로 무선 네트워크에서 충돌 발생 전 회피하기 위해 사용하는 매체 접근 제어(MAC) 프로토콜

**[핵심/키워드]**

* 충돌 회피
* 구성 (DCF, DIFS, SIFS, RTS, CTS, NAV, Contention Window, Back-off)
* CSMA / CD
* Hidden Node Problem / Exposed Node Problem 개선
