## 1. 뉴스 요약

사이언스타임즈(2026-05-06) 보도에 따르면, **한국전자통신연구원(ETRI) 주도로 “정책지능(Policy Intelligence) 프레임워크” 국제표준이 ITU-T에서 확정**됐다. 이 표준은 **공공 정책 결정 과정에 AI를 적용하기 위한 데이터·모델·운영의 표준화 절차**를 규정하며, ① **데이터 수집·연계 표준**, ② **정책 모델링·시뮬레이션 표준**, ③ **결정 지원·근거 제시 표준**, ④ **모니터링·피드백 표준**을 포함한다. 정책 의사결정의 **투명성·공정성·재현성**을 보장하고, **부처 간 정책 데이터 상호운용성**과 **국제 비교가능성**을 확보하는 것이 목표다. IT경영 관점에서 이 표준은 ① **공공 AI 거버넌스 프레임워크**의 핵심 축, ② **디지털플랫폼정부**의 정책 의사결정 인프라, ③ **EA(Enterprise Architecture) 기반 정책 데이터·서비스 매핑**, ④ **ITSM·SLA 기반 정책 AI 서비스 운영**, ⑤ **국가 AI법·ISO/IEC 42001(AIMS)와의 정합성**을 통합 설계해야 하는 공공 거버넌스 과제다.

## 2. 핵심 개념

### 정책지능 프레임워크 표준 구조

```
┌──────────────────────────────────────────────────────────────┐
│ ① 데이터 계층  : 행정·통계·민원·외부 데이터 표준 연계        │
├──────────────────────────────────────────────────────────────┤
│ ② 모델 계층    : 정책 시뮬레이션·예측·인과추론 모델          │
├──────────────────────────────────────────────────────────────┤
│ ③ 결정 지원    : 대안 비교·영향평가·근거 제시(XAI)           │
├──────────────────────────────────────────────────────────────┤
│ ④ 의사결정     : 정책 결정자·휴먼인더루프 승인               │
├──────────────────────────────────────────────────────────────┤
│ ⑤ 집행·모니터링: KPI·지표·실시간 효과 추적                   │
├──────────────────────────────────────────────────────────────┤
│ ⑥ 피드백 루프  : 결과 회귀·재학습·정책 보정                  │
├──────────────────────────────────────────────────────────────┤
│ ⑦ 거버넌스     : 윤리·공정성·개인정보·책임성 위원회          │
└──────────────────────────────────────────────────────────────┘
```

### 기존 정책 의사결정과 정책지능 비교

다음 표는 두 방식의 차이를 정리한다.

| 구분 | 기존 정책 의사결정 | 정책지능(Policy Intelligence) |
|------|---------------------|-------------------------------|
| 근거 | 부처 보고·전문가 자문 | 데이터·시뮬레이션·인과추론 |
| 속도 | 수개월 단위 | 실시간·반복 가능 |
| 투명성 | 비공개·비정형 | 모델·데이터 출처 표준 공개 |
| 재현성 | 낮음 | 표준 절차로 재현 가능 |
| 영향평가 | 정성적·사후적 | 정량적·사전 시뮬레이션 |
| 부처 협업 | 문서·회의 기반 | 표준 데이터·API 공유 |
| 국제 비교 | 어려움 | ITU-T 표준 기반 가능 |
| 책임성 | 모호 | XAI·로그·감사 의무 |

### 핵심 구성요소

- **표준화된 정책 데이터 모델**: 행정·통계·외부 데이터의 메타데이터·온톨로지 표준
- **정책 시뮬레이션 엔진**: 시스템 다이내믹스·에이전트 기반 모델·머신러닝 인과추론
- **설명가능 AI(XAI)**: 정책 권고의 근거·대안·민감도 분석 자동 제시
- **휴먼인더루프 결정**: AI 권고를 정책결정자가 검토·승인하는 의사결정 워크플로
- **모니터링·피드백 시스템**: KPI·OKR 연동, A/B 테스트, 실시간 효과 추적
- **거버넌스 체계**: 윤리위원회·개인정보영향평가(PIA)·알고리즘 영향평가(AIA)
- **상호운용성 인프라**: 부처 간 표준 API·공공데이터포털·정책 데이터 메시(Data Mesh)
- **표준 정합**: ITU-T 정책지능 표준 + ISO/IEC 42001(AIMS) + NIST AI RMF + 「인공지능법」

### 도입 시 고려사항

- **데이터 품질·편향**: 정책 데이터의 대표성·결측·편향 → 데이터 거버넌스 강화
- **알고리즘 책임성**: 정책 결정의 책임 소재 명확화, 알고리즘 영향평가 의무화
- **개인정보보호**: 가명·익명 처리, 동형암호·연합학습 등 PETs 적용
- **공정성·차별 방지**: 보호 속성 점검, 형평성(Equity) 지표 모니터링
- **시민 참여·수용성**: 결정 근거 공개, 의견 수렴 채널, 디지털 격차 대응
- **부처 협업 거버넌스**: 데이터 공유 협약, 표준 API, 정책 데이터 위원회
- **전문 인력**: 정책분석가+데이터과학자+AI엔지니어의 융합 조직 운영
- **국제 정합**: ITU-T·OECD AI 원칙·EU AI Act와의 정합성 유지

## 3. 기술사 출제 포인트

- 정책지능(Policy Intelligence)의 정의와 도입 필요성
- ITU-T 정책지능 표준의 4계층(데이터·모델·결정·피드백) 구조
- 기존 정책 의사결정과 정책지능 방식의 차이
- 디지털플랫폼정부와 정책지능의 연계 모델
- 공공 AI 거버넌스 프레임워크(ISO/IEC 42001·NIST AI RMF·인공지능법)
- 정책 시뮬레이션 기법(시스템 다이내믹스·ABM·인과추론) 비교
- 설명가능 AI(XAI)와 알고리즘 영향평가(AIA)
- EA 관점의 정책 데이터·서비스·기술 아키텍처 매핑
- ITSM·SLA 기반 정책 AI 서비스 운영 모델
- 부처 간 데이터 상호운용성과 데이터 메시(Data Mesh)
- 개인정보보호 강화기술(PETs) 적용 방안
- 공공 의사결정 AI의 윤리·공정성·책임성 거버넌스
