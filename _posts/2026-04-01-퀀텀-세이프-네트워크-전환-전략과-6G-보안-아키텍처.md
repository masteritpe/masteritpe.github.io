## 1. 뉴스 요약

6G 포럼이 차세대 보안 네트워크 체계인 '퀀텀 세이프 네트워크(QSN)' 전환 전략을 제시했다. 양자컴퓨터의 발전으로 기존 RSA·ECC 기반 공개키 암호 체계가 위협받는 상황에서, 통신 인프라 전반에 양자내성암호(PQC)를 적용하고 양자키분배(QKD)와 결합한 하이브리드 보안 아키텍처로의 전환이 본격화되고 있다. 동시에 F5는 멀티클라우드·에지 분산 환경에서 양자내성암호를 결합한 앱 전송·보안 플랫폼 강화를 발표하며, 네트워크 계층별 양자 안전성 확보가 산업 전반의 핵심 과제로 부상하고 있다.

## 2. 핵심 개념

### 퀀텀 세이프 네트워크(QSN) 아키텍처

```
┌──────────────────────────────────────────────────┐
│            Quantum-Safe Network (QSN)             │
├──────────────────────────────────────────────────┤
│  ┌────────────┐  ┌────────────┐  ┌────────────┐ │
│  │  QKD 계층   │  │  PQC 계층   │  │ 하이브리드  │ │
│  │ (물리 보안) │  │ (알고리즘)  │  │  통합 계층  │ │
│  ├────────────┤  ├────────────┤  ├────────────┤ │
│  │ 양자키분배  │  │ CRYSTALS-  │  │ QKD + PQC  │ │
│  │ BB84/E91   │  │ Kyber/     │  │ 이중 키     │ │
│  │ 프로토콜   │  │ Dilithium  │  │ 캡슐화     │ │
│  │            │  │ SPHINCS+   │  │            │ │
│  └────────────┘  └────────────┘  └────────────┘ │
├──────────────────────────────────────────────────┤
│  적용 영역: 5G/6G 코어, 전송망, 에지, 클라우드   │
└──────────────────────────────────────────────────┘
```

### 기존 네트워크 보안 vs QSN 비교

| 구분 | 기존 네트워크 보안 | 퀀텀 세이프 네트워크 |
|------|-------------------|---------------------|
| 키 교환 | RSA, ECDH | CRYSTALS-Kyber, QKD |
| 전자서명 | RSA, ECDSA | CRYSTALS-Dilithium, SPHINCS+ |
| 보안 가정 | 소인수분해·이산로그 난제 | 격자·해시 기반 수학적 난제 |
| 양자 내성 | 취약 (Shor 알고리즘) | 내성 확보 |
| 키 길이 | 2048~4096bit | 800~1568byte (Kyber) |
| 전송 오버헤드 | 낮음 | 중간~높음 |
| 전환 방식 | - | 하이브리드 모드 → 완전 전환 |

### QSN 전환 로드맵 (6G 포럼 제안)

```
[Phase 1: 2024-2026] 현황 분석 및 시범 적용
├── 암호 인벤토리(Crypto Inventory) 구축
├── CRQC(암호학적으로 유의미한 양자컴퓨터) 위협 평가
├── 핵심 인프라 PQC 시범 적용
└── QKD 테스트베드 구축

[Phase 2: 2027-2029] 하이브리드 전환
├── 기존 암호 + PQC 하이브리드 운용
├── TLS 1.3 + PQC 하이브리드 키 교환
├── 5G 코어 네트워크 PQC 적용
└── 인증서 체계 이중화(기존 + PQC)

[Phase 3: 2030-2032] 완전 전환
├── 6G 네트워크 PQC 네이티브 설계
├── QKD + PQC 통합 키 관리 체계
├── 레거시 암호 체계 완전 폐기
└── 양자 안전 인증서 체계 단일화
```

### 네트워크 계층별 PQC 적용 전략

| 계층 | 적용 대상 | PQC 알고리즘 | 고려사항 |
|------|----------|-------------|---------|
| 전송 계층 | TLS/DTLS | Kyber(KEM) + Dilithium(서명) | 핸드셰이크 지연 증가 |
| 네트워크 계층 | IPsec/IKEv2 | Kyber + 기존 ECDH 하이브리드 | 패킷 크기 증가 |
| 응용 계층 | 인증서/PKI | Dilithium/SPHINCS+ | 인증서 크기 3~10배 증가 |
| 관리 계층 | SSH/SNMP | PQC KEM | 세션 설정 시간 증가 |
| 물리 계층 | 광전송망 | QKD 연동 | 거리 제한(~100km) |

## 3. 기술사 출제 포인트

1. **퀀텀 세이프 네트워크(QSN)의 개념과 구성요소**: PQC와 QKD를 결합한 하이브리드 보안 아키텍처의 설계 원칙과 계층별 적용 방안을 설명하시오.
2. **NIST PQC 표준 알고리즘(CRYSTALS-Kyber, Dilithium, SPHINCS+)의 특성 비교**: 각 알고리즘의 수학적 기반, 키 크기, 성능 특성을 비교하고 적용 시나리오를 논하시오.
3. **네트워크 인프라의 PQC 마이그레이션 전략**: 기존 PKI 체계에서 양자내성 암호 체계로의 전환 시 하이브리드 접근법의 필요성과 단계별 전환 방안을 설명하시오.
4. **CRQC 위협 대응과 Harvest Now, Decrypt Later 공격**: 양자컴퓨터에 의한 미래 복호화 위협에 선제적으로 대응하기 위한 암호 민첩성(Crypto Agility) 확보 방안을 논하시오.

## 4. 관련 토픽 연계

| 연계 토픽 | 연관성 |
|----------|--------|
| 포스트양자암호(PQC) | QSN의 핵심 암호 알고리즘으로 격자 기반·해시 기반 암호 적용 |
| 6G 네트워크 아키텍처 | 6G 설계 단계부터 양자 안전성을 내재화한 보안 아키텍처 |
| PKI/인증서 체계 | PQC 기반 인증서로의 전환과 하이브리드 인증서 운용 |
| 제로트러스트 | QSN 환경에서 지속적 인증·검증 체계와의 통합 |
| 암호 민첩성(Crypto Agility) | 알고리즘 교체에 유연하게 대응할 수 있는 시스템 설계 원칙 |
