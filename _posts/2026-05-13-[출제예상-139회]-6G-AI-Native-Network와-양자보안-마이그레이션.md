## 2. 개념·정의

### 6G AI-Native Network
> 5G가 AI를 "기능 추가"로 결합했다면, 6G는 **설계 단계부터 AI가 네트워크 계층에 내재화**(Network for AI / AI for Network).

### AI-RAN (AI-driven Radio Access Network)
> 무선 접속망 운영에 AI를 적용. 채널 예측·빔포밍·간섭 제어·에너지 최적화를 학습 기반으로 수행.

### 양자보안 마이그레이션 (PQC·QKD)
- **PQC** (Post-Quantum Cryptography): 양자컴퓨터에 내성 있는 새로운 수학 기반 암호 (격자·해시·코드 등)
- **QKD** (Quantum Key Distribution): 양자 물리 원리로 키를 분배 (BB84 프로토콜)
- **하이브리드**: 기존 + PQC + QKD 다층 방어

## 4. 핵심 기술요소

### 4.1 6G 6대 특징 (ITU-R IMT-2030)

| 특징 | 5G | 6G 목표 |
|------|-----|--------|
| 최대 속도 | 20 Gbps | 1 Tbps |
| 지연 | 1 ms | 0.1 ms |
| 연결 밀도 | 10⁶/km² | 10⁷/km² |
| 가용성 | 99.999% | 99.99999% |
| 에너지 효율 | 1× | 100× |
| 위치 정확도 | 10cm | 1cm |

### 4.2 AI-RAN 적용 영역

| 영역 | AI 적용 | 효과 |
|------|---------|------|
| 채널 추정 | 딥러닝 기반 CSI 예측 | 정확도↑·전송 효율↑ |
| 빔포밍 | RL 기반 동적 빔 선택 | 커버리지·SINR 개선 |
| 간섭 제어 | 멀티셀 협력 학습 | 셀 경계 성능↑ |
| 자원 할당 | DRL 스케줄링 | QoS·공정성 |
| 에너지 절약 | 트래픽 예측 슬립 | OpEx 30%↓ |

### 4.3 PQC 알고리즘 (NIST 표준화)

| 알고리즘 | 용도 | 기반 수학 |
|----------|------|-----------|
| **CRYSTALS-Kyber** (FIPS 203) | 키 캡슐화 | Module-LWE |
| **CRYSTALS-Dilithium** (FIPS 204) | 서명 | Module-LWE |
| **SPHINCS+** (FIPS 205) | 서명 | Hash-based |
| **FALCON** | 서명 | NTRU lattice |

### 4.4 QKD 핵심 (BB84)

- **양자 원리**: 측정 시 상태 붕괴, 복제 불가(No-Cloning)
- **프로토콜**: 송신자가 임의 기저로 큐비트 송신 → 수신자 임의 기저로 측정 → 기저 일치만 사용
- **장점**: 정보이론적 안전 (계산 복잡도 무관)
- **한계**: 거리 100km 이내, 중계기 필요, 비용 큼

### 4.5 양자보안 마이그레이션 단계

1. **자산 인벤토리** — 암호 사용 현황 (TLS·VPN·서명·KMS)
2. **위험 평가** — Harvest Now, Decrypt Later 대비
3. **PQC 시범 적용** — 하이브리드 KEM (TLS 1.3 + Kyber)
4. **전면 전환** — 정부 가이드(2030~2035 목표)
5. **QKD 도입** — 백본·금융·국방 고가치 회선

## 5. 운영 고려사항

- 2028~2029: 시범 서비스
- 2030: 상용화

### 5.2 양자보안 전환 도전과제

| 과제 | 대응 |
|------|------|
| 키 크기 증가 | TLS 핸드셰이크 최적화 |
| 호환성 | 하이브리드 (RSA+Kyber) |
| 인증서 체인 | PKI·CA PQC 지원 |
| 성능 | HSM 가속·캐싱 |
| 표준 변화 | NIST·KISA 가이드 추종 |
