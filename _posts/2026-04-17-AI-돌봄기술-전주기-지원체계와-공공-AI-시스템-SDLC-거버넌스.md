## 1. 뉴스 요약

보건복지부와 과학기술정보통신부가 2026-04-16 제7회 과기장관회의에서 'AI 돌봄기술 전(全) 주기 지원 전략'을 공동 발표했다. R&D → 기술개발 → 실증 → 표준화 → 서비스 확산 → 운영·평가까지 AI 돌봄 제품·서비스의 라이프사이클 전반을 국가가 일관되게 지원하는 체계이며, 동시에 피지컬 AI 개발 지원과 24시간 AI+IoT 스마트홈 돌봄을 추진한다. 연합뉴스·전자신문·한겨레·동아사이언스·뉴스1 등 다수 언론이 동시 보도했으며, 공공 AI 시스템 개발에 있어 기존 소프트웨어 SDLC를 확장한 'AI 시스템 전주기 관리'의 중요성을 부각시켰다.

## 2. 핵심 개념

### AI 시스템 전주기(End-to-End Lifecycle) 확장

```
┌──────────────────────────────────────────────────────────────┐
│      공공 AI 돌봄 시스템 전주기(Lifecycle) 거버넌스            │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  [기획]──[R&D]──[데이터]──[모델]──[실증]──[운영]──[평가]      │
│    │       │       │       │       │       │       │         │
│    ▼       ▼       ▼       ▼       ▼       ▼       ▼         │
│  요구사항  과제      데이터   학습    리빙랩   배포     KPI     │
│  사회영향  공모      수집/정제 평가    현장검증  모니터링 피드백  │
│  AI영향평가 표준화   라벨링   XAI검증  위험평가  드리프트  재투자 │
│                                                              │
│  ◄────────── 거버넌스 축 (공통 적용) ──────────►               │
│  ▪ AI 윤리·신뢰성(신뢰할 수 있는 AI)                           │
│  ▪ 개인정보보호(가명처리/DP)                                   │
│  ▪ 안전성(HAZOP/FMEA)                                        │
│  ▪ 품질(ISO/IEC 25059 AI 품질모델)                            │
│  ▪ 설명가능성(XAI) · 책임성(감사로그)                          │
│                                                              │
│  ※ 전주기에 거버넌스 관문(Gate)을 배치해 위험을 선제적으로 관리  │
└──────────────────────────────────────────────────────────────┘
```

### 기존 SDLC vs AI 시스템 전주기(SDLC for AI) 비교

| 구분 | 전통 SDLC | AI 시스템 전주기 |
|------|----------|----------------|
| 산출물 | 요구명세·설계·코드 | 코드 + 데이터셋 + 학습된 모델 |
| 형상 관리 | 소스코드 버전 | 코드·데이터·모델 3중 버전 |
| 테스트 | 단위·통합·시스템 | + 데이터 품질·모델 성능·공정성 |
| 출시 후 관리 | 버그 수정·패치 | + 드리프트 감시·재학습·모델 회수 |
| 위험 관리 | 결함·장애 | + 편향·환각·오용·데이터 오염 |
| 품질 표준 | ISO/IEC 25010 | ISO/IEC 25059, ISO/IEC 42001 |
| 참여자 | 기획·개발·QA | + 데이터 사이언티스트·윤리검토위원 |

### 공공 AI 돌봄 전주기 주요 단계

- **R&D/과제 기획**: 돌봄 현장 니즈 분석, AI 영향평가, 기술 로드맵
- **데이터 전주기**: 수집(센서/IoT) → 가명처리 → 라벨링 → 데이터셋 품질 보증(ISO/IEC 5259)
- **모델 개발**: 학습·검증·공정성 평가·XAI 기반 설명가능성 확보
- **실증(리빙랩)**: 통제 환경에서 실사용자 검증, 위험요소 FMEA
- **서비스 확산**: 공공기관 도입, 표준 API/데이터 연계, MLOps 배포
- **운영·평가**: 드리프트·편향 모니터링, 재학습 트리거, KPI 기반 정책 피드백

## 3. 기술사 출제 포인트

### 예상 문제
1. **공공 AI 시스템 전주기(End-to-End Lifecycle) 관리 방안과 기존 소프트웨어 SDLC와의 차이점을 설명하시오**
   - 데이터/모델/코드 3중 형상관리, Data Version Control(DVC)
   - 품질표준: ISO/IEC 25059(AI 품질모델), ISO/IEC 42001(AI 경영시스템)
   - AI 거버넌스 관문(Gate)과 위험평가 의무 절차
   - 운영 단계의 드리프트 감시·재학습 루프

2. **AI 기반 공공서비스(돌봄·복지) 개발을 위한 소프트웨어 공학적 접근 방안을 논하시오**
   - 요구공학: 이해관계자 식별, AI 영향평가(AIA)
   - 데이터 품질 프레임워크(ISO/IEC 5259), 가명처리·DP
   - 실증(리빙랩) 기반 반복적 개발과 안전성 평가(HAZOP/FMEA)
   - 신뢰할 수 있는 AI 7대 원칙(EU AI Act/TAI)

3. **MLOps와 기존 DevOps의 차이, 전주기 관점의 통합 방안을 설명하시오**
   - CI/CD → CT/CI/CD(Continuous Training 추가)
   - 모델 레지스트리·Feature Store·데이터 계보(Lineage)
   - Shadow 배포·A/B 테스트·Canary 전략
   - 감사 가능한 ML 파이프라인(SLSA for ML)

### 답안 작성 키워드
`AI 전주기(Lifecycle)`, `ISO/IEC 25059`, `ISO/IEC 42001`, `ISO/IEC 5259`, `AI 영향평가(AIA)`, `신뢰할 수 있는 AI(TAI)`, `MLOps/CT-CI-CD`, `Feature Store`, `Data Lineage`, `XAI`, `리빙랩`, `드리프트 모니터링`, `AI 거버넌스 관문(Gate)`

## 4. 관련 토픽 연계

| 연계 토픽 | 연계 포인트 |
|-----------|------------|
| SDLC·애자일 방법론 | AI 시스템 반복 개발과 실증 기반 점진적 출시 전략 |
| MLOps | 코드·데이터·모델 3중 형상관리와 자동 재학습 파이프라인 |
| ISO/IEC 25010·25059 | AI 품질특성(정확성·견고성·공정성)과 기존 품질모델의 통합 |
| 개인정보보호(PbD) | 돌봄 데이터 가명처리·차등 프라이버시(DP) 적용 |
| AI 거버넌스·EU AI Act | 고위험 AI 요구사항과 전주기 문서화·감사가능성 |
| 피지컬 AI·AIoT | 돌봄 로봇·스마트홈 센서와 소프트웨어 통합 아키텍처 |
