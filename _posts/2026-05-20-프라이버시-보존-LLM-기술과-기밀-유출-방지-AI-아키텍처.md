## 1. 뉴스 요약

2026-05-19 인공지능신문은 "**기밀 유출 걱정 없는 LLM 구축 및 활용… 고려대 임희석 교수, 프라이버시 보존 대형언어모델 기술 개발**"을 보도했다. 핵심은 **LLM의 학습·미세조정(Fine-Tuning)·추론(Inference) 전 과정에서 사용자·기업의 민감 정보가 모델 가중치나 응답을 통해 유출되지 않도록 보장**하는 학술 연구 성과다. 같은 시기 테크데일리(2026-05-18)는 "**생성형 AI 에이전트 보안 비상… LLM 침해 가정한 방어 체계 필요**"를 보도해 LLM이 **프롬프트 인젝션·모델 인버전·멤버십 추론·학습 데이터 추출** 공격에 노출돼 있음을 경고했다. 또한 5월 18일 시행된 **AI 학습용 개인정보 활용 특례**(개정 개인정보보호법)와 맞물려, **기업이 LLM 도입 시 가명·익명 정보를 안전하게 활용할 기술적 통제**가 필요한 시점이다. 이 흐름의 핵심 기술이 **프라이버시 보존 LLM(Privacy-Preserving LLM, PP-LLM)**으로, ① **차분 프라이버시(DP-SGD)**, ② **연합학습(Federated Learning)**, ③ **동형암호(Homomorphic Encryption)**, ④ **신뢰실행환경(TEE/Confidential Computing)**, ⑤ **데이터 디아이덴티피케이션** 기법을 결합한다. AI데이터분석 영역에서 본 회차 출제 가능성이 높은 핵심 주제다.

## 2. 핵심 개념

### LLM의 4대 프라이버시 위협

| 공격 유형 | 메커니즘 | 방어 기법 |
|---|---|---|
| 학습 데이터 추출(Training Data Extraction) | 특정 프롬프트로 학습 원문 복원 | DP-SGD, 학습 데이터 디아이덴티피케이션 |
| 멤버십 추론(Membership Inference) | 특정 데이터가 학습셋에 포함됐는지 추론 | 차분 프라이버시, Regularization |
| 모델 인버전(Model Inversion) | 응답을 통해 학습 데이터 복원 | DP, Output Perturbation |
| 프롬프트 인젝션·민감정보 노출 | 컨텍스트 내 기밀 노출 | 입출력 필터, RAG 권한 제어, Guardrail |

### 프라이버시 보존 LLM 핵심 기법 비교

| 기법 | 원리 | 적용 단계 | 성능 영향 |
|---|---|---|---|
| 차분 프라이버시(DP-SGD) | 그래디언트에 노이즈 추가 | 학습·미세조정 | 정확도 5~15% 손실 가능 |
| 연합학습(Federated Learning) | 데이터 비공유·모델 파라미터만 공유 | 분산 학습 | 통신 비용 ↑, 비IID 이슈 |
| 동형암호(HE) | 암호화 상태에서 연산 수행 | 추론 | 연산 비용 100~1000배 ↑ |
| TEE/Confidential Computing | 하드웨어 격리 영역에서 처리 | 학습·추론 | 메모리 한계, 사이드채널 위험 |
| 디아이덴티피케이션·합성데이터 | 가명·익명화·합성 데이터로 대체 | 데이터 전처리 | 유틸리티-프라이버시 트레이드오프 |
| RAG 권한 분리 | 사용자 권한별 벡터DB 인덱스 분리 | 추론 | 정확도 영향 적음 |

### 프라이버시 보존 LLM 통합 아키텍처

```
┌──────────────────────────────────────────────────────────────┐
│ ① 데이터 수집·전처리 계층                                    │
│   PII 탐지·마스킹·가명화·합성데이터 생성                     │
│   K-익명성·L-다양성·T-근접성·차분 프라이버시(글로벌)         │
├──────────────────────────────────────────────────────────────┤
│ ② 학습 계층(Privacy-Preserving Training)                     │
│   DP-SGD(노이즈 주입) + 그래디언트 클리핑                    │
│   연합학습(FedAvg/FedProx) + 보안 집계(Secure Aggregation)   │
├──────────────────────────────────────────────────────────────┤
│ ③ 모델·추론 계층(Privacy-Preserving Inference)               │
│   TEE(Intel TDX·AMD SEV·NVIDIA H100 CC) 기반 추론            │
│   동형암호(CKKS·BFV) 또는 MPC 기반 추론(선택적)              │
├──────────────────────────────────────────────────────────────┤
│ ④ RAG·외부 지식 연계 계층                                    │
│   문서 권한·ABAC 기반 벡터DB 인덱스 분리                     │
│   사용자별 컨텍스트 격리, 출력 필터(Guardrail)               │
├──────────────────────────────────────────────────────────────┤
│ ⑤ 거버넌스 계층                                              │
│   개인정보영향평가(PIA)·ISMS-P·ISO 42001·NIST AI RMF         │
│   AI 학습용 개인정보 활용 특례·가명정보 위험도 차등화        │
└──────────────────────────────────────────────────────────────┘
```

### 기존 LLM 도입 vs 프라이버시 보존 LLM 도입

| 구분 | 기존 LLM 도입 | 프라이버시 보존 LLM(PP-LLM) |
|---|---|---|
| 학습 데이터 처리 | 원본 그대로 학습 | **DP·가명화·연합학습 적용** |
| 모델 위치 | 외부 API(클라우드) | **온프레미스·TEE·연합 학습 노드** |
| 추론 데이터 | 평문 전송 | **TEE·동형암호로 보호** |
| 유출 위험 | 학습·응답 모두 위험 | **수학적·하드웨어적 보장 제공** |
| 규제 정합성 | 가명정보 활용 한계 | **AI 학습 특례·ISMS-P 정합** |
| 비용·성능 | 비용 낮음, 정확도 높음 | 비용·연산 ↑, 정확도 약간 ↓ |

## 3. 기술사 출제 포인트

- **프라이버시 보존 LLM(PP-LLM)**의 개념과 필요성
- **LLM 4대 프라이버시 위협**(학습 데이터 추출·멤버십 추론·모델 인버전·프롬프트 인젝션)
- **차분 프라이버시(Differential Privacy)와 DP-SGD**의 동작 원리, 프라이버시 예산(ε, δ)
- **연합학습(Federated Learning)** 아키텍처(FedAvg/FedProx)와 보안 집계 기법
- **동형암호(Homomorphic Encryption)** 종류(부분·완전·근사 동형)와 LLM 추론 적용 한계
- **신뢰실행환경(TEE·Confidential Computing)** 기반 LLM 추론 보호(Intel TDX·NVIDIA H100 CC)
- **RAG 권한 제어**와 사용자별 벡터DB 인덱스 격리 설계
- **AI 학습용 개인정보 활용 특례**(개인정보보호법) 및 가명정보 위험도 차등화와의 연계
- **ISO 42001·NIST AI RMF·ISMS-P**와의 거버넌스 정합성
- **PP-LLM 도입 시 고려사항**: 유틸리티-프라이버시 트레이드오프, 연산 비용, 성능 검증
