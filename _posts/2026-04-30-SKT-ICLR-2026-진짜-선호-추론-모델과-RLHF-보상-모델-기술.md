## 1. 뉴스 요약

인공지능신문 보도에 따르면, **SKT가 「ICLR 2026(국제표현학습학회)」에서 「진짜 선호(True Preference)」를 찾아내는 AI 추론 모델 기술 논문을 발표**한다. 핵심은 **사람이 매긴 선호 라벨에 섞인 「잡음·일관성 결여·맥락 의존성」을 분리**하고, **LLM 정렬(Alignment)에 활용되는 보상 모델(Reward Model)** 의 학습 데이터를 정제하는 기법이다. 기존 RLHF(Reinforcement Learning from Human Feedback)는 ① 사람 평가자가 두 응답 중 하나를 선택 → ② Bradley–Terry 모델로 보상함수를 학습 → ③ PPO 등 RL로 정책(LLM)을 갱신하는 파이프라인을 따르지만, **선호 라벨에 「가짜 선호(spurious)」가 다수 섞이면 보상 해킹(Reward Hacking)·아첨(Sycophancy)·환각 강화** 같은 부작용이 발생한다. SKT 논문은 **선호 라벨을 잠재 변수(Latent)로 모델링**하고 **불일치(Disagreement) 패턴을 정량화**해 「진짜 선호」와 「작업·문맥에 따른 변동」을 분리하는 베이지안·메타학습 결합 기법을 제안한다. 이는 **DPO(Direct Preference Optimization)·KTO·IPO** 등 차세대 정렬 알고리즘과 결합 시 **데이터 효율·안정성 동반 개선** 효과가 보고되며, **공익적 AI**·**의료/법률 도메인 LLM** 정렬에서 활용도가 높다. AI데이터분석 관점에서 이는 **「데이터 품질 = 정렬 품질」** 이라는 명제를 강화한다.

## 2. 핵심 개념

### RLHF 정렬 파이프라인

```
[사전학습 LLM]
       │
       ▼
[SFT: 지시문-응답 지도학습] ──(τ_SFT)──┐
       │                              │
       ▼                              │
[선호 데이터 수집(쌍 비교)]              │
       │                              │
       ▼                              │
[보상 모델(RM) 학습] ──(Bradley-Terry)─┤
       │                              │
       ▼                              │
[PPO/DPO/KTO로 정책 갱신] ─────────────┘
       │
       ▼
[정렬 완료 LLM (Helpful/Harmless/Honest)]
```

### 보상 모델 학습 알고리즘 비교

| 알고리즘 | 학습 방식 | 특징 | 한계 |
|--------|---------|-----|-----|
| RLHF (PPO) | RM + PPO | 강력한 제어 | 불안정·자원 다소비 |
| DPO | 선호쌍 직접 최적화 | RM 불필요·안정 | 강화 신호 약함 |
| KTO | 단일 응답 호불호 | 라벨링 비용 ↓ | 균형 어려움 |
| IPO | DPO 변형 | 과적합 완화 | 하이퍼 민감 |
| Bradley–Terry | 쌍 비교 확률 모델 | 고전적·수학적 | 노이즈 취약 |

### ‘진짜 선호’ 추론(True Preference Inference) 핵심 아이디어

- **잠재 선호 변수(z)**: 평가자별 일관 선호와 변동 요인을 분리
- **불일치(Disagreement) 모델링**: 평가자 간 합의도·라벨 신뢰도 정량화
- **메타학습**: 작업·도메인·문맥별 선호 변동 보정
- **데이터 정제**: 잡음 라벨 제거·재가중·신호-잡음비 향상

### 정렬(Alignment) 부작용과 대응

- **보상 해킹(Reward Hacking)**: 보상 모델 약점 악용 → 정규화·인-콘텍스트 검증
- **아첨(Sycophancy)**: 사용자 견해 무비판 동조 → 균형 데이터·다관점 학습
- **환각 강화(Hallucination Reinforce)**: 보상 신호가 사실성 무시 → 사실성 평가 결합(RLAIF·Critic)
- **분포 이동(Distribution Shift)**: 학습-운영 분포 격차 → 지속학습·온라인 RM 갱신

### 도입 시 고려사항

- **데이터 품질 관리**: 평가자 가이드, 일관성 점검, 라벨 신뢰도 가중
- **평가 지표**: MT-Bench, AlpacaEval, Arena Hard, HHH 평가 + 도메인 평가
- **안전성**: 레드팀, 자동·수동 적대적 평가, 거부(Refusal) 제어
- **거버넌스**: 데이터셋 카드·모델 카드, 정렬 변경 추적

## 3. 기술사 출제 포인트

- **LLM 정렬(Alignment) 개념과 RLHF 파이프라인 단계**: SFT → RM → RL 흐름
- **보상 모델 학습 기법 비교**: PPO·DPO·KTO·IPO 트레이드오프와 적용 시나리오
- **Bradley–Terry 선호 모델의 수학적 구조와 한계**: 잡음·문맥 의존성 영향
- **선호 데이터 품질 관리 방안**: 평가자 일관성, 라벨 신뢰도, 가짜 선호 정제
- **정렬 부작용(보상 해킹·아첨·환각 강화)과 완화 기법**
- **공익적·도메인 특화 LLM 정렬 시 고려사항**: 의료·법률·금융 평가 체계
- **RLAIF(Reinforcement Learning from AI Feedback)와 비용 효율성**: AI 평가자 활용
- **LLM 평가 벤치마크와 정렬 품질 측정**: MT-Bench, HHH, Arena, 도메인 KPI
- **AI 모델 거버넌스와 정렬 변경 추적**: 모델 카드, 정렬 정책, 변경 이력

## 4. 관련 토픽 연계

- **공공기관 AI 챌린지와 국가 AI 인재양성 거버넌스**: 정렬 인재·평가 체계
- **AI SW 품질관리 역량강화와 AI 시스템 신뢰성 검증 체계**: 정렬 품질 검증 절차
- **AI 모델 증류 공격과 LLM 지식재산 보안**: 정렬 모델 보호 관점
- **LLM 옵저버빌리티와 생성형AI 운영관리 플랫폼**: 정렬 모니터링·드리프트 감지
- **과기정통부 에이전틱 AI 발전전략과 멀티에이전트 시스템 아키텍처 패턴**: 정렬된 에이전트 협업
- **개인정보위 가명정보 위험도 차등화와 비식별 데이터 거버넌스**: 학습 데이터 보호
- **글로벌 AI 기술 수출 통제와 AI 기술 보호주의 거버넌스**: 정렬·평가 표준 경쟁
