## 1. 뉴스 요약

전력 소모를 100배 줄인 '하이브리드 AI' 기술이 등장하여 AI의 '에너지 블랙홀' 문제 해결에 새로운 돌파구를 마련했다. 이 기술은 기존 디지털 연산 방식에 아날로그 연산을 결합한 하이브리드 아키텍처로, 대규모 AI 모델의 추론 과정에서 발생하는 막대한 전력 소비를 획기적으로 절감한다. 데이터센터 전력 수요가 급증하는 가운데, AI 연산의 에너지 효율성은 기술적·경제적·환경적 측면에서 핵심 과제로 부상하고 있다.

## 2. 핵심 개념

### AI 연산의 에너지 소비 구조

```
[AI 모델 학습/추론]
       ↓
┌──────────────────────────────────────────┐
│  에너지 소비 요인                          │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐    │
│  │ 행렬 연산 │ │ 데이터  │ │ 메모리  │    │
│  │ (MAC)    │ │ 이동    │ │ 접근    │    │
│  │ ~30%     │ │ ~40%    │ │ ~30%    │    │
│  └─────────┘ └─────────┘ └─────────┘    │
│       ↓            ↓           ↓         │
│  [연산 에너지] [통신 에너지] [저장 에너지]  │
└──────────────────────────────────────────┘
       ↓
 ★ 데이터 이동이 전체 에너지의 최대 요인
   → 메모리 벽(Memory Wall) 문제
```

### 하이브리드 AI 컴퓨팅 아키텍처 비교

| 구분 | 디지털 연산 | 아날로그 연산 | 하이브리드 AI |
|------|------------|-------------|-------------|
| 연산 방식 | 이진수(0/1) 논리 게이트 | 전압/전류 기반 물리적 연산 | 디지털+아날로그 혼합 |
| 정밀도 | 높음(FP32/FP16) | 낮음(노이즈 존재) | 작업별 적응적 정밀도 |
| 에너지 효율 | 낮음 | 매우 높음(10~100x) | 높음 |
| 프로그래밍 | 용이(SW 기반) | 어려움(하드웨어 의존) | 중간(추상화 레이어) |
| 적합 워크로드 | 제어 흐름, 정밀 연산 | 행렬곱, 패턴 매칭 | AI 추론 전반 |

### 에너지 효율 AI 하드웨어 기술 분류

| 기술 | 원리 | 에너지 절감 | 성숙도 |
|------|------|-----------|--------|
| GPU 아키텍처 최적화 | 텐서 코어, 희소 연산 가속 | 2~5x | 상용화 |
| PIM(Processing-in-Memory) | 메모리 내 연산으로 데이터 이동 제거 | 5~10x | 초기 상용화 |
| 아날로그 CIM(Compute-in-Memory) | ReRAM/PCM 기반 아날로그 MAC 연산 | 10~100x | 연구/시제품 |
| 뉴로모픽 칩 | 스파이킹 신경망, 이벤트 구동 | 100~1000x | 연구/시제품 |
| 광학 연산(Photonic Computing) | 빛의 간섭·회절로 행렬 연산 | 10~100x | 연구 단계 |

### CIM(Compute-in-Memory) 동작 원리

```
[기존 폰 노이만 구조]              [CIM 아키텍처]
 CPU ←→ 메모리 버스 ←→ 메모리      메모리 어레이 내부에서 연산
  │      (병목)          │          ┌─────────────────┐
  │  데이터 이동 비용 큼  │          │ ┌─┐┌─┐┌─┐┌─┐  │
  ↓                      ↓          │ │W││W││W││W│  │ W=Weight
 [고전력 소비]        [대기 전력]    │ └┬┘└┬┘└┬┘└┬┘  │
                                    │  ×  ×  ×  ×   │ ×=곱셈
                                    │  ↓  ↓  ↓  ↓   │
                                    │ [Σ 아날로그 합산] │
                                    └─────────────────┘
                                     → 데이터 이동 없이
                                       행렬곱 완료
```

### 그린 AI(Green AI) 지표

- **FLOPS/Watt**: 와트당 부동소수점 연산 수 — 하드웨어 에너지 효율 측정
- **Carbon Footprint**: 모델 학습·추론 전 과정의 탄소 배출량
- **PUE(Power Usage Effectiveness)**: 데이터센터 총 전력 / IT 장비 전력 (이상적 값: 1.0)
- **Energy-Delay Product(EDP)**: 에너지 소비 × 지연 시간 — 에너지와 성능의 통합 지표

## 3. 기술사 출제 포인트

### 예상 문제
1. **AI 연산의 에너지 효율 향상을 위한 하드웨어 아키텍처 기술을 분류하고, CIM(Compute-in-Memory) 기반 아날로그 연산의 장단점을 분석하시오**
2. **폰 노이만 병목(Von Neumann Bottleneck)이 AI 워크로드에 미치는 영향을 설명하고, 이를 극복하기 위한 비폰노이만(Non-Von Neumann) 아키텍처를 비교하시오**
3. **그린 AI(Green AI) 실현을 위한 기술적·제도적 방안을 하드웨어, 소프트웨어, 데이터센터 관점에서 제시하시오**

### 답안 구성 가이드
- AI 연산의 에너지 소비 구조: 연산(MAC) vs 데이터 이동 vs 메모리 접근 비율 분석
- 폰 노이만 병목과 메모리 벽(Memory Wall) 문제의 본질
- CIM 기술: ReRAM, PCM, MRAM 등 비휘발성 메모리 기반 연산 원리
- 뉴로모픽 컴퓨팅: SNN(Spiking Neural Network), 이벤트 구동 연산의 에너지 효율성
- 소프트웨어 최적화: 모델 양자화, 가지치기, 지식 증류와의 상호 보완 관계
- 데이터센터 관점: PUE 최적화, 액침냉각, 재생에너지 활용

## 4. 관련 토픽 연계

- **AI 모델 양자화**: 소프트웨어 기반 에너지 효율 최적화와 하드웨어 접근의 상호 보완
- **Edge AI**: 저전력 디바이스에서의 AI 추론을 위한 에너지 효율 아키텍처
- **뉴로모픽 컴퓨팅**: 뇌 모방 컴퓨팅의 이벤트 구동 방식과 초저전력 특성
- **AI 반도체**: HBM, PIM, NPU 등 AI 전용 하드웨어의 에너지 효율 설계
- **데이터센터 인프라**: AI 워크로드 증가에 따른 전력·냉각 인프라 설계 과제
